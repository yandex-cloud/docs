# Удаление устаревших метрик (TTL)

Метрики, для которых не поступали новые значения в течение 30 дней, автоматически удаляются из {{monitoring-full-name}}. Автоматическое удаление метрик запускается минимум раз в сутки.

## Системные метрики {#system}

Со всех активных ресурсов {{ yandex-cloud }} автоматически собираются системные метрики. Сбор метрик с конкретного ресурса прекращается после его удаления. Данные метрик для удаленных ресурсов доступны в течение 30 дней. По истечению этого срока данные автоматически удаляются.

> Например, в вашем каталоге существует виртуальная машина. Если удалить эту виртуальную машину `1 августа 2020`, то ее метрики будут доступны в {{monitoring-full-name}} до `31 августа 2020`.

## Пользовательские метрики {#custom}

В [пользовательских метриках](../operations/metric/add.md) могут быть использованы любые временные точки, в том числе в прошлом. Если вы создали метрику с использованием временных точек старше 30 дней, она будет удалена при следующем запуске процедуры автоматического удаления.

> Например, `1 августа 2020` вы через API записали в {{monitoring-full-name}} метрики с временными точками от `1 апреля 2020` до `1 мая 2020`. 
> Эти метрики будут автоматически удалены `2 августа 2020`, так как на момент запуска процедуры автоматического удаления последняя временная точка в метрике старше 30 дней.

{% note tip %}

Для тестирования метрик используйте временные точки в пределах последних 30 дней. Тогда метрика не сразу попадет под условие автоматического удаления.

{% endnote %}
