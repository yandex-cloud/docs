# Настроить доступ к облачным документам

Процедура настройки доступа к облачному документу зависит от способа его создания:

* Используйте настройки доступа вики-страницы, если вы создали новый облачный документ или загрузили файл через интерфейс {{ wiki-name }}.

* Используйте настройки доступа на Sharepoint, если вы создали документ через интерфейс Office 365 и затем встроили его на вики-страницу. 

## Управление доступом через вики-страницу {#wiki-access}

Если вы создали новый облачный документ или загрузили файл через интерфейс {{ wiki-name }}, для изменения прав доступа к документу используйте настройки вики-страницы: нажмите кнопку настройки доступа вверху страницы и выберите [режим доступа](../page-management/access-setup.md).

Пользователи, у которых ограничен доступ к вики-странице, не смогут читать и редактировать встроенный документ через интерфейс {{ wiki-name }}.

{% note alert %}

Права доступа, настроенные на {{ wiki-name }}, синхронизируются с правами доступа на портале Office 365. Если вы настроите права доступа через интерфейс Sharepoint, при синхронизации они будут перезаписаны настройками вики-страницы.

{% endnote %}

## Управление доступом через интерфейс Office 365 {#access-list}

Если вы создали документы через интерфейс портала Office 365 и хотите [вставлять их на вики-страницы](create-ms-office.md#embed-existing), настройте права доступа к документам или к папкам, в которых они находятся. Иначе документы могут быть недоступны для пользователей {{ wiki-name }}.

{% note alert %}

Настройки доступа вики-страницы не действуют для документов, созданных  через интерфейс портала Office 365.
При встраивании такого документа на вики-страницу появится предупреждение «Доступ к этому документу настроил автор на портале Sharepoint».

{% endnote %}

По умолчанию документы, которые хранятся в [личной папке OneDrive]({{ link-sharepoint-personal }}), доступны только автору. Вы можете настроить общий доступ к документу или вашей папке для других пользователей.

Если несколько документов должны быть доступны определенной группе людей, вы можете создать сайт отдела и [перенести документы](doc-access.md#move-doc) из личной папки.

Чтобы настроить права доступа для документа или папки:

1. Найдите документ или папку на портале Sharepoint или в [личной папке OneDrive]({{ link-sharepoint-personal }}).

1. Наведите указатель на документ или папку, нажмите значок ![](../../_assets/wiki/share-opt.png) и выберите **Управление доступом**.

1. На открывшейся панели в блоке **Прямой доступ** введите имя сотрудника или название группы и уровень доступа: **Может изменять** или **Может просматривать**.

   Чтобы указать подразделение на Стаффе, введите последнюю часть ссылки на подразделение. Например, для [<q>Службы документации</q>](https://staff.yandex-team.ru/departments/yandex_infra_tech_doc_dep78367) укажите `yandex_infra_tech_doc_dep78367`.

   {% note alert %}

   Если требуется выдать доступ к папке или сайту широкому кругу пользователей (одна или несколько групп общей численностью более 200–300 человек), получите согласование Службы информационной безопасности. Для этого отправьте запрос через [форму](https://forms.yandex-team.ru/surveys/106156/). 
   Права доступа, выданные на широкую аудиторию без согласования со Службой информационной безопасности, будут отозваны автоматически.

   {% endnote %}

1. Нажмите кнопку **Предоставление доступа**.

### Перенести документ из личной папки на сайт отдела {#move-doc}

По умолчанию документы, которые хранятся в [личной папке OneDrive]({{ link-sharepoint-personal }}), доступны только автору.

Если ваши документы предназначены для доступа широкой аудитории, создайте для их хранения [сайт (раздел) на портале Sharepoint](./create-sharepoint.md#shared-site).

Если ваши документы хранятся в личной папке OneDrive, перед встраиванием на {{ wiki-name }} перенесите их на сайт отдела. Иначе в случае увольнения или отзыва лицензии Office 365 все файлы в личной папке будут удалены.

Чтобы перенести документ:
1. Найдите документ в [личной папке OneDrive]({{ link-sharepoint-personal }}).

1. Наведите указатель на документ, нажмите значок ![](../../_assets/wiki/share-opt.png) и выберите **Переместить**.

1. На открывшейся панели в блоке **Места** выберите **Yandex** → **Сайт группы** → **Документы**, затем выберите папку вашего отдела или проекта. Если такой папки нет, вы можете создать новую.

1. Нажмите кнопку **Переместить сюда**.

### Доступ к документу по ссылке {#share-link}

Если вы хотите, чтобы другие пользователи могли работать с документом в интерфейсе Office 365, вы можете открыть доступ к документу по ссылке.

{% note alert %}

Этот способ не подходит для встраивания документа на {{ wiki-name }}.

{% endnote %}

1. Найдите документ на сайте отдела или в [личной папке OneDrive]({{ link-sharepoint-personal }}). Наведите указатель на документ и нажмите значок ![](../../_assets/wiki/access.png).
  Если документ открыт, выберите меню **Файл** → **Поделиться** → **Поделиться**  (в PowerPoint меню **Файл** → **Общий доступ** → **Совместно с другими людьми**).

1. Чтобы настроить права доступа по ссылке, нажмите кнопку **Указанным пользователям доступно редактирование**.

1. Чтобы отправить ссылку по почте, введите имена пользователей или почтовые адреса и нажмите кнопку **Отправить**.

1. Чтобы скопировать ссылку, нажмите **Копировать ссылку**.
