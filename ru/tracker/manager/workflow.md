# Настроить типы и статусы задач

Как правило, процесс выполнения задачи состоит из нескольких этапов. Каждому этапу в {{ tracker-name }} соответствует статус. Набор допустимых статусов задач и правила перехода между статусами называется _воркфлоу_.

В одну очередь вы можете добавить несколько типов задач, например <q>Новая возможность</q>, <q>{{ ui-key.startrek.blocks-desktop_queue-new-presets.preset--task-type--improvement }}е</q> и <q>{{ ui-key.startrek.blocks-desktop_queue-new-presets.preset--task-type--bug }}</q>. Для каждого типа задач можно выбрать свой воркфлоу. 

Чтобы настроить жизненный цикл для задач определенного типа:

1. Перейдите на [страницу очереди](../user/queue.md).

1. В верхнем правом углу нажмите ![](../../_assets/tracker/svg/settings-old.svg) **{{ ui-key.startrek.ui_components_PageQueue_header.settings }}**.

    {% note alert %}

    По умолчанию настраивать очередь может [только ее владелец](queue-access.md).

    {% endnote %}

1. [Добавьте в очередь тип задач](add-ticket-type.md) и выберите для него воркфлоу и набор резолюций.
   
   Вы можете выбрать существующий [воркфлоу из шаблона очереди](workflows.md) или [создать новый воркфлоу](add-workflow.md). При этом один и тот же воркфлоу можно использовать для нескольких типов задач.

1. Если вы создали новый воркфлоу, [задайте доступные статусы задач](workflow-status-edit.md).

1. Для каждого статуса [настройте переходы в другие доступные статусы](workflow-action-edit.md):
    - Порядок и условия перехода между статусами.
    - Дополнительные действия при переходе между статусами. Например, автоматическое изменение значений полей задачи.
    - Экраны переходов — всплывающие окна с заданными полями, которые отображаются при изменении статуса задачи.
