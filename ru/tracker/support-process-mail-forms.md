# Принимать обращения по почте и через форму

Чтобы обрабатывать обращения пользователей в {{ tracker-name }}, из них нужно создавать задачи. Большинство пользователей привыкло отправлять обращения по почте или через сайт. К тому же у {% if audience == "external" %}пользователей, которые не являются сотрудниками вашей организации,{% else %}внешних пользователей{% endif %} нет доступа к {{ tracker-name }}.

Поэтому рекомендуем настроить прием обращений по почте или через форму, созданную в сервисе [{{ forms-full-name }}]({{ link-forms }}).

### Принимать обращения по почте {#mail}

Чтобы принимать обращения пользователей по почте, {% if audience == "external" %}настройте почтовый адрес очереди{% else %}[настройте почтовый адрес для очереди](manager/queue-mail.md#sec-mail-yandex), в которой будут создаваться задачи по обращениям пользователей{% endif %}. Из писем, которые придут на этот адрес, будут автоматически создаваться задачи в {{ tracker-name }}.

{% if audience == "external" %}

1. Перейдите в настройки очереди и в разделе **Почтовые ящики** [добавьте адрес очереди](manager/queue-mail.md#section_gwv_hqb_hgb).

    Если добавление адреса очереди недоступно, значит у вашей организации нет домена. Домен нужен для создания почтовых ящиков и рассылок, в том числе чтобы создать адрес очереди. Домен можно бесплатно [подключить в сервисе Яндекс 360 для бизнеса]({{ support-business-domain }}).

1. Если ваши пользователи не являются сотрудниками организации, [разрешите принимать письма с внешних ящиков](manager/queue-mail.md#mail_tasks).

{% endif %}

Если задача создана из письма, вы можете отвечать пользователю со страницы задачи, например запросить дополнительную информацию или сообщить о решении проблемы. Для этого [настройте отправку писем из задач](#send_mail).

### Принимать обращения через форму {#form}

Принимать обращения через форму удобно, если нужно получить от пользователя определенные данные. В форме можно задать список вопросов, указать, какие из них обязательны, и сохранить ответы на вопросы в параметрах задачи в {{ tracker-name }}.

1. Перейдите в сервис [{{ forms-full-name }}]({{ link-forms }}) и создайте новую форму.

1. Добавьте на форму вопросы, чтобы пользователь мог сообщить информацию, которая нужна для регистрации обращения.

1. Настройте для формы [интеграцию с {{ tracker-name }}](../forms/create-task.md):

    1. Укажите очередь и другие параметры задачи.

    1. В поле **Описание задачи** добавьте ответы на вопросы формы.

    1. Чтобы сохранить в параметрах задачи ответ на определенный вопрос формы, добавьте поле задачи, нажмите **Переменные** → **Ответ на вопрос** и выберите вопрос.

    1. Сохраните параметры интеграции.

1. [Опубликуйте](../forms/publish.md#section_link) форму — встройте ее в сайт или открывайте по ссылке.
