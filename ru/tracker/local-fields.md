# Локальные поля задач

Если требуется добавить в задачи новый параметр, которого нет среди существующих в {{ tracker-name }} полей, вы можете добавить локальные поля в вашу очередь.

{% note info %}

Список существующих полей можно посмотреть на странице [Настройки {{ tracker-name }}](https://st.yandex-team.ru/admin/fields).

{% endnote %}

Локальное поле можно использовать только в задачах той очереди, к которой оно привязано. Преимущество локальных полей в том, что владелец очереди может управлять такими полями без риска повлиять на процессы работы в других очередях. Пользователи, которые работают в других очередях, не будут видеть это поле в своих задачах.

## Ограничения локальных полей {#restrictions}

#### Где можно использовать локальные поля

- На странице задачи: локальные поля можно [использовать](user/edit-ticket.md#section_jqw_ppn_jz) так же, как стандартные поля задачи.

- В фильтрах задач: локальное поле можно указать в [условиях фильтра](user/create-filter.md) или в [языке запросов](user/query-filter.md#local_fields), если указана очередь, к которой привязано поле.

#### Где нельзя использовать локальные поля

Локальные поля временно не поддерживаются в некоторых сценариях работы с задачами. Поддержка этих сценариев будет добавлена позже.

- Локальные поля не отображаются на [карточках на доске задач](manager/edit-agile-board.md#sec_layout).

- Локальные поля нельзя использовать в [триггерах](user/trigger.md), [автодействиях](user/autoactions.md), [макросах](manager/create-macroses.md), [правилах SLA](sla-head.md), [функциях перехода между статусами](manager/workflow-action-edit.md).

- Значения локальных полей нельзя изменять помощью [групповых операций](manager/bulk-change.md).

- Значения локальных полей нельзя изменять через [API](user/API.md).

- Локальные поля не выгружаются в YT.

#### Перенос и клонирование задачи

При [переносе](user/edit-ticket.md#section_xwx_qpn_jz) и [клонировании](user/clone.md) задачи с локальными полями в другую очередь значения локальных полей будут удалены.

## Как добавить локальное поле {#add}

Отправьте заявку на добавление нового поля через форму ниже. В комментарии к заявке напишите, что хотите добавить локальное поле.

Обращения обрабатываются в течение 2-3 рабочих дней. Если задача срочная, поднимите ее приоритет до <q>критичного</q> и напишите в комментариях, почему это срочно.
