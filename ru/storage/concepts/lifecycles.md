# Жизненные циклы объектов в бакете

{{ objstorage-name }} позволяет определить действия, которые автоматически производятся с отдельными объектами или группами объектов в бакете в заданные моменты времени.

Виды действий:

- Изменение [класса хранилища](storage-class.md) для объектов.  
- Удаление объектов.

Задать конфигурацию жизненных циклов объектов можно только для каждого отдельного бакета. Нельзя задать конфигурацию жизненных циклов для группы бакетов, каталога или облака. 

Для управления жизненными циклами объектов можно применять:

- [Консоль управления](../operations/buckets/lifecycles.md) Яндекс.Облака.
- [HTTP API, совместимый с Amazon S3](../s3/index.md).

    Таким образом, конфигурацией жизненных циклов можно управлять при помощи [инструментов](../tools/index.md), поддерживающих  HTTP API Amazon S3.

При управлении жизненными циклами с помощью HTTP API, совместимого с Amazon S3, задайте [конфигурацию в формате XML](../s3/api-ref/lifecycles/xml-config.md). Различные инструменты могут требовать другие форматы конфигурации, смотрите пример для AWS CLI в разделе [{#T}](../operations/buckets/lifecycles.md).
