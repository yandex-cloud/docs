# Собственный домен

Для публикации сайта вы можете использовать собственный домен, например, `example.com`.

Чтобы поддержать собственный домен:

- Назовите бакет так же, как домен.

- Создайте в [{{ dns-name }}](../../../dns/operations/resource-record-create.md) запись: 

    ```
    example.com ANAME example.com.{{ s3-web-host }}
    ```

    Рекомендуется использовать [записи ANAME](../../../dns/concepts/resource-record.md#aname) в {{ dns-name }} при размещении сайта {{ objstorage-name }}. Такие записи позволяют использовать для хостинга домены второго уровня и, в отличие от CNAME, не ограничивают использование других типов записей в одной с ними зоне. 

    Если требуется, вы можете использовать записи CNAME на внешних DNS-серверах. В этом случае используйте для хостинга в Object Storage домены не ниже третьего уровня — это связано с особенностями обработки CNAME-записей на DNS-хостингах. Подробнее см. п. 2.4 [RCF 1912](https://www.ietf.org/rfc/rfc1912.txt).

{% note info %}

По умолчанию сайт доступен только по протоколу HTTP. Чтобы поддержать для сайта протокол HTTPS, [загрузите собственный сертификат безопасности](certificate.md) в {{ objstorage-name }}.

{% endnote %}
