# Node.js-артефакт

_Node.js-артефакт_ — это пакет, который содержит код и информацию о нем, хранящийся в формате [npm](https://www.npmjs.com/) реестра. [Артефакты](artifacts.md) используются для распространения библиотек, фреймворков и других компонентов, которые можно добавить в проекты на различных языках и технологиях JavaScript-экосистемы.

Артефакты из npm-реестра могут использоваться с различными менеджерами пакетов: [npm](https://www.npmjs.com/), [yarn](https://yarnpkg.com/) и [pnpm](https://pnpm.io/).

Node.js-артефакты поддерживают не только проекты на Node.js, но и проекты на других технологиях JavaScript-экосистемы, включая:
* [JavaScript](https://ru.wikipedia.org/wiki/JavaScript) — основной язык экосистемы
* [TypeScript](https://www.typescriptlang.org/) — типизированный надмножество JavaScript
* [Node.js](https://ru.wikipedia.org/wiki/Node.js) — серверная платформа для JavaScript
* [React](https://react.dev/) — библиотека для создания пользовательских интерфейсов
* [Vue.js](https://vuejs.org/) — фреймворк для создания пользовательских интерфейсов
* [Angular](https://angular.io/) — платформа для веб-приложений
* Другие фреймворки и библиотеки JavaScript-экосистемы

## Менеджеры пакетов {#package-managers}

Артефакты из npm-реестра могут использоваться с различными менеджерами пакетов:

### npm {#npm-inf}

[npm](https://www.npmjs.com/) (Node Package Manager) — это менеджер пакетов для [JavaScript](https://ru.wikipedia.org/wiki/JavaScript) и стандартный менеджер пакетов для Node.js. Он помогает разработчикам легко устанавливать, обновлять и управлять зависимостями в своих проектах. С его помощью можно искать, загружать и публиковать пакеты, а также создавать новые.

В Node.js npm уже имеется по умолчанию, поэтому его отдельная установка не требуется.

Для управления артефактами проекта npm использует файл `package.json`, в котором описываются зависимости проекта.

### yarn {#yarn-inf}

[yarn](https://yarnpkg.com/) — это быстрый, надежный и безопасный менеджер пакетов для JavaScript, созданный компанией Facebook (Meta). Yarn совместим с npm-реестрами и может использовать те же пакеты, что и npm.

Yarn использует файл `package.json` для описания зависимостей, но также создает файл `yarn.lock` для фиксации точных версий зависимостей, что обеспечивает воспроизводимость сборок.

### pnpm {#pnpm-inf}

[pnpm](https://pnpm.io/) — это быстрый и эффективный менеджер пакетов для JavaScript, который использует жесткие ссылки для экономии дискового пространства. pnpm полностью совместим с npm-реестрами и может использовать те же пакеты.

pnpm использует файл `package.json` для описания зависимостей и создает файл `pnpm-lock.yaml` для фиксации версий зависимостей.

## Формат артефактов {#artifact-format}

Артефакты в npm — это пакеты (packages), которые представляют собой директории с файлами, содержащими код и метаданные. Каждый пакет должен содержать файл `package.json` с описанием пакета, его зависимостей и метаданных.

Примеры типов артефактов:
* Модули (packages) — библиотеки и утилиты
* Зависимости (dependencies) — пакеты, необходимые для работы приложения
* devDependencies — пакеты, необходимые только для разработки
* peerDependencies — пакеты, которые должны быть установлены в проекте-потребителе

Подробнее о менеджерах пакетов читайте в документации:
* [npm](https://docs.npmjs.com/)
* [yarn](https://yarnpkg.com/getting-started)
* [pnpm](https://pnpm.io/motivation)
