# OCI-артефакт в {{ cloud-registry-full-name }}

_OCI-артефакт_ — это артефакт, хранящийся в соответствии со спецификацией [Open Container Initiative (OCI)](https://opencontainers.org/). OCI-артефакты обеспечивают единый стандарт для хранения различных типов артефактов в реестрах, что обеспечивает совместимость между различными инструментами и платформами.

{{ cloud-registry-name }} поддерживает спецификацию OCI версии [1.1](https://opencontainers.org/posts/blog/2024-03-13-image-and-distribution-1-1), которая определяет стандартный формат для упаковки, хранения и распространения артефактов.

## Преимущества OCI-формата {#benefits}

* **Единый стандарт**: OCI обеспечивает единообразный способ хранения различных типов артефактов
* **Совместимость**: OCI-артефакты совместимы с широким спектром инструментов и платформ
* **Гибкость**: В формате OCI можно хранить не только контейнерные образы, но и другие типы артефактов, такие как Helm-чарты, WASM-модули и другие
* **Стандартизация**: Использование открытого стандарта упрощает интеграцию с различными экосистемами

## Типы артефактов в формате OCI {#oci-artifact-types}

В {{ cloud-registry-name }} в формате OCI хранятся следующие типы артефактов:

* [Docker-образы](docker.md) — контейнерные образы, совместимые с Docker
* [Helm-чарты](helm.md) — пакеты для развертывания приложений в Kubernetes
* Другие артефакты, соответствующие спецификации OCI

## Управление версиями {#version}

Управление версиями OCI-артефакта осуществляется с помощью тегов и хешей, аналогично другим типам артефактов в реестре.

_Тег_ — присваивается пользователем. Тег должен быть уникален в пределах одного [реестра](registry.md) и может быть изменен. Тег позволяет обращаться к артефакту по понятному имени версии.

_Хеш_ — генерируется автоматически при загрузке артефакта в реестр, является уникальным и однозначно определяет конкретную версию артефакта. Хеш обеспечивает неизменяемость и целостность артефакта.

Обратиться к определенной версии OCI-артефакта можно одним из способов:
* `<реестр>/<имя_артефакта>:<тег>`
* `<реестр>/<имя_артефакта>@<хеш>`

OCI-артефакт и все его версии хранятся в реестре в соответствии со спецификацией OCI 1.1.
