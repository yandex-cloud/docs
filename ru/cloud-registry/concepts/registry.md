# Реестр в {{ cloud-registry-full-name }}

_Реестр_ — это централизованное хранилище и инструмент управления различными файлами, которые используются при разработке и развертывании приложений.

В каждом реестре может храниться набор версий пакетов, связанных с определенными ресурсами. Для получения и публикации пакетов воспользуйтесь соответствующими для данного вида артефактов инструментами (Подробнее в разделе [Артефакты](artifacts.md)).

С помощью реестра можно быстро получить доступ к необходимым артефактам и управлять их версиями. Использование реестра упрощает процесс управления зависимостями между компонентами проекта. Эта функция полезна для команд разработчиков, работающих над крупными и сложными проектами.

Реестры размещены в одних дата-центрах с вашей инфраструктурой, что обеспечивает быструю работу без лишних затрат на трафик.

## Типы реестров в {{ cloud-registry-name }} {#registry-types}

В зависимости от места хранения артефактов реестры {{ cloud-registry-name }} могут быть локальными, удаленными или виртуальными.

### Локальные реестры {#local-registry}

_Локальные реестры_ - это хранилища для артефактов, созданных внутри вашей [организации](../../organization/concepts/organization.md). Локальные реестры позволяют хранить, безопасно распространять и управлять внутренними артефактами разработки.

**Основное предназначение:**
- Хранение приватных внутренних артефактов, созданных вашей организацией
- Контроль версий и управление жизненным циклом артефактов
- Обеспечение быстрого и безопасного доступа к артефактам без зависимости от внешних сервисов

### Удаленные реестры {#remote-registry}

_Удаленные реестры_ - это прокси-хранилища для внешних публичных репозиториев, таких как [Docker Hub](https://hub.docker.com/). При обращении к удаленному реестру артефакты автоматически кешируются локально в вашей инфраструктуре после первого запроса.

**Основное предназначение:**
- Прозрачный доступ к артефактам из внешних репозиториев через единую точку входа
- Кеширование артефактов для ускорения последующих запросов
- Снижение нагрузки на внешние репозитории и уменьшение зависимости от их доступности
- Оптимизация трафика и ускорение развертывания за счет использования локального кеша

### Виртуальные реестры {#virtual-registry}

_Виртуальные реестры_ - это абстракция, которая объединяет под одним именем несколько локальных и/или удаленных реестров, предоставляя единую точку доступа для поиска и скачивания артефактов. При обращении к виртуальному реестру система автоматически ищет запрашиваемый артефакт во всех связанных реестрах в порядке их приоритета.

**Основное предназначение:**
- Упрощение управления доступом к артефактам через единую точку входа
- Абстракция от физического местоположения реестров для пользователей и приложений
- Объединение локальных и удаленных реестров в единое логическое хранилище
- Гибкая организация доступа к артефактам с возможностью изменения конфигурации без изменения настроек клиентов
