# Python-артефакт

_Артефакт Python_ — это файл пакета, содержащий архив с кодом на языке программирования [Python](https://www.python.org/), метаданные, зависимости и другие ресурсы, хранящийся в формате [PyPI](https://pypi.org/) (Python Package Index) реестра. [Артефакты](artifacts.md) используются для распространения библиотек, инструментов и компонентов, которые можно интегрировать в проекты на языке Python.

Артефакты из PyPI-реестра могут использоваться с различными инструментами управления пакетами: [pip](https://pip.pypa.io/), [pipenv](https://pipenv.pypa.io/), [poetry](https://python-poetry.org/), [conda](https://docs.conda.io/) и другими.

Python-артефакты используются в различных областях разработки, включая:
* Веб-разработку — фреймворки [Django](https://www.djangoproject.com/), [Flask](https://flask.palletsprojects.com/), [FastAPI](https://fastapi.tiangolo.com/)
* Научные вычисления — библиотеки [NumPy](https://numpy.org/), [Pandas](https://pandas.pydata.org/), [SciPy](https://scipy.org/)
* Машинное обучение — фреймворки [TensorFlow](https://www.tensorflow.org/), [PyTorch](https://pytorch.org/), [scikit-learn](https://scikit-learn.org/)
* Автоматизацию и DevOps — инструменты для развертывания и управления инфраструктурой
* Другие области применения Python-экосистемы

## Инструменты управления пакетами {#package-managers}

Артефакты из PyPI-реестра могут использоваться с различными инструментами управления пакетами:

### pip {#pip-inf}

[pip](https://pip.pypa.io/) — это стандартный менеджер пакетов для Python, который устанавливается вместе с Python. pip позволяет устанавливать, обновлять и удалять пакеты из PyPI и других репозиториев.

Для управления артефактами проекта pip использует файл `requirements.txt`, в котором описываются зависимости проекта. pip также поддерживает файл `pyproject.toml` для более современного подхода к управлению зависимостями.

### pipenv {#pipenv-inf}

[pipenv](https://pipenv.pypa.io/) — это инструмент для управления зависимостями Python-проектов, который объединяет pip и virtualenv. pipenv автоматически создает и управляет виртуальными окружениями, а также фиксирует точные версии зависимостей.

pipenv использует файлы `Pipfile` и `Pipfile.lock` для описания и фиксации зависимостей проекта.

### poetry {#poetry-inf}

[poetry](https://python-poetry.org/) — это современный инструмент для управления зависимостями и сборки Python-пакетов. Poetry обеспечивает простое управление зависимостями, версионирование и публикацию пакетов.

poetry использует файл `pyproject.toml` для описания зависимостей проекта и создает файл `poetry.lock` для фиксации точных версий зависимостей.

### conda {#conda-inf}

[conda](https://docs.conda.io/) — это менеджер пакетов и окружений, который может работать как с пакетами из репозитория conda, так и с пакетами из PyPI. conda особенно популярен в научных вычислениях и машинном обучении, так как позволяет управлять не только Python-пакетами, но и бинарными зависимостями.

conda использует файл `environment.yml` для описания зависимостей окружения.

## Формат артефактов {#artifact-format}

Артефакты в PyPI — это пакеты в различных форматах, которые содержат код, метаданные и зависимости. Основные форматы артефактов:

* [whl (wheel)](https://packaging.python.org/en/latest/specifications/binary-distribution-format/) — современный формат бинарных дистрибутивов Python, который обеспечивает быструю установку пакетов
* [tar (tarball)](https://ru.wikipedia.org/wiki/Tar) — архив исходного кода в формате `.tar.gz` или `.tar.bz2`, содержащий исходные файлы пакета
* [egg](https://setuptools.readthedocs.io/en/latest/deprecated/python_eggs.html) — устаревший формат, замененный на wheel

Каждый пакет должен содержать файл метаданных (например, `PKG-INFO` или информацию в `pyproject.toml`), который описывает пакет, его зависимости и другую информацию.

Примеры типов артефактов:
* Библиотеки — модули и пакеты для использования в других проектах
* Приложения — исполняемые программы и утилиты
* Инструменты разработки — утилиты для разработки и тестирования
* Плагины и расширения — дополнения для других пакетов и фреймворков
