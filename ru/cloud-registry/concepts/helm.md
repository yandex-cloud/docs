# Helm-чарт в {{ cloud-registry-full-name }}

_Helm-чарт_ — это пакет, содержащий все необходимые ресурсы для развертывания приложения в [Kubernetes](https://kubernetes.io/ru/). Helm-чарт представляет собой набор файлов, описывающих связанный набор ресурсов Kubernetes: шаблоны манифестов, значения конфигурации, метаданные и зависимости.

[Helm-чарт](https://helm.sh/docs/topics/charts/) состоит из шаблонов, значений конфигурации и метаданных.

* Шаблоны определяют структуру ресурсов Kubernetes, которые будут созданы при установке чарта.
* Значения конфигурации позволяют настраивать параметры развертывания без изменения шаблонов.
* Метаданные содержат информацию о чарте: название, версию, описание и зависимости от других чартов.

## Управление версиями {#version}

Управление версиями Helm-чарта осуществляется с помощью версий и хешей.

_Версия_ — присваивается пользователем при создании или упаковке чарта. Версия должна соответствовать формату [Semantic Versioning](https://semver.org/lang/ru/) (например, `1.2.3`) и должна быть уникальна для каждого чарта с одинаковым именем в пределах одного [реестра](registry.md). Версия указывается в файле `Chart.yaml` чарта и может быть изменена при создании новой версии.

У одного Helm-чарта может быть несколько версий. Каждая версия представляет собой отдельный релиз чарта с определенным набором изменений и функциональности.

_Хеш_ — генерируется автоматически при загрузке чарта в реестр, является уникальным и однозначно определяет конкретную версию Helm-чарта.

Обратиться к определенной версии Helm-чарта можно одним из способов:
* `<реестр>/<имя_чарта>:<версия>`
* `<реестр>/<имя_чарта>@<хеш>`

Helm-чарт и все его версии хранятся в реестре в формате OCI (Open Container Initiative), что обеспечивает совместимость с различными инструментами управления контейнерами.
