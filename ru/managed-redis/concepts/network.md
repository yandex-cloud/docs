# Сеть и кластеры {{ mrd-short-name }}

{% if audience != "internal" %}

При создании кластера вы можете:

- задать сеть для самого кластера;
- задать подсети для каждого из хостов кластера.

Вы можете создать кластер, не задавая подсети для хостов, если выбранная для каждого хоста зона доступности содержит ровно одну подсеть указанной сети.

## Сетевой доступ к кластеру {{ mrd-short-name }} {#network-access-to-a-cluster}

Подключиться к кластеру {{ RD }} можно только с той виртуальной машины {{ yandex-cloud }}, которая подключена к той же сети, что и кластер. Получить публичный IP-адрес для хоста в таком кластере невозможно.

{% else %}

Все кластеры создаются внутри нашей сети, куда уже [есть доступ](../../mdb/access.md#network-access) из большинства сетей и от большинства сотрудников. Если у вас нет доступа, запросите доступ в [Панчере](https://puncher.yandex-team.ru/) к макросу `_PGAASINTERNALNETS_`. Для подключения к {{ RD }} в заявке нужно указать порты 26379 (Sentinel) и 6379 (Redis).

{% endif %}

## Имя хоста и FQDN {#hostname}

Имя для каждого хоста в кластере {{ mrd-short-name }} генерирует при его создании. Это имя будет являться доменным именем хоста (FQDN). Имя хоста и, соответственно, FQDN невозможно изменить.

{% if audience != "internal" %}

FQDN можно использовать для доступа к хосту внутри облачной сети.

{% endif %}
