# Настроить квоты для клиентов

Вы можете настроить квоты для [облаков](#cloud-quotas) клиентов, чтобы ограничить использование сервисов. Примеры использования квот:

* Полное ограничение использования DNS в облаке клиента. Для этого установите значение `0` в квотах {{ dns-name }}.
* Ограничение количества ядер для {{ ydb-name }}. Для этого укажите нужное значение в квоте `ydb.dedicatedComputeCores.count`.
* Ограничение количества виртуальных машин в облаке. Для этого укажите нужное значение в квоте `compute.instances.count`.

Подробнее о квотах см. в разделе «Квоты и лимиты» сервисов.

{% note info %}

Квоты доступны только для облаков, связанных с вашим сабаккаунтом.  

Вы можете самостоятельно уменьшить квоту на сервис без согласования с поддержкой. Если новое значение превышает текущую квоту, система автоматически создаст запрос в техническую поддержку.

Для восстановления исходных значений квот обратитесь в [техническую поддержку]({{ link-console-support }}).

Если облако будет переведено на другой сабаккаунт или прямой биллинг-аккаунт, значения квот не вернутся к исходным.

{% endnote %}

## Настроить квоты облака {#cloud-quotas}

{% list tabs group=instructions %}

- Партнерский портал {#partner}

  1. Перейдите на [партнерский портал]({{ link-cloud-partners }}).
  1. На панели слева выберите ![icon](../../_assets/console-icons/credit-card.svg) **Сабаккаунты**.
  1. Выберите нужный сабаккаунт из списка.
  1. Перейдите на вкладку ![icon](../../_assets/console-icons/chart-donut.svg) **Квоты и инстансы**.
  1. Выберите нужное облако из списка и нажмите **Перейти**.
  1. На странице **Квоты облака** воспользуйтесь строкой поиска, чтобы найти нужный сервис.
  1. Напротив нужного сервиса нажмите ![icon](../../_assets/console-icons/chevron-down.svg).
  1. Выберите одну или несколько квот и нажмите **Изменить**.
  1. В открывшемся окне измените значение.
  1. Нажмите **Изменить**.

{% endlist %}
