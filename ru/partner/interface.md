# Обзор интерфейса партнерского портала

## Раздел «Заказчики»

В разделе «Заказчики» находятся сведения о клиентах партнера-интегратора. Здесь можно посмотреть статистику потребления ресурсов заказчиками за неделю, предоставить заказчику дополнительные услуги Облака, а также зарегистрировать новых заказчиков, приостановить и возобновить действие договоров с заказчиками. 

У заказчика может быть один из трех статусов:

* **Активный заказчик** — с заказчиком есть действующий договор. Партнер ежемесячно оплачивает ресурсы Облака, потребленные заказчиком
* **Неподтвержденный** — заказчик еще не подтвердил согласие на работу с партнером-интегратором. Чтобы начать потребление ресурсов Облака заказчик должен нажать кнопку **Подтвердить** в письме-подтверждении регистрации. 
* **Бывший заказчик** — с заказчиком заключен договор, работа по которому приостановлена.

Чтобы перейти к просмотру подробных сведений о заказчике, выберите его в списке.

### Вкладка «Обзор»

На вкладке находится основная информация о заказчике. Здесь можно [приостановить оказание услуг](operations/stop-service.md) заказчику и [добавить](operations/new-cloud.md) для него новое облако.

### Вкладка «Потребление услуг» 

На этой вкладке находится подробная детализация потребления облачных ресурсов заказчиком. 

Данные можно отсортировать по продуктам, выставить фильтры по сервисам и датам. 

### Вкладка «Пользователи»

На этой вкладке можно добавлять новых пользователей и настраивать их роли в облаке заказчика.

Чтобы создать нового пользователя, нажмите кнопку **Добавить пользователя**. 

Чтобы изменить роли существующего пользователя, нажмите кнопку **Настроить роли** в строке нужного пользователя.

Пользователю можно назначить одну из следующих ролей: 

* `Viewer` — пользователь сможет просматривать списки существующих облачных ресурсов в облаке заказчика.
* `Editor` — пользователь сможет создавать и изменять облачные ресурсы в облаке заказчика.

## Раздел «Управление доступом»

В разделе можно добавить нового пользователя партнерского портала. Для регистрации нового пользователя партнерского портала можно использовать только учетные записи Яндекс.Паспорта.

## Раздел «Вознаграждения»

В зависимости от объема потребленных заказчиками ресурсов интегратор получает вознаграждения — грант за удержание и прирост потребления. 

В разделе **Вознаграждения** отображается статистика вознаграждений интегратора в разрезе премий за прирост и удержание, а также графики изменения размеров вознаграждений по месяцам. 

## Раздел «Недельное потребление»

В разделе **Недельное потребление** находится статистика потребления ресурсов в облаках заказчиков по дням за последнюю неделю.
