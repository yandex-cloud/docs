# Удалить целевую группу

{% note alert %}

Перед удалением целевой группы ее необходимо отключить от сетевого балансировщика.
После удаления целевую группу невозможно восстановить.

{% endnote %}

{% list tabs %}

- Консоль управления
  
  Чтобы удалить [целевую группу](../concepts/target-resources.md):
  
  1. В [консоли управления]({{ link-console-main }}) выберите каталог, где требуется удалить целевую группу.
  1. В списке сервисов выберите **{{ network-load-balancer-name }}**.
  1. На панели слева выберите ![image](../../_assets/trgroups.svg) **Целевые группы**.
  1. В строке целевой группы, которую требуется удалить, нажмите значок ![image](../../_assets/horizontal-ellipsis.svg) .
  1. В открывшемся меню выберите **Удалить**.
  1. Подтвердите действие.
  
- CLI
  
  Если у вас еще нет интерфейса командной строки {{ yandex-cloud }}, [установите его](https://cloud.yandex.ru/docs/cli/quickstart#install).
  
  {% include [default-catalogue](../../_includes/default-catalogue.md) %}
  
  1. Посмотрите описание команды CLI для удаления целевой группы:
  
     ```
     yc load-balancer target-group delete --help
     ```
  
  1. Получите список целевых групп:
  
     ```
     yc load-balancer target-group list
     ```
	 
	 Результат:
	 
	 ```
     +----------------------+-------------------+-------+
     |          ID          |       NAME        |  ...  |
     +----------------------+-------------------+-------+
     ...
     | b7roi767je4c574iivrk | test-target-group |  ...  |
     ...
     +----------------------+-------------------+-------+
     ```
  
  1. Выберите идентификатор (`ID`) или имя (`NAME`) нужной целевой группы.
  1. Удалите целевую группу из каталога по умолчанию:
  
     ```
     yc load-balancer target-group delete b7roi767je4c574iivrk
     ```
  
- API
  
  Удалить целевую группу можно с помощью метода API [delete](../api-ref/TargetGroup/delete.md).
