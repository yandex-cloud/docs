# Мониторинг

Cетевой балансировщик отправляет следующие метрики в сервис [{{ monitoring-full-name }}](../../monitoring/concepts/index.md):
* `processed_bytes` — трафик в байтах, прошедший через балансировщик в исходящем (egress) и входящем (ingress) направлениях. С учетом IP-заголовков. За вычетом трафика для проверок состояния.
* `processed_packets` — трафик в пакетах, прошедший через балансировщик в исходящем (egress) и входящем (ingress) направлениях. С учетом IP-заголовков. За вычетом трафика для проверок состояния.

{% note info %}

Следует помнить, что метрики в сервисе мониторинга можно будет увидеть только после их отправки со стороны наблюдаемого объекта. Так, перечисленные выше метрики будут отправляться сетевым балансировщиком только при прохождении сетевого трафика через него.

{% endnote %}
