# Правила тарификации для {{ mms-name }}

## Статус кластера {#running-stopped}

В зависимости от статуса кластера тарифы применяются различным образом:

* Кластер запущен (`Running`) — тарифицируются лицензии ПО, вычислительные ресурсы, объем хранилища и резервных копий.
* Кластер остановлен (`Stopped`) — объем хранилища и резервных копий тарифицируется в полном объеме, а для лицензий ПО применяются правила, описанные в разделе [{#T}](#license). Вычислительные ресурсы не тарифицируются.

  {% note alert %}

  Если кластер использует локальное хранилище (`local-ssd`), вычислительные ресурсы кластера не освобождаются при его остановке и тарифицируются в полном объеме.

  {% endnote %}

### Использование лицензий ПО {#license}

Использование лицензий тарифицируется на авансовой основе. Это означает, что плата взимается в начале периода тарификации еще до фактического потребления ресурсов сервиса. При этом:

* Период тарификации — один календарный месяц.
* Если в течение периода тарификации вы создали новый кластер, восстановили кластер из резервной копии или запустили существующий кластер впервые с начала периода, то взимается полная стоимость лицензий за весь период тарификации.

    > Например, если вы запустили кластер 15 марта впервые с начала месяца, в этот момент спишется стоимость лицензии за весь март. Следующее списание произойдет 1 апреля — за весь апрель.

* Если в течение периода тарификации вы остановили или удалили кластер, стоимость лицензий в этом периоде не возвращается.
* Если в течение периода тарификации вы изменили конфигурацию кластера и снизили потребление ресурсов, стоимость освободившихся лицензий в этом периоде не возвращается. Новая стоимость начинает действовать с первого числа следующего периода тарификации.
* Если в течение периода тарификации вы изменили конфигурацию кластера и увеличили потребление ресурсов, то вам понадобятся дополнительные лицензии. При этом взимается полная стоимость этих лицензий за весь период тарификации независимо от даты внесения изменений.
* Многохостовый кластер {{ MS }} с [нечитаемыми вторичными репликами](../concepts/replication.md#readable-and-non-readable-replicas) требует лицензии на Microsoft SQL Server Enterprise только для первичной реплики.
* Многохостовый кластер {{ MS }} с [читаемыми вторичными репликами](../concepts/replication.md#readable-and-non-readable-replicas) требует лицензии на Microsoft SQL Server Enterprise для каждой реплики.

### Использование хостов БД {#rules-hosts-uptime}

Стоимость использования хостов БД вычисляется в соответствии с их классом. Точные характеристики классов приведены в разделе [{#T}](../concepts/instance-types.md). Плата за использование взимается в порядке, предусмотренном заключенным договором.

Минимальная единица тарификации — минута (например, стоимость 1,5 минут работы хоста равна стоимости 2 минут). Время, когда хост БД не может выполнять свои основные функции, не тарифицируется.

### Использование дискового пространства {#rules-storage}

Плата за использование дискового пространства взимается в порядке, предусмотренном заключенным договором. Оплачивается:

* Объем хранилища, выделенный для кластеров БД.

    * Хранилище на быстрых локальных дисках (`local-ssd`) можно заказывать только для кластеров с тремя хостами и более:
      {% if product == "yandex-cloud" %}* для платформ Intel Broadwell и Intel Cascade Lake — с шагом 100 ГБ;{% endif %}
       * для платформы Intel Ice Lake — с шагом {{ local-ssd-v3-step }}.

    * Хранилище на нереплицируемых сетевых дисках (`network-ssd-nonreplicated`) можно заказывать только для кластеров с тремя хостами и более, с шагом 93 ГБ.

* Объем, занимаемый резервными копиями баз данных сверх заданного хранилища для кластера.

    * Хранение резервных копий не тарифицируется пока сумма размера БД и всех резервных копий остается меньше выбранного объема хранилища.

    * При автоматическом резервном копировании {{ mms-short-name }} не создает новую копию, а сохраняет изменения БД по сравнению с предыдущей копией. Поэтому потребление хранилища автоматическими резервными копиями растет только пропорционально объему изменений.

    * Количество хостов кластера не влияет на объем хранилища и, соответственно, на бесплатный объем резервных копий.

Цена указывается за 1 месяц использования. Минимальная единица тарификации — 1 ГБ в минуту (например, стоимость хранения 1 ГБ в течение 1,5 минут равна стоимости хранения в течение 2 минут).

## Примеры расчета стоимости кластера {#examples}

### Расчет стоимости кластера без изменения конфигурации {#example-simple}

Например, вы создали кластер:

* с 3 стандартными хостами {{ MS }} класса `s3-c4-m16` (Intel Ice Lake, 4 vCPU, 16 ГБ RAM);
* с любой версией {{ MS }} в редакции Enterprise Edition;
* с 100 ГБ хранилища на сетевых HDD-дисках (`network-hdd`).

Стоимость использования ресурсов:

* 1 расчетный период использования лицензии Windows Server Datacenter за 1 vCPU — 1202,40 ₽;
* 1 расчетный период использования лицензии Microsoft SQL Server Enterprise за 1 vCPU — 23 227,35 ₽;
* 1 час использования ядра хоста {{ MS }} класса Ice Lake с 100% vCPU — 1,0800 ₽;
* 1 час использования 1 ГБ RAM хоста {{ MS }} класса Ice Lake — 0,2880 ₽;
* 1 месяц использования 1 ГБ хранилища на сетевых HDD-дисках (`network-hdd`) — 2,2881 ₽.

В зависимости от используемого [типа вторичных реплик](../concepts/replication.md#readable-and-non-readable-replicas), общая стоимость работы кластера за расчетный период будет вычисляться по-разному. Предполагается, что в расчетном периоде 720 часов (30 дней).

{% list tabs %}

- С нечитаемыми вторичными репликами

    * Лицензии ПО:

        * Стоимость лицензий Windows Server Datacenter — взимается за каждый хост кластера. Итого:

            `3 хоста × (4 vCPU × 1202,40 ₽) = 14 428,80 ₽` за расчетный период.

        * Стоимость лицензий Microsoft SQL Server — взимается только за один хост кластера (первичную реплику). Итого:

            `1 хост × (4 vCPU × 23 227,35 ₽) = 92 909,4 ₽` за расчетный период.

        Общая стоимость лицензий: `14 428,80 ₽ + 92 909,40 ₽ = 107 338,20 ₽`

    * Вычислительные ресурсы хостов:

        Стоимость одного часа работы трех хостов:

        `3 хоста × (4 vCPU × 1,0800 ₽ + 16 ГБ RAM × 0,2880 ₽) = 26,7840 ₽`

        Общая стоимость вычислительных ресурсов: `720 часов × 26,7840 ₽ = 19 284,48 ₽` за расчетный период.

    * Хранилища (без учета резервных копий):

        Общая стоимость использования хранилищ трех хостов: `3 хоста × 100 ГБ × 2,2881 ₽ = 686,43 ₽` за расчетный период.

    Итоговая стоимость кластера: `107 338,20 ₽ + 19 284,48 ₽ + 686,43 ₽ = 127 309,11 ₽`

- С читаемыми вторичными репликами

    * Лицензии ПО:

        * Стоимость лицензий Windows Server Datacenter — взимается за каждый хост кластера. Итого:

            `3 хоста × (4 vCPU × 1202,40 ₽) = 14 428,80 ₽` за расчетный период.

        * Стоимость лицензий Microsoft SQL Server — взимается за каждый хост кластера. Итого:

            `3 хоста × (4 vCPU × 23 227,35 ₽) = 278 728,20 ₽` за расчетный период.

        Общая стоимость лицензий: `14 428,80 ₽ + 278 728,20 ₽ = 293 157,00 ₽`

    * Вычислительные ресурсы хостов:

        Стоимость одного часа работы трех хостов:

        `3 хоста × (4 vCPU × 1,0800 ₽ + 16 ГБ RAM × 0,2880 ₽) = 26,7840 ₽`

        Общая стоимость вычислительных ресурсов: `720 часов × 26,7840 ₽ = 19 284,48 ₽` за расчетный период.

    * Хранилища (без учета резервных копий):

        Общая стоимость использования хранилищ трех хостов: `3 хоста × 100 ГБ × 2,2881 ₽ = 686,43 ₽` за расчетный период.

    Итоговая стоимость кластера: `293 157,00 ₽ + 19 284,48 ₽ + 686,43 ₽ = 313 127,91 ₽`

{% endlist %}

### Расчет стоимости кластера с изменением конфигурации {#example-difficult}

Например, 10 марта (здесь и далее принимаем время внесения изменений за 00:00 UTC+3 для простоты расчета) вы создали кластер:

* с 3 стандартными хостами {{ MS }} класса `s3-c4-m16` (Intel Ice Lake, 4 vCPU, 16 ГБ RAM);
* с любой версией {{ MS }} в редакции Enterprise Edition;
* с 100 ГБ хранилища на сетевых HDD-дисках (`network-hdd`).

15 марта вы решили изменить класс хостов кластера на `s3-c12-m48` (Intel Ice Lake, 12 vCPU, 48 ГБ RAM), 25 марта остановили кластер и 30 марта окончательно его удалили.

Стоимость использования ресурсов:

* 1 расчетный период использования лицензии Windows Server Datacenter за 1 vCPU — 1202,40 ₽;
* 1 расчетный период использования лицензии Microsoft SQL Server Enterprise за 1 vCPU — 23 227,35 ₽;
* 1 час использования ядра хоста {{ MS }} класса Ice Lake с 100% vCPU — 1,0800 ₽;
* 1 час использования 1 ГБ RAM хоста {{ MS }} класса Ice Lake — 0,2880 ₽;
* 1 месяц использования 1 ГБ хранилища на сетевых HDD-дисках (`network-hdd`) — 2,2881 ₽.

В зависимости от используемого [типа вторичных реплик](../concepts/replication.md#readable-and-non-readable-replicas), общая стоимость работы кластера будет вычисляться по-разному:

{% list tabs %}

- C нечитаемыми вторичными репликами

    * Лицензии ПО:

        Стоимость лицензий списывается при создании и масштабировании кластера в сторону увеличения.

        * Стоимость лицензий Windows Server Datacenter — взимается за каждый хост кластера. Итого при создании кластера:

            `3 хоста × (4 vCPU × 1202,40 ₽) = 14 428,80 ₽`

        * Стоимость лицензий Microsoft SQL Server — взимается только за один хост кластера (первичную реплику). Итого при создании кластера:

            `1 хост × (4 vCPU × 23 227,35 ₽) = 92 909,4 ₽`

        10 марта будет списано: `14 428,80 ₽ + 92 909,40 ₽ = 107 338,20 ₽`

        15 марта при добавлении 8 ядер к каждому из трех хостов кластера будет списано:

        `3 хоста × (8 vCPU × 1202,40 ₽) + 1 хост × (8 vCPU × 23 227,35 ₽) = 214 676,40 ₽`

        Дополнительные лицензии потребуются только для тех ядер, которые не покрываются ранее активированными в этом месяце лицензиями.

        Общая стоимость лицензий: `107 338,20 ₽ + 214 676,40 ₽ = 322 014,60 ₽`

    * Вычислительные ресурсы хостов:

        * Стоимость одного часа работы трех хостов класса `s3-c4-m16` (Intel Ice Lake, 4 vCPU, 16 ГБ RAM):

            `3 хоста × (4 vCPU × 1,0800 ₽ + 16 ГБ RAM × 0,2880 ₽) = 26,7840 ₽`

        За период с 10 по 14 марта будет списано: `5 дней × 24 часа × 26,7840 ₽ = 3214,08 ₽`

        * Стоимость одного часа работы трех хостов класса `s3-c12-m48` (Intel Ice Lake, 12 vCPU, 48 ГБ RAM):

            `3 хоста × (12 vCPU × 1,0800 ₽ + 48 ГБ RAM × 0,2880 ₽) = 80,3520 ₽`

        За период с 15 по 24 марта будет списано: `10 дней × 24 часа × 80,3520 ₽ = 19 284,48 ₽`

        После остановки кластера 25 марта списаний за вычислительные ресурсы не будет.

        Общая стоимость вычислительных ресурсов: `3214,08 ₽ + 19 284,48 ₽ = 22 498,56 ₽`

    * Хранилища (без учета резервных копий):

        Стоимость использования хранилища рассчитывается за время жизни кластера, то есть с 10 по 30 марта:

        `3 хоста × 100 ГБ × (20 дней / 30 дней) × 2,2881 ₽ = 457,62 ₽`

    Итоговая стоимость кластера: `322 014,60 ₽ + 22 498,56 ₽ + 457,62 ₽ = 344 970,78 ₽`

- С читаемыми вторичными репликами

    * Лицензии ПО:

        Стоимость лицензий списывается при создании и масштабировании кластера.

        15 марта при добавлении 8 ядер к каждому из трех хостов кластера будет списано:

        `3 хоста × (8 vCPU × 1202,40 ₽) + 3 хоста × (8 vCPU × 23 227,35 ₽) = 586 314,00 ₽`

        Дополнительные лицензии потребуются только для тех ядер, которые не покрываются ранее активированными в этом месяце лицензиями.

        Общая стоимость лицензий: `293 157,00 ₽ + 586 314,00 ₽ = 879 471,00 ₽`

    * Вычислительные ресурсы хостов:

        * Стоимость одного часа работы трех хостов класса `s3-c4-m16` (Intel Ice Lake, 4 vCPU, 16 ГБ RAM):

            `3 хоста × (4 vCPU × 1,0800 ₽ + 16 ГБ RAM × 0,2880 ₽) = 26,7840 ₽`

        За период с 10 по 14 марта будет списано: `5 дней × 24 часа × 26,7840 ₽ = 3 214,08 ₽`
