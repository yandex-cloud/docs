# Классы хостов

Класс хостов определяет вычислительные мощности, которые выделяются для каждого хоста в кластере. При изменении класса хостов для кластера меняются характеристики всех уже созданных в нем хостов.

{% if audience != "internal" %} Доступный объем хранилища не зависит от класса хостов. Ограничения хранилища приведены в разделе [{#T}](limits.md). {% endif %}

Хосты в кластерах {{ mms-name }} разворачиваются на базе виртуальных машин {{ compute-full-name }}. Эти виртуальные машины могут быть созданы на любой из платформ, которые поддерживает {{ compute-name }}. Подробное описание платформ приведено в разделе [{#T}](../../compute/concepts/vm-platforms.md).

Ниже приведен полный список возможных конфигураций хостов на каждой из платформ:

* **s2** — стандартные конфигурации, с соотношением количества vCPU к количеству гигабайт RAM 1:4.
* **m2** — конфигурации с уменьшенным соотношением количества vCPU к количеству гигабайт RAM (1:8 в отличие от 1:4 для стандартных классов). Такие конфигурации могут быть полезны для кластеров с повышенными требованиями к кэшу.

  Хосты с гарантированной долей vCPU ниже 100% не могут использоваться в многохостовых кластерах.
