# Сборка и управление зависимостями функции на .NET Core

Сервис {{ sf-name }} поддерживает два способа управления зависимостями функции на .NET Core — автоматическую установку через `nuget` и ручную поставку зависимостей вместе с исходным кодом.

## NuGet

[NuGet](https://docs.microsoft.com/ru-ru/nuget/what-is-nuget) — система управления пакетами для платформы `.net`.

Для поставки зависимостей при помощи `nuget` следует добавить ссылку на необходимый пакет в файл проекта `*.csproj`. Никаких дополнительных действий не требуется. Это можно сделать с помощью [.NET Core CLI](https://docs.microsoft.com/ru-ru/dotnet/core/tools/dotnet-add-package).

## Ручная поставка зависимостей

Чтобы настроить зависимости вручную, положите скомпилированный проект со всеми зависимостями в корень архива с проектом.
Это можно сделать при помощи команды `publish` [.NET Core CLI](https://docs.microsoft.com/ru-ru/dotnet/core/tools/dotnet-publish)
В этом случае не нужно добавлять исходный код и файлы проекта в архив.  

Процесс установки зависимостей имеет ограничения по ресурсам и времени исполнения. Подробнее об этом читайте в разделе [{#T}](../../concepts/limits.md). Ознакомиться с журналом установки зависимостей можно по ссылке, которая отображается в списке операций.
