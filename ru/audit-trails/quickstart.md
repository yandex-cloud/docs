# Как начать работать с {{ at-name }}

Чтобы включить сбор аудитных логов, обратитесь в [службу технической поддержки]({{ link-console-support }}). В обращении укажите:

1. Идентификатор каталога, в котором нужно разместить [трейл](concepts/index.md#trail).
  
    Ресурс не будет виден на [стадии Preview](../overview/concepts/launch-stages.md).
1. Имя бакета, в который будут загружаться аудитные логи.
1. Путь к папке в бакете, в которую будут загружаться файлы с аудитными логами.
1. Если требуется включить аудитные логи только по некоторым сервисам, укажите эти сервисы.

{% cut "Пример обращения" %}

```text
1. Идентификатор каталога b1g8jm1ska9r0ngh8ryk.
2. Имя бакета backet-for-audit-logs.
3. Путь к папке в бакете folder/for-logs.
4. Логи по сервисам {{ compute-name }}, {{ iam-name }}.
```

{% endcut %}

По вашему обращению будет создан трейл и начнется загрузка аудитных логов в бакет.
