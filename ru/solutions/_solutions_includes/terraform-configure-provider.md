1. В начале конфигурационного файла необходимо задать настройки провайдера.

   ```
   provider "yandex" {
     token     = "<OAuth или статический ключ сервисного аккаунта>"
     folder_id = "<идентификатор каталога>"
     zone      = "ru-central1-a"
   }
   ```

   * `provider` — название провайдера.
   * `token` — [OAuth-токен](../../iam/concepts/authorization/oauth-token.md) для доступа к Яндекс.Облаку.
   * `folder_id` — [идентификатор каталога](../../resource-manager/operations/folder/get-id), в котором Terraform создаст облачные ресурсы.
   * `zone` — зона доступности, в которой по умолчанию будут создаваться все облачные ресурсы.

1.  Выполните команду `terraform init` в папке с конфигурационным файлом. Эта команда инициализирует провайдеров, указанных в конфигурационных файлах и позволяет работать с ресурсами и источниками данных провайдера.
