# Политика поддержки версий {{ MY }}

Данный документ описывает жизненный цикл мажорных версий {{ MY }} в сервисе {{ mmy-full-name }}. Политика основывается на [официальном графике поддержки Oracle {{ MY }}](https://www.mysql.com/support/eol-notice.html) и графике поддержки [Extended Support от Percona](https://www.percona.com/release-lifecycle-overview/#lifecycle) и предназначена для обеспечения баланса между стабильностью работы [кластеров](./index.md) и поддержанием современной и безопасной инфраструктуры.

## Терминология и этапы жизненного цикла версий {{ MY }} {#version-lifecycle}

Для каждой мажорной версии {{ MY }} в сервисе {{ mmy-name }} действует жизненный цикл продолжительностью 8–10 лет, включающий следующие этапы (состояния):

| Состояние версии | Описание и ключевые [действия](#actions-available) | Примерные сроки^1^                     |
|:------|:-------------|:----------------------------|
| Актуальная (`Actual`) | Самая последняя [LTS](https://ru.wikipedia.org/wiki/Долгосрочная_поддержка_программного_обеспечения)-версия с активной поддержкой. **Рекомендуется для всех новых проектов**. | с первого по пятый год |
| Поддерживаемая (`Supported`)  | Предыдущая LTS-версия с поддержкой `Extended Support` от Percona. Полностью поддерживается, создание новых кластеров разрешено. | с шестого по восьмой год |
| Устаревающая (`Deprecated`) | Версия с завершающейся поддержкой `Extended Support` от Percona. Создание новых кластеров заблокировано. Существующие кластеры работают в штатном режиме. | девятый год |
| Устаревшая с повышенной стоимостью (`Legacy Extra Paid`)^2^ | Версия, поддержка которой официально прекращена. Кластеры работают, но подлежат тарификации по повышенной ставке. Техническая поддержка ограничена. | десятый год |
| Выведенная из эксплуатации (`EOL`)^2^ | Версия более не поддерживается. Техническая поддержка не предоставляется. Кластеры работают, но подлежат тарификации по повышенной ставке. | одиннадцатый год и далее |

^1^ Сроки указаны относительно даты релиза мажорной версии. Точные даты перехода между состояниями публикуются в официальных анонсах сервиса.
^2^ Яндекс предварительно [уведомит](#notifications) об изменениях и начале тарификации версий по повышенной ставке в соответствии с условиями Договора.

## Ключевые принципы политики поддержки версий {{ MY }} {#key-principles}

Политика поддержки версий {{ MY }} основывается на следующих ключевых принципах:

* **Длительность поддержки**. Каждая LTS-версия находится в состоянии актуальной (`Actual`) или поддерживаемой (`Supported`) в течение восьми лет благодаря поддержке [Extended Support от Percona](https://www.percona.com/release-lifecycle-overview/#lifecycle).
* **Поэтапный вывод**. На девятый год версия переходит в состояние устаревающей (`Deprecated`), и создание новых кластеров становится невозможным.
* **Повышенная стоимость**^2^. На десятый год версия переходит в состояние устаревшей с повышенной стоимостью (`Legacy Extra Paid`). Кластеры подлежат повышенной тарификации, техническая поддержка ограничивается.
* **Завершение жизненного цикла**^2^. По достижении даты `EOL` (одиннадцатый год и далее) техническая поддержка не предоставляется. Восстановление из резервных копий невозможно. Кластеры работают, но подлежат повышенной тарификации.

^2^ Яндекс предварительно [уведомит](#notifications) об изменениях и начале тарификации версий по повышенной ставке в соответствии с условиями Договора.

## Политика обновлений {#update-policy}

* **Минорные обновления** (в пределах мажорной версии, например, 8.0.34 → 8.0.35) устанавливаются **автоматически** в рамках технического обслуживания, выполняемого в заданное для кластера [окно обслуживания](./maintenance.md#maintenance-window) либо по требованию клиента. Эти обновления содержат исправления безопасности и ошибок. Обновление требует кратковременной поочередной перезагрузки хостов кластера.
* **Мажорные обновления** (смена основной версии, например, 8.0.x → 8.1.x) происходят в момент перехода версии из состояния устаревающей (`Deprecated`) в состояние устаревшей (`Legacy`). Запланированная операция обновления появляется в интерфейсе {{ yandex-cloud }} заранее и может быть отменена через [обращение]({{ link-console-support }}) в службу технической поддержки. Версии в состоянии `EOL` невозможно обновить до новой мажорной версии. **Настоятельно рекомендуется выполнить обновление до наступления даты EOL**.

{% note info "Особенности Percona Server for {{ MY }}" %}

Благодаря поддержке `Extended Support` от Percona, версии {{ MY }} получают обновления безопасности и исправления ошибок дольше, чем при стандартном графике поддержки Oracle {{ MY }}.

{% endnote %}

## Уведомления {#notifications}

Сервис {{ mmy-name }} заблаговременно уведомит вас о предстоящих изменениях:

* При переходе версии {{ MY }} в состояние **устаревающей (`Deprecated`)** (начало действия ограничений) сервис за 90 дней уведомит о невозможности создания новых кластеров с текущей версией {{ MY }} и порекомендует запланировать обновление существующих кластеров.
* При переходе версии {{ MY }} в состояние **устаревшей (`Legacy`)** (начало действия повышенного тарифа) сервис за 90 дней уведомит о начале действия повышенной ставки тарификации и предупредит о приближающемся прекращении поддержки.

Уведомления о таких изменениях отправляются пользователям в соответствии с заданными [настройками уведомлений](../../console/operations/update.md#notifications).
