# Подключение виртуальных машин {{ compute-name }} к {{ backup-name }}

Если вы хотите создавать резервные копии [виртуальной машины {{ compute-full-name }}](../../compute/concepts/vm.md) в сервисе {{ backup-full-name }}, ее нужно подключить к сервису и корректно настроить.

Подключить виртуальную машину к {{ backup-name }} можно при ее [создании](../../compute/operations/vm-create/create-linux-vm.md), выбрав соответствующую опцию. Опция появляется, только если на ВМ устанавливается [поддерживаемая операционная система](#os). Подробнее о подключении см. в [инструкции](../operations/create-vm.md).

Чтобы подключение работало корректно, при создании ВМ к ней нужно привязать [сервисный аккаунт](#sa) с ролью `backup.editor`, [публичный IP-адрес](#public-ip-address) и [группу безопасности](#security-groups) с определенными правилами (если эта функциональность включена в вашем облаке).

После подключения к {{ backup-name }} и настройки виртуальную машину нужно привязать к [политике резервного копирования](policy.md).


## Поддерживаемые операционные системы {#os}

{{ backup-name }} поддерживает резервное копирование виртуальных машин с операционной системой Ubuntu версий 20.04 и ниже.

ОС должна быть установлена из публичного образа (продукта {{ marketplace-name }}). При создании ВМ можно выбрать ОС напрямую либо использовать [образ](../../compute/concepts/image.md) или [снимок диска](../../compute/concepts/snapshot.md) с другой ВМ, если ОС на нее тоже устанавливалась из публичного образа.

Позже в {{ backup-name }} будут поддержаны Ubuntu 22.04 и CentOS 7.

## Сервисный аккаунт {#sa}

[Сервисный аккаунт](../../iam/concepts/users/service-accounts.md) — специальный аккаунт, от имени которого копии ВМ создаются и загружаются в хранилище {{ backup-name }}.

Когда вы создаете ВМ, для которой хотите настроить резервное копирование в {{ backup-name }}, к ВМ нужно привязать сервисный аккаунт с [ролью `backup.editor`](../security/index.md).

Вы можете [назначить роль](../../iam/operations/sa/assign-role-for-sa.md) существующему сервисному аккаунту или [создать](../../iam/operations/sa/create.md) сервисный аккаунт с нужными ролями.


## Публичный IP-адрес {#public-ip-address}

ВМ, подключенная к {{ backup-name }}, должна обмениваться данными с серверами [провайдера резервного копирования](index.md#providers). Для этого при создании ВМ к ней нужно привязать [публичный IP-адрес](../../vpc/concepts/address.md#public-addresses) {{ vpc-full-name }}.

Использование публичного IP-адреса тарифицируется. Подробнее см. в разделе [{#T}](../../vpc/pricing.md).


## Группы безопасности {#security-groups} 

{% include [security-groups-note](../../application-load-balancer/_includes_service/security-groups-note.md) %}

Группы безопасности позволяют управлять доступом ВМ к ресурсам {{ yandex-cloud }} или ресурсам в интернете.

Чтобы ВМ могла обмениваться данными с серверами провайдера резервного копирования, при ее создании и подключении к {{ backup-name }} нужно выбрать группу безопасности, которая разрешает исходящий TCP-трафик на диапазон адресов `193.32.199.0/24` на порты `443`, `7770-7800`, `43234` и `45284`. Для этого в группе можно создать следующие (или более <q>широкие</q>) правила:

{% list tabs %}

- Исходящий трафик

  | Диапазон портов | Протокол | Тип назначения | Назначение |
  | --- | --- | --- | --- |
  | 443 | TCP | CIDR | 193.32.199.0/24 |
  | 7770-7800 | TCP | CIDR | 193.32.199.0/24 |
  | 43234 | TCP | CIDR | 193.32.199.0/24 |
  | 45284 | TCP | CIDR | 193.32.199.0/24 |

{% endlist %}

Вы можете [добавить правила](../../vpc/operations/security-group-add-rule.md) в существующую группу безопасности или [создать](../../vpc/operations/security-group-create.md) группу с нужными правилами.
