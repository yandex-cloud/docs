# Создать виртуальную машину с подключением к {{ backup-name }}

{% note info %}

Подключить ВМ к {{ backup-name }} можно только при создании. Чтобы подключить существующую ВМ к {{ backup-name }}, [сделайте](../../compute/operations/disk-control/create-snapshot.md) снимки дисков этой ВМ и [создайте](../../compute/operations/vm-create/create-from-snapshots.md) новую из этих снимков, выбрав опцию резервного копирования. Поддерживаются операционные системы Ubuntu (версии 20.04 и ниже) и CentOS.

{% endnote %}

{% list tabs %}

- Консоль управления

  1. В [консоли управления]({{ link-console-main }}) выберите каталог, в котором нужно создать ВМ.
  1. В списке сервисов выберите **{{ compute-name }}**.
  1. Нажмите кнопку **Создать ВМ**.
  1. В блоке **Выбор образа/загрузочного диска** выберите операционную систему Ubuntu (версии 20.04 и ниже) или CentOS.
  1. Укажите необходимые параметры ВМ. Подробнее см. [Создать виртуальную машину из публичного образа Linux](../../compute/operations/vm-create/create-linux-vm).
  1. В блоке **Резервное копирование** выберите опцию подключения ВМ к сервису {{ backup-name }}.
  1. В блоке **Доступ** выберите сервисный аккаунт с [ролью](../security/index.md#backup-editor) `backup.editor`. Если у вас нет сервисного аккаунта:
      1. Нажмите кнопку **Создать новый**.
      1. В открывшемся окне введите имя сервисного аккаунта.
      1. В поле **Роли в каталоге** нажмите кнопку ![plus](../../_assets/plus-sign.svg) и выберите роль `backup.editor`.
      1. Нажмите кнопку **Создать**.
  1. Нажмите кнопку **Создать ВМ**. 

{% endlist %}

После перехода ВМ в статус `Running` начнется установка агента {{ backup-name }}. Этот процесс может занять несколько минут. После этого ВМ станет доступна для добавления в [политики резервного копирования](../concepts/policy.md).
