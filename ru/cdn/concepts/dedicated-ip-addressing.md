# Выделенная IP-адресация CDN-ресурсов

По умолчанию один и тот же публичный IP-адрес может одновременно обслуживать трафик разных клиентов.

По запросу вы можете [получить](../operations/resources/dedicated-ip-addressing.md) выделенный публичный IP-адрес, который будет использоваться исключительно для ваших [CDN-ресурсов](resource.md) в {{ cdn-full-name }}. При этом контент будет доставляться через все [точки присутствия](points-of-presence.md).

Выделенный IP-адрес привязывается к каталогу. Поэтому в рамках одного каталога все CDN-ресурсы будут подключены к услуге выделенной IP-адресации и будут иметь один IP-адрес.

Для [балансировки CDN-ресурсов](https://yandex.cloud/ru/blog/posts/2025/04/load-balancers-guide) с выделенной IP-адресацией используются методы [BGP Anycast](https://{{ lang }}.wikipedia.org/wiki/Anycast) и GSLB, основанные на технологиях DNS. Эти методы отличаются от методов балансировки, применяемых к ресурсам без выделенной IP-адресации. BGP Anycast подразумевает анонс IP-адреса из всех локаций точек присутствия CDN. Поэтому, несмотря на выдачу одного IP-адреса, сервис продолжает эффективно обслуживать всю сеть точек присутствия.

Выделенная IP-адресация CDN-ресурсов может потребоваться:

* для включения ресурсов в [белый список Минцифры](../../overview/concepts/info-for-federal-ip-whitelist.md);
* для повышения безопасности;
* для изоляции ресурсов клиента от ресурсов других пользователей CDN, в том числе в случае необходимости доступа к ресурсам через настройки [групп безопасности](../../vpc/concepts/security-groups.md).
