# Следование перенаправлениям запросов

По умолчанию, когда [источник](origins.md) возвращает код перенаправления `3xx`, {{ cdn-name }} кеширует этот ответ и передает его клиенту. С помощью опции следования перенаправлениям вы можете разрешить [CDN-ресурсу](resource.md) следовать перенаправлениям от источника. При включенной опции {{ cdn-name }} выполняет запрос по адресу, указанному в ответе, и затем возвращает клиенту контент с конечного адреса.

Опция использует только один уровень перенаправления — если после первого перенаправления снова получен код `3xx`, он будет передан клиенту и закеширован на CDN.

{% note info %}

Опция работает только при включенной функции [экранирования источников](origins-shielding.md).

{% endnote %}

Следование перенаправлениям может потребоваться, например, в следующих сценариях:

* Миграция контента — при переносе файлов на другой домен или сервер.
* Балансировка нагрузки — при распределении запросов между несколькими источниками.
* А/В-тестирование — при переключении трафика между версиями контента.
