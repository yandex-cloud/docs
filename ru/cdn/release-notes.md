# История изменений в {{ cdn-full-name }}

## I квартал 2026 {#q1-2026}

* Добавлена возможность в [локационных правилах](./concepts/location-rules.md) задать другой [источник](./concepts/origins.md) для [CDN-ресурса](./concepts/resource.md) без наследования настроек из основной конфигурации.
* Реализованы [профили безопасности TLS](./concepts/clients-to-servers-tls.md#tls-profiles) для CDN-ресурса, содержащие определенный набор разрешенных шифров TLS.
* Поддержана [настройка пользовательского имени сервера для SNI](./operations/resources/configure-server-name.md).
* Внедрена возможность настроить в [консоли управления]({{ link-console-main }}) для CDN-ресурса разрешенный HTTP-метод `POST` в запросах от клиентов, а также поддержку протокола [WebSocket](https://{{ lang }}.wikipedia.org/wiki/WebSocket). Опции становятся доступными после их включения через [техническую поддержку]({{ link-console-support }}). Подробнее читайте в разделах [{#T}](./troubleshooting.md#post-responses) и [{#T}](./troubleshooting.md#websocket-support).
* Добавлена опция [настройки HTTP-ответов](./operations/resources/configure-response.md), с помощью которой можно задать код перенаправления с CDN-ресурса или статический код ответа с заранее заданным телом ответа.
* Реализовано отображение в консоли управления статуса блокировки CDN-ресурса. Статус отображается, если CDN-ресурс был заблокирован за нарушение правил пользования сервисом.
* Поддержана возможность задать любое время [кеширования контента](./concepts/caching.md).
* Внедрена настройка [следования перенаправлениям запросов](./concepts/follow-redirects.md) в консоли управления.

## IV квартал 2025 {#q4-2025}

* Реализована функциональность [локационных правил](./concepts/location-rules.md) — точечного переопределения настроек CDN-ресурса для запросов, пути которых соответствуют заданному шаблону.
* Добавлена возможность разрешить CDN-ресурсу [следовать перенаправлениям запросов](./concepts/follow-redirects.md) от источника. При включенной опции {{ cdn-name }} выполняет запрос по адресу, указанному в ответе источника, и затем возвращает клиенту контент с конечного адреса.
* Поддержан протокол [WebSocket](https://ru.wikipedia.org/wiki/WebSocket). Для включения обратитесь в [техническую поддержку]({{ link-console-support }}).
* Добавлены новые [точки присутствия](./concepts/points-of-presence.md).
* Поддержан [калькулятор цен для {{ cdn-name }}](https://yandex.cloud/ru/prices?state=cc0daabb430b#calculator).
* Добавлено [событие {{ at-name }}](./at-ref.md) уровня конфигурации `ResourceDelete` — удаление CDN-ресурса.

## III квартал 2025 {#q3-2025}

* Добавлен провайдер {{ cdn-full-name }} на основе собственных технологий Яндекса.

* Добавлена возможность выбора провайдера при создании [ресурса](./operations/resources/create-resource.md) и [группы источников](./operations/origin-groups/create-group.md).

* Обновлен интерфейс создания CDN-ресурса:

  * упрощена настройка и создание ресурса с помощью визарда;
  * добавлена возможность копирования конфигурации уже существующего CDN-ресурса.

* Добавлена функциональность управления правилами использования ресурсов для провайдера {{ cdn-full-name }}. Настройка доступна в [API](./api-ref/ResourceRules/index.md).

* Исправлены ошибки в {{ TF }}:

  * некорректное отображение доменного имени ресурса `provider_cname`;
  * невозможность выбора провайдера для ресурса.

## IV квартал 2024 {#q4-2024}

Поддержано [назначение](./operations/resources/labeling.md) CDN-ресурсам [меток](./concepts/labels.md) {{ resmgr-full-name }}.

## III квартал 2024 {#q3-2024}

* Добавлена функциональность [перенаправления запросов](./concepts/http-rewrite.md). Настройка доступна в консоли управления, CLI и API.
* Поддержана возможность задавать [политику доступа по IP-адресам](./concepts/ip-address-acl.md) в консоли управления.

## II квартал 2024 {#q2-2024}

Добавлена возможность [настраивать доступ по защищенному токену](./operations/resources/enable-secure-token.md) в консоли управления.

## I квартал 2024 {#q1-2024}

* Добавлена поддержка [политик доступа по IP-адресам](./concepts/ip-address-acl.md) для CDN-ресурсов.
* Реализована механика ограничения доступа к CDN-ресурсу с помощью [защищенных токенов](./concepts/secure-tokens.md).
* Убрана поддержка автоматического выпуска TLS-сертификатов для CDN-ресурсов на стороне CDN-провайдера EdgeЦентр. Теперь TLS-сертификаты должны быть предварительно загружены в {{ certificate-manager-full-name }}. Подробнее см. [{#T}](./concepts/clients-to-servers-tls.md).
