# Диагностика ошибок в работе модуля {{ vuln-man-name }} ({{ vuln-man-short-name }})

## Access denied to resource `Id` {#account-permissions}

У сервисного аккаунта недостаточно прав на один или несколько ресурсов в области сканирования. Убедитесь, что сервисный аккаунт имеет необходимые [роли](../security/index.md) для сканирования заданных ресурсов. После этого перезапустите сканирование.

## Resource `Id` was not found in scope {#resource-not-found}

Ресурс, указанный в области сканирования, был удален или перемещен. Проверьте актуальность конфигурации области сканирования в [настройках окружения](../operations/workspaces/update.md) в разделе **{{ ui-key.yacloud_org.security.workspaces.WorkspaceEditPageLayout.tabResources_vJuSN }}**. Удалите отсутствующий ресурс из списка целей сканирования или восстановите его, а затем перезапустите сканирование.

## OS is not supported {#unsupported-os}

Образ использует операционную систему, для которой у модуля {{ vuln-man-short-name }} нет актуальных баз уязвимостей. Сканирование этого образа пропускается. Рассмотрите возможность обновления образа до поддерживаемой версии ОС. Если этот тип ОС для вас критичен, обратитесь в [службу поддержки]({{ link-console-support }}).

## An unexpected technical error occurred while scanning image {#internal-error}

Возникла непредвиденная проблема на стороне модуля {{ vuln-man-name }}. Если ошибка повторяется, обратитесь в [службу поддержки]({{ link-console-support }}). В сообщении укажите код ошибки и идентификатор запуска сканирования.
