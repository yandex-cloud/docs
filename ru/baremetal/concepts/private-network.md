# Приватная сеть {#private-network}

_Приватная сеть_ — это локальная сеть к которой подключены все серверы одного [пула](./servers.md#server-pools) серверов. Логически объединяет серверы в изолированные пользовательские сети. Некоторые [конфигурации](./server-configurations.md) серверов {{ baremetal-name }} поддерживают резервирование каналов приватной сети [MC-LAG](./mc-lag.md).

В рамках одного пула между арендованными серверами возможна сетевая связность на уровнях [сетевой модели OSI](https://ru.wikipedia.org/wiki/Сетевая_модель_OSI) L2 ([приватная подсеть](#private-subnet)) и L3 ([VRF](#vrf-segment)).

Между серверами, физически размещенными в разных пулах, доступна только L3-связность с помощью VRF.

## Приватная подсеть {#private-subnet}

_Приватная подсеть_ — это [виртуальная сеть VLAN](https://ru.wikipedia.org/wiki/VLAN), объединяющая серверы в одном пуле, физически изолированная от интернета и логически — от виртуальных сетей других пользователей.

Приватной подсети автоматически присваивается уникальный идентификатор VLAN в соответствии со стандартом [IEEE 802.1Q](https://ru.wikipedia.org/wiki/IEEE_802.1Q). При этом идентификатор VLAN присваивается подсети не при создании, а при ее привязке к любому серверу {{ baremetal-name }}.

{% note info %}

Идентификатор VLAN удаляется из приватной подсети, если она на протяжении 30 дней не была привязана ни к одному серверу. В дальнейшем при привязке такой подсети к серверу {{ baremetal-name }} ей снова будет присвоен идентификатор VLAN, но новый идентификатор может отличаться от того, который был удален ранее.

{% endnote %}

К [сетевому интерфейсу](./servers.md#network-interfaces) (или группе агрегирования MC-LAG) сервера, которые подключены к приватной сети, можно [привязать](../operations/servers/set-up-tagged-vlan.md) одновременно несколько приватных подсетей: одну [основную](#native-vlan-subnet) и несколько [дополнительных](#tagged-vlan-subnet). При этом одна и та же приватная подсеть может одновременно являться основной на одних серверах и дополнительной — на других.

### Основная приватная подсеть {#native-vlan-subnet}

К любому серверу {{ baremetal-name }} должна быть привязана одна приватная подсеть, которая будет считаться для него _основной_. Основная приватная подсеть сервера представляет собой виртуальную сеть с нативным (`native`) VLAN. В нее направляется весь нетегированный трафик, поступающий на сетевой интерфейс (в группу агрегирования) сервера.

В основной приватной подсети сервера может использоваться сервис [DHCP](./dhcp.md).

Заменить основную приватную подсеть можно только на серверах, к которым не привязаны дополнительные приватные подсети.

{% note tip %}

Если требуется заменить основную приватную подсеть на сервере с дополнительными подсетями:
1. Отвяжите от сервера все дополнительные приватные подсети.
1. Замените основную приватную подсеть.
1. Повторно привяжите к серверу дополнительные приватные подсети.
1. Обновите сетевые настройки в операционной системе сервера.

{% endnote %}

### Дополнительная приватная подсеть {#tagged-vlan-subnet}

К серверу {{ baremetal-name }} можно привязать _дополнительные приватные подсети_. Дополнительная приватная подсеть сервера представляет собой виртуальную сеть с тегированным (`tagged`) VLAN. В дополнительную подсеть направляется трафик, поступающий на сетевой интерфейс (группу агрегирования) сервера и отмеченный тегом, который соответствует уникальному для этой подсети идентификатору VLAN (тегированный трафик).

По умолчанию к сетевому интерфейсу сервера {{ baremetal-name }} можно привязать только одну приватную подсеть — основную. Чтобы иметь возможность привязывать к серверу дополнительные приватные подсети, обратитесь в [службу поддержки]({{ link-console-support }}) с запросом на увеличение [квоты](./limits.md#baremetal-quotas) `baremetal.privateSubnetsPerServerInterface.count`.

Привязать дополнительную приватную подсеть к серверу можно при его изменении. Подробнее читайте в разделе [{#T}](../operations/servers/server-update.md).

После привязки к серверу дополнительной приватной подсети необходимо вручную настроить VLAN на сетевом интерфейсе в операционной системе сервера. Подробнее читайте в разделе [{#T}](../operations/servers/set-up-tagged-vlan.md).

## Виртуальный сегмент сети (VRF) {#vrf-segment}

Для обеспечения L3-маршрутизации приватные подсети, в которых настроена маршрутизация, объединяются в виртуальные сегменты сети (VRF).

Серверы из одного или разных пулов, подключенные к разным приватным подсетям, объединенным в VRF, смогут поддерживать между собой сетевой обмен по L3. Чтобы настроить такой сетевой обмен, для соответствующих подсетей в блоке **{{ ui-key.yacloud.baremetal.title_routing-settings }}** необходимо выбрать одинаковый VRF.
