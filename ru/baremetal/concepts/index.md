# Обзор сервиса {{ baremetal-full-name }}

{{ baremetal-full-name }} предоставляет возможность [арендовать](../operations/servers/server-lease.md) физический сервер, все аппаратные ресурсы которого полностью предоставляются в ваше индивидуальное использование. 

Для создания и удобной эксплуатации инфраструктуры с использованием арендованных физических серверов в сервисе {{ baremetal-name }} пользователю предоставляются все необходимые инструменты для управления серверами, а также настройками сети, к которой они подключены.

Механизм гибких [квот](limits.md) и лимитов на ресурсы определяет количество доступных серверов, сетей и место для образов собственных ОС для каждого пользователя.

## Серверы {#servers}

Для заказа доступны серверы следующих конфигураций:

* Готовая конфигурация — получите готовый к работе сервер с ОС или без нее в течение нескольких минут. Выбирайте сервер среди нескольких десятков конфигураций под типовые задачи и нагрузки.

   [Арендовать сервер с готовой конфигурацией](server-configurations.md)
* Своя конфигурация — создайте сервер под ваш профиль нагрузки с помощью конструктора из северных компонентов в наличии, и через 4 дня сервер будет в вашем распоряжении. 

  [Собрать свою конфигурацию сервера](server-custom-configurations.md)
* Конфигурация по запросу — если вам требуется сервер для разнообразных задач или нагрузок, запросите предложение индивидуальной конфигурации сервера. 
  
  [Опишите конфигурацию по запросу для индивидуального предложения](https://yandex.cloud/ru/services/baremetal#contact-form)

[Серверы](servers.md) можно арендовать на срок от одного дня до года. При заказе сервера можно установить операционную систему из готовых [образов](images.md) Marketplace или для установки использовать собственный дистрибутив, [загрузив](../operations/image-upload.md) его в [{{ objstorage-full-name }}](../../storage/), настроить публичную и приватную IP-адресацию, SSH-доступ, политики резервного копирования сервера через сервис [{{ backup-full-name }}](../../backup/).

## Сеть {#network}

Все серверы имеют несколько физических сетевых интерфейсов, которые подключены к [публичной сети](./public-network.md) интернет и к [приватной сети](./private-network.md) через разные, независимые коммутаторы, собранные в единую IP-фабрику с топологией spine-leaf.

В публичной сети на каждый сервер бесплатно выделяется один публичный IPv4-адрес. Также можно арендовать блоки [выделенных публичных подсетей](./public-network.md#public-subnet) разного размера и использовать их для назначения адресов серверам. Для серверов с сетевыми картами 10 и 25 Гбит/с можно выбрать суточный пакет для потребления интернет-трафика.

В приватной сети можно объединять серверы на разных уровнях через [приватные подсети](./private-network.md#private-subnet) (L2, VLAN 802.1Q) и, используя маршрутизацию, собрать их в [VRF](./private-network.md#vrf-segment) (L3). При необходимости можно назначать на один порт сервера несколько приватных подсетей (L2, trunk-port 802.1Q) или добавить статические маршруты в VRF.

Для обеспечения стабильной работы всех сетей в сервисе {{ baremetal-name }} существует ряд [ограничений](network-restrictions.md).

Серверы {{ baremetal-name }} могут связываться по приватной сети с облачной инфраструктурой или с другими сервисами {{ yandex-cloud }} с помощью сервиса [{{ interconnect-name }}](../../interconnect/), который позволяет объединить в единую L3-сеть физические серверы и облачную инфраструктуру. Так можно реализовать разные сценарии построения инфраструктуры, например, использование физических серверов в качестве узлов [{{ managed-k8s-name }}](../tutorials/k8s-connect-bms-as-node.md) или под выделенное хранение данных.

## Настройка и управление {#control}

Доступ к серверу может осуществляться с помощью [KVM-консоли](../operations/servers/server-kvm.md) или через [SSH](../../glossary/ssh-keygen.md). Управлять серверами так же можно с помощью [CLI](../cli-ref/) и [API](../api-ref/).

Для настройки, управления и аудита действий с объектами сервис предоставляет следующие возможности: управление с помощью консоли управления {{ yandex-cloud }}, [CLI](../cli-ref/) и [API](../api-ref). Аудитные логи операций можно получать с помощью сервиса [{{ at-name }}](../../audit-trails/), а управлять ролями и доступами — через [{{ iam-name }}](../../iam/).

## Квоты и лимиты {#quotas-and-limits}

Количество доступных серверов, сетей и образов ограничено рядом [квот](limits.md). Помимо этого, присутствует ряд [ограничений](network-restrictions.md) в работе сетей.
