# Рекомендуемые конфигурации мастера

[Конфигурацию мастера](./index.md#master) можно выбрать при [создании](../operations/kubernetes-cluster/kubernetes-cluster-create.md) или [изменении](../operations/kubernetes-cluster/kubernetes-cluster-update.md#manage-resources) кластера. Для стабильной работы конфигурация мастера должна соответствовать нагрузке на кластер.

С помощью таблицы можно подобрать конфигурацию мастера по параметрам кластера. Приведенные в таблице значения ориентировочные — выбирайте строку, к параметрам которой больше всего приближен ваш кластер.

На выбор конфигурации мастера влияют профиль нагрузки, используемые в кластере ресурсы, работа операторов и другие факторы. В зависимости от этих параметров может потребоваться более мощная конфигурация.

{% note tip %}

В таблице приведены конфигурации мастера типа Standard с соотношением объема RAM к vCPU, равным 4 к 1. Если такая конфигурация не подходит для вашего профиля нагрузки, выберите ближайшую по ресурсам конфигурацию CPU Optimized или Memory Optimized из [списка доступных конфигураций](../operations/kubernetes-cluster/kubernetes-cluster-configuration-list.md).

{% endnote %}

Выбранная конфигурация задает минимальные значения ресурсов для мастера. При повышении нагрузки объем RAM и количество vCPU мастера могут автоматически увеличиваться. А если в кластере работает [автоматическое масштабирование мастера](./autoscale.md#master-autoscaler), выбранная конфигурация будет использоваться в качестве нижней границы масштабирования.
