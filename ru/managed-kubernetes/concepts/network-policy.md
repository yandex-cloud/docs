# Сетевые политики кластера {{ k8s }}

Сетевые политики {{ k8s }} позволяют настроить сетевое взаимодействие между группами подов и узлами сети. Вы можете создать сетевые политики с помощью [Kubernetes Network Policy API](https://kubernetes.io/docs/concepts/services-networking/network-policies/), который создает правила фильтрации трафика на уровне подов. Правила определяют, какие поды и сервисы в кластере {{ k8s }} могут получить доступ друг к другу.

Сетевые политики помогает разграничить доступ в кластере {{ k8s }} для многоуровневых приложений, чтобы одни сервисы не могли взаимодействовать с другими.

## Включение сетевых политик {#policy-enable}

Вы можете включить применение сетевых политик при создании кластера {{ k8s }}:
* В консоли управления, выбрав опцию **Включить сетевые политики**.
* С помощью CLI, указав флаг `--enable-network-policy`.
* С помощью метода [create](../api-ref/Cluster/create.md) для ресурса [Cluster](../api-ref/Cluster).

После разрешения применять сетевые политики в кластере {{ k8s }}, вы можете создать ее с помощью Kubernetes Network Policy API. Как работают сетевые политики показано на странице [{#T}](../operations/running-network-policy.md).

{% note info %}

Применение сетевых политик можно настроить только при создании кластера {{ k8s }}.

{% endnote %}

## Ограничения {#limitations}

Для включения сетевых политик в кластере {{ k8s }} проверьте, что у вас:

* Достаточное количество ресурсов в группах узлов.
  Применение сетевых политик требует дополнительных ресурсов памяти и vCPU.

* В кластере {{ k8s }} две узла или более.
  Рекомендуемый минимальный размер кластера {{ k8s }} для включения сетевых политик — два узла.
