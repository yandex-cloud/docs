# Термины и определения {{ yq-full-name }}

## Запрос {#query}

_Запрос_ — это выражение на языке [YQL](https://ydb.tech/ru/docs/yql/reference/syntax/), диалекте SQL, позволяющее унифицированно выполнять потоковые и аналитические запросы к данным. 

Запрос состоит из текста запроса на YQL, информации о [подключении](#connection) к [источнику данных](#source) и [схему данных](#schema) в источнике.


## Подключение {#connection}

_Подключение_ — это набор параметров, необходимых для подключения {{ yq-full-name }} к [источнику данных](#source). Например, при использовании в качестве источника данных файла из {{ objstorage-full-name }}, подключение содержит информацию имени бакета и параметрах авторизации в нем.

Подробнее читайте в разделе [{#T}](../operations/connection.md).


## Привязки к данными {#binding}

Один и тот же YQL-запрос можно выполнять на данных, которые находятся в разных [источниках](#source) (например, для потоковой и аналитической обработки). В этом случае для каждого из источников можно создать _привязку к данным_ — ресурс, который содержит информацию о [подключении](connection), формате и [схеме данных](#schema).

Подробнее читайте в разделе [{#T}](../operations/binding.md).


## Информация о выполненных запросах {#job}

Для каждого выполнения `запроса` {{ yq-full-name }} сохраняет следующую информацию:

* результаты выполнения запроса;
* статус выполнения запроса;
* дата и время начала выполнения запроса;
* продолжительность выполнения запроса;
* имя пользователя, который выполнил запрос;
* метрики исполнения запроса.

{% note info %}

Результат выполнения сохраняется только для последнего выполнения YQ-запроса.

{% endnote %}

## Источник данных {#source}

_Источник данных_ — это объект, содержащий структурированные данные. В {{ yq-full-name }} в качестве источников данных могут использоваться:

* потоки данных [{{yds-full-name}}](../../data-streams/concepts/index.md);
* файлы в хранилище [{{ objstorage-full-name }}](../../storage/concepts/index.md).


## Схема данных {#schema}

_Схема данных_ — это перечень полей и типов данных источника, которые будут использованы в запросе.

## Контрольная точка {#checkpoint}

Системы потокового анализа обрабатывают бесконечные (не имеющие начала и конца) потоки данных. При перезапуске запроса, чтобы не обрабатывать все данные из потока данных каждый раз с начала, {{yq-full-name}} запоминает смещения (offsets) обработанных данных. При остановке и последующем восстановлении обработки {{yq-full-name}} перематывает поток данных на сохраненное смещение и продолжает обработку с этого места. 

Контрольные точки содержат информацию про потоковый запрос, в том числе про смещения внутри потоков данных.

Если добавить в текст запроса обращение к новым потоковым источникам данных, то в контрольных точках не будет информации про смещения внутри потоков данных. Это может привести к тому, что часть данных будет считываться из существующих потоков данных с последней точки, а часть данных с момента появления новых сообщений в новых потоках данных.

{% note info %}

Настройки способа выполнения запроса: обработка данных с контрольной точки или обработка данных заново - задаются в момент [запуска запроса](../operations/query.md#run). 

{% endnote %}
