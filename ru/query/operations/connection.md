# Работа с соединениями

Соединение в {{yq-full-name}} позволяют организовать взаимодействие с источниками данных. {{yq-full-name}} поддерживает взаимодействие с:
- [{{objstorage-full-name}}](../../storage/concepts/index.md);
- [{{yds-full-name}}](../../data-streams/concepts/index.md);
- [{{monitoring-name}}](../../monitoring/concepts/index.md).

## Создание соединений

Для создания соединения выполните следующие действия:

1. Перейдите в раздел **Connections**.
1. Нажмите кнопку **Create**.
1. Укажите имя соединения. Требования к имени:

    {% include [name-format](../../_includes/query/connection-name-format.md) %}

1. Укажите тип источника: 
- **Data Streams** для взаимодействия {{yds-full-name}}.
- **Object Storage** для взаимодействия {{objstorage-full-name}}.
- **Monitoring** для взаимодействия {{monitoring-name}}.

### Соединение с {{objstorage-full-name}}

Для создания соединения с {{objstorage-full-name}} выполните следующие действия:
1. Укажите тип [бакета](../../storage//concepts/bucket.md) {{objstorage-full-name}}: **Public** или **Private**.
    - Укажите **Public**, если для подключения к бакету не требуется авторизация.
        - Введите название бакета, где расположены данные.
    - Укажите **Private**, если для подключения к бакету требуется авторизация.    
        - Укажите cloud и folder, где находится источник данных. 
        - Выберите название бакета, где расположены данные.
        - Выберите [сервисный аккаунт](../../iam/concepts/users/service-accounts.md), от имени которого будет выполняться доступ к данным.
1. Нажмите кнопку **Create** для завершения создания подключения.


### Соединение с {{yds-full-name}}
Для создания соединения с {{yds-full-name}} выполните следующие действия:
1. Укажите cloud и folder, где находится источник данных. 
1. Выберите базу данных [{{ydb-name}}](../../ydb/concepts/index.md), где создан поток данных [{{yds-full-name}}](../../data-streams/concepts/index.md)
1. Выберите [сервисный аккаунт](../../iam/concepts/users/service-accounts.md), от имени которого будет выполняться доступ к данным.
1. Нажмите кнопку **Create** для завершения создания подключения.

### Соединение с {{monitoring-name}}
Для создания соединения с {{monitoring-name}} выполните следующие действия:
1. Укажите cloud и folder, куда будут сохраняться метрики. 
1. Выберите [сервисный аккаунт](../../iam/concepts/users/service-accounts.md), от имени которого будет выполняться запись метрик.
1. Нажмите кнопку **Create** для завершения создания подключения.

## Получение информации о соединении { #info }

1. В разделе **Connections** найдите нужное подключение.
1. Нажмите кнопку ![info](../../_assets/query/info.svg) для просмотра информации о соединении.

В разделе **Meta** находится метаинформация о соединении:
- идентификатор;
- время создания;
- автор.

В разделе **General** находится информация о типе источника, дополнительная информация и данные сервисного аккаунта, используемые для подключения.

## Редактирование соединений

1. В разделе **Connections** найдите нужное подключение.
1. Нажмите кнопку ![pencil](../../_assets/query/pensil.svg) для редактирования параметров соединения.
1. Укажите новые параметры подключения. Тип подключения изменять нельзя.
1. Нажмите кнопку **Modify** для применения изменений.

## Удаление соединений
1. В разделе **Connections** найдите нужное подключение.
1. Нажмите кнопку ![delete](../../_assets/query/delete.svg) для удаления соединения.
1. Подтвердите удаление соединения.
