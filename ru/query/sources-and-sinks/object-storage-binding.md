# Чтение данных из {{ objstorage-name }} с помощью привязок к данным {{ yq-name }}

При работе с {{ objstorage-full-name }} с помощью привязок к данным удобно выполнять запросы к хранимым данным на регулярной основе без необходимости указывать все детали работы с этими данными.

```sql
SELECT
        *
FROM
    bindings.`logs`
WHERE
    version>1
```

## Настройка привязки к данным

Для чтения данных из {{ objstorage-full-name }} c помощью привязок к данным необходимо:
1. [Настроить подключение к данным](./object-storage.md#create_connection).
1. В поле тип выбрать `Object storage`.
1. В выпадающем поле `Соединение` выбрать соединение, созданное в первом пункте.
1. В поле `Name` указать название привязки к данным.
1. В поле `Путь` указать [путь к данным внутри бакета](object-storage.md#path_format). Поддерживается указание частей имен файлов через \*, например, "yellow\*.csv". 
1. В поле `Сжатие` указать способ [сжатия данных](formats.md#compression).
1. В поле `Формат` указать [формат хранения данных](formats.md#formats).
2. В полях `Колонки` указать список колонок в данных и их типы данных. 
3. В полях `Партиционирование данных` указать список [колонок партиционирования](../concepts/partitioning.md).
4. При использовании `Расширенного партиционирования` указать колонки [расширенного партиционирования](../concepts//partition-projection.md) и правила работы с ними в разделе `Дополнительно`.
5. Для проверки данных нажать кнопку `Предпросмотр`.
6. Создать привязку к данным, нажав кнопку `Create`.


## Модель данных

Данные хранятся в {{ objstorage-full-name }} хранятся в файлах в бинарном виде. Чтение данных выполняется с помощью SQL-выражения

```sql
SELECT
    <expression>
FROM
    bindings.`binding_name`
WHERE
    <filter>;
```

Где:

- `bindings` - ключевое слово, означающую работу с {{objstorage-full-name}} с помощью привязок к данным.
- `binding_name` — название ранее созданной привязки к данным.
