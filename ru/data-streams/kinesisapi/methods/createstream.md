# CreateStream

Создает [поток данных](../../concepts/glossary.md#stream-concepts).

При успехе метод инициирует создание потока, переводит создаваемый поток в статус `CREATING` и возвращает HTTP-ответ с кодом 200. Операции чтения и записи возможны только для потока в статусе `ACTIVE`. Чтобы узнать статус потока, используйте метод [DescribeStream](describestream.md).

При попытке создать больше потоков, чем установлено [квотой](../../concepts/limits.md) для вашей учетной записи, запрос вернет ошибку `LimitExceededException`.

## Запрос {#request}

Запрос содержит данные в формате JSON.

```json
{
  "StreamName": "string",
  "ShardCount": number,
  "WriteQuotaKbPerSec": number,
  "RetentionPeriodHours": number
}
```

### Параметры запроса {#request-options}

Параметр | Описание
----- | -----
`StreamName` | Имя создаваемого потока данных.<br/><br/>**Тип**: Строка<br/>**Размер**: `1`-`128` символов.<br/>**Возможные значения**: `[a-zA-Z][a-zA-Z0-9-]+*(?<!-)$`<br/>**Обязательный**: Да
`ShardCount` | Количество [сегментов](../../concepts/glossary.md#shard) в потоке.<br/><br/>**Тип**: Целое число<br/>**Минимальное значение**: `1`.<br/>**Обязательный**: Да
`WriteQuotaKbPerSec` | Ограничение на запись в поток данных, КБ/с.<br/><br/>**Тип**: Целое число<br/>**Возможные значения**: `128`, `512`, `1024`.<br/>**Обязательный**: Нет
`RetentionPeriodHours` | Время хранения сообщений, часы.<br/><br/>**Тип**: Целое число<br/>**Возможные значения** `4`, `12`, `24`.<br/>**Обязательный**: Нет

## Ответ {#response}

В случае успеха возвращается HTTP-ответ с кодом 200 и пустым телом.

## Ошибки {#errors}

Тип ошибки | Описание | Код HTTP
----- | ----- | -----
`InvalidArgumentException` | Недопустимый аргумент. Дополнительные сведения смотрите в сообщении об ошибке. | 400
`LimitExceededException` | Превышено максимальное число запросов. | 400
`ResourceInUseException` | Ресурс используется другой операцией. | 400

Могут возникать [ошибки](../common-errors.md), общие для всех методов.
