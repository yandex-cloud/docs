# DNS-резолвер

При создании в [облачной сети (VPC)](../../vpc/concepts/network.md#network) новой [подсети](../../vpc/concepts/network.md#subnet) второй с начала IP-адрес в этой подсети (условно `.2`) всегда резервируется для *DNS-резолвера*, который обрабатывает DNS-запросы от ресурсов, подключенных к этой подсети.

При создании ВМ в подсети IP-адрес резолвера отправляется вместе с другими параметрами сетевого интерфейса по протоколу DHCP к создаваемой ВМ. Этот IP-адрес используется внутри ВМ как адрес DNS-сервера.

IP-связность от ВМ до резолвера обеспечивается с помощью специального компонента [VRouter-agent](../../vpc/concepts/network-overview.md#vrouter-agent), который доступен только внутри ВМ и недоступен извне. Для интеграции корпоративных DNS-сервисов, связанных с облачными ресурсами, необходимо [развернуть и настроить ВМ с функцией DNS Forwarder](../tutorials/dns-integration.md).
