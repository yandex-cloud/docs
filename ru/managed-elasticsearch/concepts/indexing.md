# Индексы

{% include [Elasticsearch-end-of-service](../../_includes/mdb/mes/note-end-of-service.md) %}

Когда документ сохраняется в {{ ES }}, он индексируется и помещается в определенный _индекс_ по выбору пользователя, становясь доступным для поиска и анализа. Индекс можно рассматривать как аналог таблицы с данными в классических СУБД.

С точки зрения {{ ES }}, документ — это набор полей, где каждое поле — это пара `ключ: значение`. Индекс хранит документ в оптимизированном виде, чтобы обеспечить возможность быстрого поиска по полям в документе. Оптимизация достигается за счет того, что каждое поле документа имеет определенный тип — это позволяет эффективно хранить данные этого поля в индексе.

В отличие от классических СУБД, {{ ES }} не требует явного задания схемы (взаимосвязей между полями документа и их типами), чтобы сохранить документ в индексе. Хотя это и рекомендуемый подход, можно сохранять документы в индекс без явного указания типа полей — {{ ES }} попытается определить тип автоматически для каждого поля документа. Это позволяет быстро наполнить хранилище {{ ES }} документами и начать работу с ними.
 
Подробнее об устройстве индексов см. в [документации {{ ES }}](https://www.elastic.co/guide/en/elasticsearch/reference/current/documents-indices.html).

В многохостовых кластерах доступны [шардирование и репликация](scalability-and-resilience.md) индексов. Это упрощает масштабирование кластера и повышает его отказоустойчивость.
