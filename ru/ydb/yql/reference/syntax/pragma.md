# PRAGMA

Переопределяет настройки разного рода. Влияет на все идущие следом выражения. При необходимости и логической возможности допустимо менять значение настройки несколько раз в одном запросе, чтобы оно было разным на разных этапах выполнения.

Общая схема `PRAGMA x.y = "z";` или `PRAGMA x.y("z", "z2", "z3");`, где:

* `x` — категория настройки, опционально;
* `y` — название настройки;
* `z` — её значение, опционально для флагов.

Если значение измеряется в байтах, то можно использовать суффиксы `Kb`, `Mb`, `Gb`. Если временем — суффиксы `sec`, `min`, `h`, `d`.

За некоторым исключением, значение настроек можно вернуть в состояние по умолчанию с помощью `PRAGMA my_pragma = default;` (без кавычек).

Полный список доступных настроек приведен в таблице ниже.

**Примеры**

```sql
PRAGMA AutoCommit;
```

## Список доступных PRAGMA в YQL

### Глобальные

| Название | Тип значения | По умолчанию | Описание |
| --- | --- | --- | --- |
| `AutoCommit` | Флаг | false | Автоматически выполнять [COMMIT](select.md#commit) после каждого выражения. |
| `SimpleColumns` | Флаг | false | При использовании  `SELECT foo.* FROM ... AS foo` убрать префикс  `foo.` у имен результирующих колонок.<br><br>Работает в том числе и для [JOIN](join.md), но в этом случае имеет право упасть в случае конфликта имен (который можно разрешить с помощью [WITHOUT](../syntax/select.md#without) и переименования колонок). |
| `TablePathPrefix` | Строка | — | Добавить указанный префикс к путям таблиц внутри кластеров. Работает по принципу объединения путей в файловой системе: поддерживает ссылки на родительский каталог `..` и не требует добавления слеша справа. Например, `PRAGMA TablePathPrefix = "home/yql"; SELECT * FROM test;`.<br><br>Префикс не добавляется, если имя таблицы указано как абсолютный путь (начинается с /). |
| `Warning` | 1. Действие<br>2. Код предупреждения либо символ "*"| — | Действие: <ul><li>`disable` — отключить;</li><li>`error` — приравнять к ошибке;</li><li>`default` — вернуть поведение по умолчанию.</li></ul> Код предупреждения возвращается вместе с самим текстом (в веб-интерфейсе отображается справа).<br><br>Пример:<br>`PRAGMA Warning("error", "*");`<br> `PRAGMA Warning("disable", "1101");`<br> `PRAGMA Warning("default", "4503");`<br> В данном случае все предупреждения будут считаться ошибками, за исключением предупреждение с кодом 1101, которое будет отключено, и 4503, которое будет обрабатываться по умолчанию (то есть останется предупреждением). |
| `ClassicDivision` | Флаг | true | В классическом варианте результат целочисленного деления остаётся целочисленным (по умолчанию).<br>Если отключить — результат всегда становится Double. |
| `AllowDotInAlias` | Флаг | false | Разрешить использовать точку в именах результирующих колонок. По умолчанию отключено, т.к. дальнейшее использование таких колонок в JOIN полностью не реализовано. |
| `GroupByLimit` | Положительное число | 32 | Увеличение лимита на число  группировок в [GROUP BY](group_by.md). |
| `GroupByCubeLimit` | Положительное число | 5 | Увеличение лимита на число размерностей [GROUP BY](group_by.md#cube).<br><br>Использовать нужно аккуратно, так как вычислительная сложность запроса растет экспоненциально по отношению к числу размерностей. |
| `kikimr.IsolationLevel` | Serializable, ReadCommitted, ReadUncommitted или ReadStale | Serializable | Экспериментальная pragma, позволяет ослабить уровень изоляции текущей транзакции в YDB. |
