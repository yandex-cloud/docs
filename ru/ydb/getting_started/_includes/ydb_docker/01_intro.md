# Использование Docker-контейнера YDB

Для отладки или тестирования вы можете запустить Yandex Database в Docker-контейнере. Docker-контейнер содержит актуальную версию сборки YDB, ревизия сборки может отличаться.

В режиме локального запуска для работы с базой данных предоставляется только YDB API. API доступен на эндпоинте `grpcs://localhost:2135`, имя базы данных `/local`. Работа с базой выполняется с помощью консольного клиента YDB ([YDB CLI](../../../reference/ydb-cli/index.md)), предустановленного внутри Docker-образа. 

Yandex Database в Docker-контейнере принимает входящие соединения, защищённые TLS. Сертификаты генерируются автоматически. Для использования сертификатов необходимо смонтировать на хост-системе директорию `/ydb_cert` Docker-контейнера.

Для локального сохранения состояния базы данных на хост-системе монтируется директория `/ydb_data` Docker-контейнера.

{% note warning %}

При работе локальной базы данных, в зависимости от задач, может использоваться значительная часть вычислительных ресурсов хост-системы.

{% endnote %}
