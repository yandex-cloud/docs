## Как начать работать с YDB {#how-to-get-started-with-ydb}

Основная точка входа в YDB - [https://ydb.yandex-team.ru](https://ydb.yandex-team.ru)

### Попробовать YDB {#try-ydb}
Если вы хотите попробовать YDB, достаточно зайти в [web-интерфейс](https://ydb.yandex-team.ru) и нажать кнопку "Create database"

![Create database](../_assets/db_ui_create_database_button.png)

1. Выберите расположение базы данных

Можно выбрать географическое расположение для вашей БД.
Если достаточно кластера в одном датацентре, выбирайте dev-sas-slice2 (Сасово).
Если нужен кросс-датацентровый кластер, выбирайте ydb-ru (Мытищи-Владимир-Сасово) или ydb-eu (Мянтсяля-Владимир-Сасово).

2. Выберите аккаунт

Если вы только начинаете работать с кластерами YDB, и аккаунта вашего проекта еще нет, заполните, пожалуйста, [форму](https://st.yandex-team.ru/createTicket?queue=KIKMIR) на создание аккаунта.
По-умолчанию будет выбран аккаунт home.

3. Создайте или выберите папку

По-умолчанию создатся папка с вашим именем пользователя

4. Выберите имя для базы данных

По-умолчанию создастся база данных с именем mydb.
Для имени базы данных разрешены к использованию только печатные символы латинского алфавита.

### Вычислительные ресурсы {#computing-resources}
Вычислительные слоты позволяют увеличить количество запросов в секунду, которое будет обрабатывать база данных.
Один слот ограничен XXGb оперативной памяти и YY ядрами.

5. Выберите количество вычислительных слотов, которое вам требуется. Мы рекомендуем N слотов для нагрузки ZZZZrps.

6. Если выбрано более одного вычислительного слота, и, кросс-датацентровый кластер, можно настроить политику распределения слотов по датацентрам.

Возможные варианты :
* Auto - слоты будут расположены в одном датацентре, в случае выхода из строя датацентра, слоты переедут в доступные датацентры.
* In one DC - слоты будут расположены в одном датацентре, в случае его выхода из строя, все слоты переедут в другой датацентр, но также будут располагаться в пределах одного датацентра
* Uniform distribution - слоты равномерно распределены по датацентрам, в случае выхода из строя одного из датацентров, слоты переезжают в оставшиеся датацентры.


### Хранение данных {#data-storage}
Можно выбирать хранилища разного типа с учетом необходимой пропускной способности, профиля нагрузки, rps запросов и требованиям к избыточности (отказоустойчивости).
Для хранение холодных данных выбирайте HDD.
Для хранения горячих данных (OLTP профиля нагрузки) выбирайте SSD.

Доступны хранилища со следующими видами избыточности:
* mirror-3 - данные хранятся в виде 3 полноценных реплик, восстановление возможно при потере не более 2 реплик.
* block-M-N (block-4-2) - данные хранятся в виде M фрагментов, являющихся непересекающимися подстроками исходных данных, дополняемых N фрагментами избыточности. Восстановление возможно при потере не более N реплик.

### Старое доброе, нужно переработать и удалить {#good-old-need-to-be-recycled-and-removed}

Попробовать KiKiMR в деле можно на общедоступном кластере Public Playground (имя балансера: playground.ydb.yandex.net). Это кластер, развёрнутый в одном дата-центре.

С кластером можно работать через любой API (C++/Python/HTTP/CLI), подробнее можно почитать в разделе [APIs and References](../api/api_libs_overview/).

Кластер KiKiMR Public Playground доступен через [YQL](http://yql.yandex-team.ru). Необходимо выбрать тип кластер KiKiMR и конкретный кластер `Ydbtest` (вот [пример операции](https://yql.yandex-team.ru/Operations/Wfbe98qy16E0f0wX9hDQ5RGv71WN_1iIKXfxWdmsRz4=)).

Ещё можно попробовать кросс-дц кластер, доступный за балансером  3dc-ydb-dev.yandex.net
Этот кластер тоже доступен через yql веб-интерфейс под именем `ydbtest_3dc`

Вот пример [операции](https://yql.yandex-team.ru/Operations/WrKQwSdq_dyaIHdHnoUtld_w8u2XDxLLHaZZTp-Mo_A=)
