# Трансляции в {{ video-name }}

С помощью сервиса {{ video-name }} вы можете организовать [онлайн-трансляции](#streams) в интегрированном [видеоплеере](./player.md). Каждая трансляция — отдельный [эпизод](#episodes).

## Трансляции {#streams}

_Трансляция_ — это основной ресурс {{ video-name }} для передачи видеопотока в прямом эфире.

В настройках трансляции устанавливаются параметры источника видеосигнала.

Поддерживаются протоколы входного потока [RTMP](https://ru.wikipedia.org/wiki/RTMP) и [SRT](https://en.wikipedia.org/wiki/Secure_Reliable_Transport).

## Эпизоды {#episodes}

Каждая трансляция состоит из _эпизодов_. Эпизоды бывают двух типов:

* `{{ ui-key.yacloud_video.streams.label_episode-type-live }}` — просмотр в реальном времени с перемоткой назад.
* `{{ ui-key.yacloud_video.streams.label_episode-type-broadcast }}` — событие с заданным временем начала и окончания и записью.

Для эпизодов доступна загрузка пользовательских обложек. Обложка будет отображаться в интерфейсе {{ video-name }}, а также в плеере на сайте, где размещен эпизод трансляции.

{% include [image-characteristic](../../_includes/video/image-characteristic.md) %}

Вы можете [опубликовать](../operations/streams/get-link.md) эпизод трансляции по прямой ссылке или разместить на сайте. Чтобы встроить на сайте только часть трансляции, укажите необходимый временной интервал при создании или редактировании эпизода.

Эпизоды трансляции можно группировать в [плейлисты](./playlists.md) в определенной последовательности. Подробнее о публикации плейлистов см. в разделе [{#T}](../operations/playlists/get-link.md).

После завершения эпизода трансляции вы можете оставить доступной его запись или удалить его.

### Статусы {#statuses}

Статусы отображаются в блоке **{{ ui-key.yacloud_video.streams.title_stream-episodes }}** в соответствующем столбце.

Статусы эпизодов трансляции:

* `{{ ui-key.yacloud_video.streams.status_offline }}` — трансляция создана и еще не начата.
* `{{ ui-key.yacloud_video.streams.status_preparing }}` — происходит подготовка отказоустойчивой инфраструктуры {{ yandex-cloud }} для транскодирования видеосигнала, ожидается получение видеосигнала. Трансляция переходит в этот статус за несколько минут до начала.
* `{{ ui-key.yacloud_video.streams.status_ready }}` — инфраструктура готова, видеосигнал получен, можно начинать трансляцию.
* `{{ ui-key.yacloud_video.streams.status_on-air }}` — трансляция запущена. Переход в этот статус происходит после нажатия кнопки **{{ ui-key.yacloud_video.streams.action_start-stream }}** или автоматически в указанное время.
* `{{ ui-key.yacloud_video.streams.status_finished }}` — трансляция окончена. Переход в этот статус происходит после нажатия кнопки **{{ ui-key.yacloud_video.streams.action_stop-stream }}** или автоматически в указанное время. Завершенную трансляцию можно добавить в плейлист.

### Параметры публикации эпизода трансляции {#stream-parameters}

Вы можете изменять следующие базовые настройки воспроизведения эпизода трансляции при [генерации](../operations/streams/get-link.md) прямой ссылки или кода для вставки трансляции на сайт:

* воспроизведение без звука по умолчанию;
* автоматический запуск воспроизведения при открытии;
* отображение элементов управления трансляцией в плеере.

{% include [iframe-settings](../../_includes/video/iframe-settings.md) %}

По умолчанию задержка между источником и воспроизведением трансляции составляет 15–20 секунд — это обеспечивает высокое качество картинки и устойчивость к нестабильному соединению. Для сценариев с активным взаимодействием со зрителями задержку можно сократить до 4–5 секунд, но это требует высокого качества сети.
