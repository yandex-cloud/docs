# Решение проблем с загрузкой и обработкой видео

В этом разделе описаны возможные проблемы при загрузке и обработке видеофайлов в {{ video-name }} и способы их решения.

## Ошибка обработки видеофайла {#processing-error}

{{ video-name }} не может корректно обработать файл из-за поврежденных данных в контейнере, в видео- или аудиопотоках. Это бывает при сбоях записи из-за неполадок с камерой или картой памяти, повреждении при копировании, перекодировании и так далее.

При этом на вашем компьютере файл может открываться и проигрываться, так как плееры могут маскировать ошибки, пропускать битые пакеты и перестраивать изображение по ключевым кадрам. {{ video-name }} не позволяет загрузить такой файл — его нужно исправить.

### Как проверить файл {#check-file}

Повторно [загрузите](../operations/video/upload.md) файл. Если ошибка сохраняется — файл, скорее всего, поврежден.

Проверьте это, открыв его в стороннем видеоплеере (Windows Media Player, QuickTime, VLC и т. д.). В поврежденном файле возможны зависания, артефакты, черный экран и другие визуальные или звуковые ошибки.

Учитывайте, что проблемы могут проявляться в любом месте файла: даже несколько поврежденных кадров делают файл некорректным для обработки.

### Как исправить файл {#fix-file}

1. Повторно экспортируйте файл, если он создан в видеоредакторе.
1. Если нет возможности восстановить исходный файл, перекодируйте его в рекомендованный формат с помощью утилиты [FFmpeg](https://www.ffmpeg.org), игнорируя поврежденные участки:

   ```bash
   ffmpeg -err_detect ignore_err \
          -i <входной_файл> \
          -c:v libx264 \
          -crf 23 \
          -c:a aac \
          -b:a 192k \
          -map 0 \
          <выходной_файл>
   ```

1. (Опционально) При необходимости обрежьте поврежденный фрагмент и сохраните новую версию.
1. Повторно загрузите исправленный файл в {{ video-name }}.

### Когда обращаться в поддержку {#contact-support}

Если после перекодирования файл воспроизводится без ошибок, но обработка в {{ video-name }} по-прежнему завершается ошибкой, обратитесь в [поддержку]({{ link-console-support }}) и укажите идентификатор видео.
