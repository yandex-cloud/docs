## Общие вопросы {#overview}

### {{ datalens-name }} нельзя использовать бесплатно? {#for-free}

Можно, для индивидуального использования {{ datalens-name }} доступен бесплатно вместе со всеми возможностями. Платной является только командная работа.

### Пользователь только смотрит дашборды, ему нужно рабочее место? {#viewer}

Да, рабочие места нужны всем пользователям, которые авторизуются в сервисе, независимо от того, какие у них роли.

### Если я не каждый день использую {{ datalens-name }}, рабочее место будет тарифицироваться только в дни использования? {#activity}

Нет, если рабочее место куплено, то будет списана его полная стоимость, даже если в некоторые дни пользователь с этим рабочим местом не открывал сервис. Администратор может переназначать рабочие места между пользователями в периоды неактивности.

## Управление рабочими местами {#manage-seats}

### Где настраиваются рабочие места? {#seats-settings}

У администратора есть инструмент для управления рабочими местами в [настройках сервиса](../../datalens/settings/seats.md). Он может:
* просматривать количество рабочих мест в экземпляре, в том числе свободных (которые никому не назначены);
* просматривать список пользователей с рабочими местами;
* переназначать рабочие места от пользователя к пользователю;
* увеличивать или сокращать число рабочих мест в экземпляре.

### После увольнения сотрудника его рабочее место освобождается автоматически? {#retire}

Нет, администратор должен назначить это рабочее место другому сотруднику.

### Можно ли использовать одно рабочее место нескольким пользователям, если они не будут работать одновременно? {#multiple-users}

У администратора есть возможность вручную переназначать рабочие места.

## Запросы из непублично встроенных объектов {#embeds}

### Пользователям, которые просматривают непублично встроенные объекты, нужно купить рабочие места в {{ datalens-name }}? {#seats-for-external-clients}

Рабочие места нужны пользователям, которые работают непосредственно в сервисе {{ datalens-name }}. Если на внешних ресурсах, например в личных кабинетах ваших клиентов, есть [непублично встроенные чарты](../../datalens/security/private-embedded-objects.md), то покупать рабочие места для этих клиентов не нужно. Для просмотра непублично встроенных объектов им не нужно заходить в {{ datalens-name }}.

Подсчитывается количество запросов к источникам из непублично встроенных чартов (в визарде, Editor, QL-чартов), которые выполняются при отрисовке этих объектов. Считаются только успешно выполненные запросы на получение данных для отрисовки чартов. Если для отрисовки одного чарта выполняется несколько запросов, это учитывается как один запрос. Для непублично встроенных дашбордов суммируются запросы из добавленных на него чартов.

Общее количество запросов от всех непубличных чартов экземпляра за месяц не должно превышать значения:

> 2 000 запросов × Количество рабочих мест

Если вы не укладываетесь в это ограничение, вам потребуется докупить рабочие места. При превышении лимита на количество запросов из непублично встроенных чартов и включенной [автоматической покупкой](../../datalens/settings/seats.md#purchase-automatically) рабочих мест автоматическая докупка новых рабочих мест не происходит.

### Как я могу понять, сколько у меня выходит запросов из непублично встроенных объектов? {#statistics}

Оцените количество выполняемых запросов из непублично встроенных объектов с помощью подключения [Usage Analytics](../../datalens/operations/connection/create-usage-tracking.md), которое предоставляет статистику использования сервиса. Количество запросов из непублично встроенных объектов можно найти в таблице `Top Users (Requests)`, в строке `_ANONYMOUS_USER_OF_EMBEDDED_DATALENS_`.

### Что делать, если мне нужно больше запросов из непублично встроенных объектов? {#more-queries}

Если текущее количество рабочих мест не покрывает необходимое количество запросов из непублично встроенных объектов, вам потребуется приобрести дополнительные рабочие места.

### Если у меня одно рабочее место, но в месяц выходит больше 2 000 запросов из непублично встроенных объектов, у меня все бесплатно? {#individual-more-queries}

Нет, в этом случае вам потребуется приобрести дополнительные рабочие места, чтобы укладываться в ограничение на количество запросов из непублично встроенных объектов.

### У меня на сайте встроен публичный чарт/дашборд, запросы из него тоже будут подсчитываться? {#public-embeds}

[Публичные ссылки](../../datalens/concepts/datalens-public.md) на дашборды и чарты, которые доступны без аутентификации, при подсчете количества запросов к непублично встроенным объектам не учитываются.

В подключении [Usage Analytics](../../datalens/operations/connection/create-usage-tracking.md), которое предоставляет статистику использования сервиса, количество запросов из публичных объектов можно найти в таблице `Top Users (Requests)`, в строке `_ANONYMOUS_USER_OF_PUBLIC_DATALENS_`.
