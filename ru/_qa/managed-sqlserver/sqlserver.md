#### Какие версии {{ MS }} поддерживает {{ mms-short-name }}? {#supported-version}

#### Включено ли резервное копирование кластеров БД по умолчанию? {#backup}

Да, по умолчанию резервное копирование включено. Для кластеров {{ MS }} выполняется полное резервное копирование один раз в сутки и сохраняются все журналы транзакций кластера БД. Это позволяет восстановить состояние кластера на любой момент времени в пределах периода хранения резервных копий, за исключением последних 30 секунд.

#### Шифруется ли соединение с кластером БД {{ MS }}? {#encryption}

Соединение между кластером БД и приложением всегда шифруется с использованием SSL. Отключить шифрование соединений с кластером нельзя.

#### Что такое реплика для чтения в {{ MS }}? {#read-only-instance}

Реплика для чтения — это доступный только для чтения хост в кластере БД {{ MS }}, данные на котором синхронизируются с хостом-мастером (применимо только если в кластере работает более 1 хоста). Вы можете использовать реплику для чтения для снижения нагрузки на мастер в базах данных с большим объемом запросов на чтение.
