# Вопросы о регулярном экспорте детализации

#### За какой самый ранний период можно выгрузить данные при экспорте за прошлые периоды? {#earliest-period}
Можно выгрузить данные начиная с даты создания [платежного аккаунта](../../billing/concepts/billing-account.md).

#### Что произойдет, если при создании экспорта за прошлые периоды указать дату, за которую нет данных? {#no-data-period}
Система выгрузит данные начиная с даты создания [платежного аккаунта](../../billing/concepts/billing-account.md). Ошибки не возникнет.

#### Обновляются ли CSV-файлы за прошлые периоды после их первоначальной выгрузки? {#csv-files-update}
Файлы за прошлые периоды обновляются в случае изменения данных о потреблении в ранее выгруженный период.

#### Какие именно типы шифрования бакетов поддерживаются для регулярного экспорта? {#encryption-types}
Регулярный экспорт поддерживает бакеты {{ objstorage-full-name }} с использованием ключей шифрования сервиса [{{ kms-full-name }}](../../kms/).

Подробнее см. в разделе [Шифрование в Object Storage](../../storage/concepts/encryption.md) документации сервиса {{ objstorage-full-name }}.

#### Как проверить, что выгрузка в бакет с шифрованием прошла успешно? {#check-encrypted-bucket}
Перейдите в бакет {{ objstorage-full-name }}, указанный при настройке [регулярного экспорта](../../billing/operations/get-folder-report.md#regular-download). Убедитесь в наличии CSV-файлов. Файлы появятся через 5–10 минут после создания регулярной выгрузки. Экспорт за длительные периоды (более года) может занять больше времени.

#### Можно ли изменить параметры регулярного экспорта после его создания? {#change-parameters}
Нет. Чтобы изменить параметры (язык, тип детализации, бакет), [прекратите](../../billing/operations/get-folder-report.md#stop-export) существующий регулярный экспорт и [настройте новый](../../billing/operations/get-folder-report.md#set-up-regular-download).

#### Можно ли создать экспорт за прошлые периоды с использованием зашифрованного бакета? {#past-period-encrypted}
Да. Вы можете создать экспорт за прошлые периоды с использованием зашифрованного бакета.

#### Что произойдет, если выгрузка в бакет завершилась с ошибкой? {#export-failed}
Статус регулярной выгрузки сменится на `Failed`. Владелец платежного аккаунта получит письмо на почту.
