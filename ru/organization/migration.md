# Переезд из Яндекс.Коннекта

По сравнению с Почтой для домена, Яндекс.Коннект предоставляет больше возможностей для управления доменами и почтовыми ящиками пользователей. Миграция в Коннект происходит в автоматическом режиме, вам не нужно ничего для этого делать. Изменения коснутся только инструментов администратора, на работе почтовых ящиков миграция никак не скажется.

> [!NOTE]
> Вы можете не дожидаться автоматического переезда и запустить его [вручную](#section_lx2_cz1_dcb).

## Мигрировать в Коннект {#section_lx2_cz1_dcb}

Чтобы перенести ваш домен и почтовые ящики с Почты для домена в Коннект:

1. Убедитесь, что вы авторизованы в Почте для домена как администратор.
1. Перейдите на .
1. Нажмите на ссылку **Мигрировать в Коннект**.
1. Дождитесь сообщения об успешном переносе ваших данных в Коннект. Обычно это занимает несколько минут.


> [!WARNING]
> Аккаунт администратора Почты для домена не подходит для работы с сервисами Коннекта. Чтобы иметь возможность использовать сервисы Коннекта, [создайте](org-structure.md#manage-user) для себя аккаунт сотрудника. При необходимости наделите его правами администратора.

## Отличия Яндекс.Коннекта и Почты для домена {#section_gyg_dx1_dcb}

При миграции с Почты для домена в Яндекс.Коннект происходят следующие изменения:

- Почтовые ящики Почты для домена становятся [аккаунтами сотрудников](org-structure.md) вашей организации.
- Рассылки Почты для домена становятся [командами](manage-users.md) Коннекта. Пользователи, которые подписаны на рассылку, становятся участниками команды.
    У автоматически созданных команд нет администраторов. Назначить администратора такой команды может только администратор организации в Коннекте.
- Администратор Почты для домена получает возможность управлять настройками организации в Коннекте.
- Подтверждение доменов в Коннекте реализовано на странице  и описано в разделе помощи Коннекта.
- Настройки DNS-записей для делегированных на Яндекс доменов теперь доступны на странице .


> [!NOTE]
> Если к Почте для домена подключено несколько доменов (кроме ), для них будут созданы отдельные организации.

Я пользуюсь API Почты для домена. Смогу ли я использовать API после переезда?
:   Да, сможете. После переезда настраивать API заново не понадобится, а все его возможности сохранятся.
    Для новых доменов советуем вам попробовать API Коннекта. API Почты для доменов больше не будет развиваться, а в API Коннекта мы планируем внедрять новые полезные функции.
    Посмотреть, как устроено API Коннекта, можно в .

Мой почтовый домен подключен к Битрикс24. Переезд в Коннект не повлияет на его работу?
:   Работа над интеграцией с Битрикс24 еще не завершена. Пока мы не переносим пользователей Битрикс24 в Коннект в автоматическом режиме. Если вы используете Битрикс24, не переносите свои домены вручную. Мы сообщим, когда появится возможность подключения доменов в Яндекс.Коннект к Битрикс24.

Я пользуюсь только Почтой и другие сервисы мне не нужны. Могу я не переезжать в Коннект?
:   Владельцы ящиков на вашем домене не заметят изменений — интерфейсы почты в ПДД и Яндекс.Коннекте почти не отличаются. Для доменов, которые мы переносим автоматически, по умолчанию отключены сервисы Коннекта для совместной работы.
    Основные изменения касаются интерфейса управления доменами и аккаунтами сотрудников. В Коннекте у администратора больше возможностей. Например, он может создать систему учётных записей сотрудников, которая будет отражать структуру компании.

Как просмотреть список всех ящиков на домене?
:   При переезде из ПДД почтовые ящики становятся аккаунтами сотрудников. Просмотреть список сотрудников можно в разделах  и .

Вижу в списке ящик `all@yourdomain.com`. Что это такое?
:   Адрес вида `all@yourdomain.com` — это общая рассылка вашей организации. У рассылки нет отдельного почтового ящика. Письма, отправленные на адрес общей рассылки, получат все сотрудники вашей организации.

Как создать почтовый ящик?
:   Почтовые ящики в Яндекс.Коннекте привязаны к аккаунтам сотрудников. Чтобы добавить нового сотрудника:
    1. Убедитесь, что вы авторизованы в Яндекс.Коннекте как администратор.
    1. Перейдите на страницу .
    1. Под списком сотрудников выберите .
    1. Настройте параметры и сохраните изменения.
    Подробнее о добавлении сотрудников читайте в разделе [[!TITLE org-structure.md]](org-structure.md).

Как создать рассылку?
:   При переезде из ПДД рассылки становятся командами. У каждой команды есть почтовый адрес. Если отправить письмо на адрес команды, его получат все ее участники.
    Чтобы создать новую команду:
    1. Откройте страницу [Команды](https://connect.yandex.ru/portal/staff/groups) и нажмите **Добавить**.
    1. Настройте параметры команды и сохраните изменения.
    1. Чтобы добавить участников команды, выберите ее из списка и нажмите **Добавить** в блоке **Участники**.
    Подробнее о командах читайте в разделе [[!TITLE manage-users.md]](manage-users.md).

Как изменить пароль от почтового ящика?
:   Почтовые ящики в Яндекс.Коннекте привязаны к аккаунтам сотрудников. Чтобы изменить пароль сотрудника:
    1. Убедитесь, что вы авторизованы в Яндекс.Коннекте как администратор.
    1. Перейдите на страницу .
    1. Выберите сотрудника из списка.
    1. На карточке сотрудника нажмите .
    1. Настройте пароль и сохраните изменения.
    Подробнее о настройке аккаунтов сотрудников читайте в разделе [[!TITLE edit-account.md]](edit-account.md).
