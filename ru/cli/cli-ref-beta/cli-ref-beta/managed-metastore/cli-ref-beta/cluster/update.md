# yc beta managed-metastore cluster update

Updates the configuration of the specified Metastore Cluster.

#### Command Usage

Syntax:

`yc beta managed-metastore cluster update <CLUSTER-ID>`

#### Flags

#|
||Flag | Description ||
|| `--cluster-id` | `string`

ID of the Metastore Cluster to update. ||
|| `--config-spec` | `shorthand/json`

Metastore Cluster configuration.

Shorthand Syntax:

```hcl
{
  resources = {
    resource-preset-id = string
  }
}
```

JSON Syntax:

```json
{
  "resources": {
    "resource-preset-id": "string"
  }
}
```

Fields:

```
resources -> (struct)
  Configuration for computational resources for Metastore server instances.
  resource-preset-id -> (string)
    ID of the preset for computational resources allocated to an instance (e.g., CPU, memory, etc.).
``` ||
|| `--deletion-protection` | Deletion Protection prevents deletion of the cluster. ||
|| `--description` | `string`

New description of the Metastore Cluster. ||
|| `--labels` | `map<string><string>`

Custom labels for the Metastore Cluster as '' key:value '' pairs. For example: {"env": "prod"}. ||
|| `--logging` | `shorthand/json`

Cloud logging configuration.

Shorthand Syntax:

```hcl
{
  destination = folder-id=string | log-group-id=string,
  enabled = boolean,
  min-level = TRACE|DEBUG|INFO|WARN|ERROR|FATAL
}
```

JSON Syntax:

```json
{
  "destination": {
    "folder-id": "string",
    "log-group-id": "string"
  },
  "enabled": "boolean",
  "min-level": "TRACE|DEBUG|INFO|WARN|ERROR|FATAL"
}
```

Fields:

```
enabled -> (boolean)
  Logs generated by the Metastore server are delivered to Cloud Logging.
min-level -> (struct)
  Minimum severity level for log entries. See [LogLevel.Level] for details.
destination -> (oneof<folder-id|log-group-id>)
  Oneof destination field
  folder-id -> (string)
    Logs will be written to the default log group of the specified folder.
  log-group-id -> (string)
    Logs will be written to the log group specified by its ID.
``` ||
|| `--maintenance-window` | `shorthand/json`

Maintenance window.

Shorthand Syntax:

```hcl
{
  policy = anytime={} | weekly-maintenance-window={
    day = MON|TUE|WED|THU|FRI|SAT|SUN,
    hour = integer
  }
}
```

JSON Syntax:

```json
{
  "policy": {
    "anytime": {},
    "weekly-maintenance-window": {
      "day": "MON|TUE|WED|THU|FRI|SAT|SUN",
      "hour": "integer"
    }
  }
}
```

Fields:

```
policy -> (oneof<anytime|weekly-maintenance-window>)
  Oneof policy field
  anytime -> (struct)
    The cluster may be restarted for maintenance at any time.
  weekly-maintenance-window -> (struct)
    Maintenance is allowed only within the specified weekly window.
    day -> (struct)
      Day of the week when maintenance can occur.
    hour -> (integer)
      Hour of the day in UTC when the maintenance window starts.
``` ||
|| `--name` | `string`

New name of the Metastore Cluster. ||
|| `--network-spec` | `shorthand/json`

Network-related configuration options.

Shorthand Syntax:

```hcl
{
  security-group-ids = string,...
}
```

JSON Syntax:

```json
{
  "security-group-ids": [
    "string", ...
  ]
}
```

Fields:

```
security-group-ids -> ([]string)
  User security groups.
``` ||
|| `--service-account-id` | `string`

Service account used to access Cloud resources. ||
|| `--version` | `string`

Metastore server version. ||
|| `--async` | Display information about the operation in progress, without waiting for the operation to complete. ||
|#

#### Global Flags

#|
||Flag | Description ||
|| `--profile` | `string`

Set the custom profile. ||
|| `--region` | `string`

Set the region. ||
|| `--debug` | Debug logging. ||
|| `--debug-grpc` | Debug gRPC logging. Very verbose, used for debugging connection problems. ||
|| `--no-user-output` | Disable printing user intended output to stderr. ||
|| `--pager` | `string`

Set the custom pager. ||
|| `--format` | `string`

Set the output format: text, yaml, json, table, summary \|\| summary[name, instance.id, instance.disks[0].size]. ||
|| `--retry` | `int`

Enable gRPC retries. By default, retries are enabled with maximum 5 attempts.
Pass 0 to disable retries. Pass any negative value for infinite retries.
Even infinite retries are capped with 2 minutes timeout. ||
|| `--timeout` | `string`

Set the timeout. ||
|| `--token` | `string`

Set the IAM token to use. ||
|| `--impersonate-service-account-id` | `string`

Set the ID of the service account to impersonate. ||
|| `--no-browser` | Disable opening browser for authentication. ||
|| `--query` | `string`

Query to select values from the response using jq syntax ||
|| `-h`, `--help` | Display help for the command. ||
