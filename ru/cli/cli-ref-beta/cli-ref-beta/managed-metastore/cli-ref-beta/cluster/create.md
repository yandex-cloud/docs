# yc beta managed-metastore cluster create

Creates a Metastore Cluster.

#### Command Usage

Syntax: 

`yc beta managed-metastore cluster create <FOLDER-ID>`

#### Flags

| Flag | Description |
|----|----|
|`-r`,`--request-file`|<b>`string`</b><br/>Path to a request file.|
|`--example-json`|Generates a JSON template of the request.<br/><br/>The template can be customized and used as input for the command.<br/><br/>Usage example:<br/>1. Generate template: yc beta compute instance create --example-json > request.json<br/>2. Edit the template: vim request.json<br/>3. Run with template: yc beta compute instance create -r request.json|
|`--example-yaml`|Generates a YAML template of the request.<br/><br/>The template can be customized and used as input for the command.<br/><br/>Usage example:<br/>1. Generate template: yc beta compute instance create --example-yaml > request.yaml<br/>2. Edit the template: vim request.yaml<br/>3. Run with template: yc beta compute instance create -r request.yaml|
|`--config-spec`|<b>`shorthand/json`</b><br/>Configuration of the Metastore Cluster.<br/><br/>Example:<br/>--config-spec resources={resource-preset-id=value}<br/><br/>Shorthand Syntax:<br/>{resources={resource-preset-id=str}}<br/>Fields:<br/>resources  struct  — Configuration for computational resources for Metastore server instances.<br/>resource-preset-id  string  required  — ID of the preset for computational resources allocated to an instance (e.g., CPU, memory, etc.).<br/>|
|`--deletion-protection`|Deletion Protection prevents deletion of the cluster.|
|`--description`|<b>`string`</b><br/>Description of the Metastore Cluster.|
|`--folder-id`|<b>`string`</b><br/>ID of the folder to create the Metastore Cluster in.|
|`--labels`|<b>`stringToString`</b><br/>Custom labels for the Metastore Cluster as '' key:value '' pairs.<br/>For example: {"env": "prod"}.|
|`--logging`|<b>`shorthand/json`</b><br/>Cloud logging configuration.<br/><br/>Example:|
|`--logging`|<b>`destination={folder-id=value},`</b><br/>enabled=true, min-level=TRACE<br/><br/>Shorthand Syntax:<br/>{destination={folder-id=str \| log-group-id=str}, enabled=bool, min-level=TRACE\|DEBUG\|INFO\|WARN\|ERROR\|FATAL}<br/>Fields:<br/>enabled      bool                                     — Logs generated by the Metastore server are delivered to Cloud Logging.<br/>min-level    enum<DEBUG\|ERROR\|FATAL\|INFO\|TRACE\|WARN>  — Minimum severity level for log entries.<br/><br/>See [LogLevel.Level] for details.<br/>destination  oneof<folder-id\|log-group-id>            — Oneof destination field<br/>folder-id     string  — Logs will be written to the default log group of the specified folder.<br/>log-group-id  string  — Logs will be written to the log group specified by its ID.<br/>|
|`--maintenance-window`|<b>`shorthand/json`</b><br/>Maintenance window.<br/><br/>Example:<br/>--maintenance-window policy={anytime={}}<br/><br/>Shorthand Syntax:<br/>{policy={anytime={} \| weekly-maintenance-window={day=MON\|TUE\|WED\|THU\|FRI\|SAT\|SUN, hour=int}}}<br/>Fields:<br/>policy  oneof<anytime\|weekly-maintenance-window>  — Oneof policy field<br/>anytime                    struct  — The cluster may be restarted for maintenance at any time.<br/>weekly-maintenance-window  struct  — Maintenance is allowed only within the specified weekly window.<br/>day   enum<FRI\|MON\|SAT\|SUN\|THU\|TUE\|WED>  — Day of the week when maintenance can occur.<br/>hour  int                                — Hour of the day in UTC when the maintenance window starts.<br/>|
|`--name`|<b>`string`</b><br/>Name of the Metastore Cluster.<br/>The name must be unique within the folder.|
|`--network`|<b>`shorthand/json`</b><br/>Network-related configuration options.<br/><br/>Example:|
|`--network`|<b>`security-group-ids=value,`</b><br/>subnet-ids=value<br/><br/>Shorthand Syntax:<br/>{security-group-ids=str,..., subnet-ids=str,...}<br/>Fields:<br/>security-group-ids  []string  — User security groups.<br/>subnet-ids          []string  — IDs of VPC network subnets where instances of the cluster are attached.<br/>|
|`--service-account-id`|<b>`string`</b><br/>Service account used to access Cloud resources.|
|`--version`|<b>`string`</b><br/>Metastore server version.|
|`--async`|Display information about the operation in progress, without waiting for the operation to complete.|
