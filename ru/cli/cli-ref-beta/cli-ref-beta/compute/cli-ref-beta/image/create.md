# yc beta compute image create

Creates an image in the specified folder.

#### Command Usage

Syntax: 

`yc beta compute image create <FOLDER-ID>`

#### Flags

| Flag | Description |
|----|----|
|`-r`,`--request-file`|<b>`string`</b><br/>Path to a request file.|
|`--example-json`|Generates a JSON template of the request.<br/><br/>The template can be customized and used as input for the command.<br/><br/>Usage example:<br/>1. Generate template: yc beta compute instance create --example-json > request.json<br/>2. Edit the template: vim request.json<br/>3. Run with template: yc beta compute instance create -r request.json|
|`--example-yaml`|Generates a YAML template of the request.<br/><br/>The template can be customized and used as input for the command.<br/><br/>Usage example:<br/>1. Generate template: yc beta compute instance create --example-yaml > request.yaml<br/>2. Edit the template: vim request.yaml<br/>3. Run with template: yc beta compute instance create -r request.yaml|
|`--description`|<b>`string`</b><br/>Description of the image.|
|`--family`|<b>`string`</b><br/>The name of the image family to which this image belongs. For more information, see [Image family](https://yandex.cloud/ru/docs/compute/concepts/image#family).<br/><br/>To get an information about the most recent image from a family, use a [ImageService.GetLatestByFamily] request.|
|`--folder-id`|<b>`string`</b><br/>ID of the folder to create an image in.<br/>To get the folder ID, use a [yandex.cloud.resourcemanager.v1.FolderService.List] request.|
|`--hardware-generation`|<b>`shorthand/json`</b><br/>Specify the overrides to hardware_generation of a source disk, image or snapshot,<br/>or to the default values if the source does not define it.<br/><br/>Example:<br/>--hardware-generation features={generation2-features={}}<br/><br/>Shorthand Syntax:<br/>{features={generation2-features={} \| legacy-features={pci-topology=PCI_TOPOLOGY_V1\|PCI_TOPOLOGY_V2}}}<br/>Fields:<br/>features  oneof<generation2-features\|legacy-features>  — Oneof features field<br/>legacy-features       struct  —<br/>pci-topology  enum<PCI_TOPOLOGY_V1\|PCI_TOPOLOGY_V2>  —<br/>generation2-features  struct  —<br/>|
|`--labels`|<b>`stringToString`</b><br/>Resource labels as 'key:value' pairs.|
|`--min-disk-size`|<b>`int`</b><br/>Minimum size of the disk that will be created from this image.<br/>Specified in bytes. Should be more than the volume of source data.|
|`--name`|<b>`string`</b><br/>Name of the image.|
|`--os`|<b>`shorthand/json`</b><br/>Operating system that is contained in the image.<br/><br/>If not specified and you used the 'image_id' or 'disk_id' field to set the source, then the value can be inherited from the source resource.<br/><br/>Example:<br/>--os type=LINUX<br/><br/>Shorthand Syntax:<br/>{type=LINUX\|WINDOWS}<br/>Fields:<br/>type  enum<LINUX\|WINDOWS>  — Operating system type. The default is 'LINUX'.<br/><br/>This field is used to correctly emulate a vCPU and calculate the cost of using an instance.<br/>|
|`--pooled`|When true, an image pool will be created for fast creation disks from the image.|
|`--product-ids`|<b>`strings`</b><br/>License IDs that indicate which licenses are attached to this resource.<br/>License IDs are used to calculate additional charges for the use of the virtual machine.<br/><br/>The correct license ID is generated by the platform. IDs are inherited by new resources created from this resource.<br/><br/>If you know the license IDs, specify them when you create the image.<br/>For example, if you create a disk image using a third-party utility and load it into Object Storage, the license IDs will be lost.<br/>You can specify them in this request.|
|`--image-id`|<b>`string`</b><br/>ID of the source image to create the new image from.|
|`--disk-id`|<b>`string`</b><br/>ID of the disk to create the image from.|
|`--snapshot-id`|<b>`string`</b><br/>ID of the snapshot to create the image from.|
|`--uri`|<b>`string`</b><br/>URI of the source image to create the new image from.<br/>Currently only supports links to images that are stored in Object Storage.<br/>Must be a valid [pre-signed URL](https://yandex.cloud/ru/docs/storage/concepts/pre-signed-urls).<br/>Currently only supports Qcow2, VMDK, and RAW image formats.|
|`--async`|Display information about the operation in progress, without waiting for the operation to complete.|
