# yc beta organization-manager idp user create

Creates a user in the specified userpool.

#### Command Usage

Syntax: 

`yc beta organization-manager idp user create <EXTERNAL-ID>`

#### Flags

| Flag | Description |
|----|----|
|`-r`,`--request-file`|<b>`string`</b><br/>Path to a request file.|
|`--example-json`|Generates a JSON template of the request.<br/>The template can be customized and used as input for the command.<br/>Usage example:<br/><br/>1. Generate template: yc beta compute instance create --example-json > request.json<br/>2. Edit the template: vim request.json<br/>3. Run with template: yc beta compute instance create -r request.json|
|`--example-yaml`|Generates a YAML template of the request.<br/>The template can be customized and used as input for the command.<br/>Usage example:<br/><br/>1. Generate template: yc beta compute instance create --example-yaml > request.yaml<br/>2. Edit the template: vim request.yaml<br/>3. Run with template: yc beta compute instance create -r request.yaml|
|`--email`|<b>`string`</b><br/>Email address of the user.|
|`--external-id`|<b>`string`</b><br/>External identifier for the user.|
|`--family-name`|<b>`string`</b><br/>Last name of the user.|
|`--full-name`|<b>`string`</b><br/>Full name of the user.|
|`--given-name`|<b>`string`</b><br/>First name of the user.|
|`--is-active`|Whether the user is active. Default is true.|
|`--phone-number`|<b>`string`</b><br/>Phone number of the user.|
|`--username`|<b>`string`</b><br/>Username for the new user.|
|`--userpool-id`|<b>`string`</b><br/>ID of the userpool to create the user in.|
|`--password-spec`|<b>`shorthand/json`</b><br/>Password specification. Credentials type. Exactly one of credentials type must be specified.<br/>Shorthand Syntax:<br/>{<br/>generation-proof = str,<br/>password = str<br/>}<br/>JSON Syntax:<br/>"{<br/>"generation-proof": "str",<br/>"password": "str"<br/>}"<br/>Fields:<br/>generation-proof -> (string)<br/>Proof that the password was generated by the system.<br/>password -> (string)<br/>The password string.|
|`--password-hash`|<b>`shorthand/json`</b><br/>Password hash. Credentials type. Exactly one of credentials type must be specified.<br/>Shorthand Syntax:<br/>{<br/>password-hash = str,<br/>password-hash-type = AD_MD4<br/>}<br/>JSON Syntax:<br/>"{<br/>"password-hash": "str",<br/>"password-hash-type": "AD_MD4"<br/>}"<br/>Fields:<br/>password-hash -> (string)<br/>The password hash string.<br/>password-hash-type -> (enum<AD_MD4>)<br/>Type of the password hash.|
|`--async`|Display information about the operation in progress, without waiting for the operation to complete.|
