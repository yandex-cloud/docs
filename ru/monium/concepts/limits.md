# Квоты и лимиты в {{ monium-name }}

В {{ monium-name }} действуют ограничения (квоты и лимиты) на ряд ресурсов.

{% include [quotes-limits-def.md](../../_includes/quotes-limits-def.md) %}

## Посмотреть действующие квоты {#view-quotas}

{% list tabs group=instructions %}

- Интерфейс {{ monium-name }} {#console}
  
  1. На главной странице [{{ monium-name }}]({{ link-monium }}) слева выберите **Квоты**.
  1. Слева вверху выберите нужный проект или оставьте текущий.
  1. На вкладке **Квоты проекта** посмотрите значения установленных квот по группам ресурсов.
  1. Чтобы посмотреть квоты по отдельным шардам, на странице **Квоты** перейдите на вкладку **Распределение по шардам**.
  1. Для каждого шарда приведены значения:
     * **Используется** — текущее потребление ресурса в шарде.
     * **Лимит** — максимальное значение квоты для шарда.
     * **Утилизация** — доля использованной квоты (отношение потребления к лимиту). Показывает, насколько близко текущее потребление к квоте.

{% endlist %}

## Изменить квоты {#change-quotas}

{% list tabs group=instructions %}

- Интерфейс {{ monium-name }} {#console}
  
  1. Убедитесь, что ваш аккаунт имеет роль [monium.admin](../security/index.md#monium-admin) или [monium.editor](../security/index.md#monium-editor).
  1. На главной странице [{{ monium-name }}]({{ link-monium }}) слева выберите **Квоты**.
  1. Слева вверху выберите нужный проект или оставьте текущий.
  1. На вкладке **Квоты проекта** в нужной группе квот нажмите ![image](../../_assets/console-icons/pencil.svg) **Изменить объём квот**.
  1. В открывшемся окне введите новые значения для квот этой группы.
  1. Нажмите **Отправить**.

{% endlist %}

Квоты можно увеличивать и уменьшать.

Увеличение квоты подтверждается автоматически, но только до определенного предела. Если требуется большее значение, система предложит обратиться в [поддержку]({{ link-console-support }}) для дальнейшего согласования.

Уменьшение квоты помогает ограничить неожиданный рост потребления. Например, после релиза приложение начало записывает слишком много логов. В этом случае сначала можно снизить квоту, а затем выяснить причину, откатить релиз или внести исправление. Это поможет контролировать расходы.

## Квотируемые ресурсы {#quota-resources}

* **Метрики**:
  * `Metrics Write` — скорость записи точек в метрики шарда, в точках в секунду. При исчерпании квоты сервис начнет случайным образом отбрасывать точки для сохраняемых метрик. На графиках могут появиться пропуски.
  * `Stored Metrics` — количество уникальных метрик (временных рядов). При исчерпании квоты новые метрики не будут сохраняться. Запись данных в уже существующие метрики продолжится.
* **Алерты**:
  * `Alerts Count` — количество алертов и подалертов в проекте. При исчерпании квоты вы не сможете создать новые алерты и подалерты.
* **Логи**:
  * `Logs Write` — поток записываемых логов, [мебибайт](https://ru.wikipedia.org/wiki/Мебибайт) в секунду. При исчерпании квоты сервис начнет случайным образом отбрасывать входящие логи.
* **Трейсы**:
  * `Spans Write` — поток записываемых спанов, мебибайт в секунду. При исчерпании квоты сервис начнет случайным образом отбрасывать входящие спаны. Из-за этого часть трейсов может собираться не полностью.
