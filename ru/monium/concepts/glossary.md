# Основные понятия {{ monium-name }}

В разделе описаны понятия, которые используются в сервисе и документации {{ monium-name }}.

## Общие термины {{ monium-name }} {#common-terms}

### Быстрые ссылки {#project-menu}

_Быстрые ссылки_ — набор ссылок в интерфейсе {{ monium-name }}, который можно настроить для каждого [проекта](#project) через интерфейс или API.

### Временной ряд (Timeseries) {#timeseries}

_Временной ряд_ (Timeseries) — упорядоченное по времени множество значений какой-либо величины с указанием времени каждого измерения: `[(временная_метка_1, значение_1), (временная_метка_2, значение_2), ...]`. Подробнее в разделе [{#T}](data-model.md).

### График {#graph}

1. Визуализация набора [метрик](#metric).
1. Элемент [дашборда](#dashboard).

### Дашборд (Dashboard) {#dashboard}

_Дашборд (Dashboard)_ — объект конфигурации {{ monium-name }}, содержащий набор [графиков](#graph) и настройки их отображения.

### Кластер (Cluster) {#cluster}

_Кластер (Cluster)_ — объект конфигурации {{ monium-name }}, определяющий множество хостов, которые {{ monium-name }} будет опрашивать для сбора метрик при использовании pull-режима передачи метрик. Как правило, кластер соответствует окружению (инсталляции) клиентского приложения, которое передает метрики. Например, боевой и тестовый кластеры, кластеры для разных регионов.

### Метка (Label) {#label}

_Метка (Label)_ — пара строк в формате `<имя>="<значение>"`, например, `host="monitoring-kfront-man-00"` или `code="200"`. Метки описывают различные срезы, по которым необходимо визуализировать метрики, передаваемые в {{ monium-name }}. Набор меток идентифицирует [метрику](#metric). Подробнее в разделе [{#T}](data-model.md).

### Метрика {#metric}

_Метрика (Metric)_ — [временной ряд](#timeseries) и идентифицирующий его набор [меток](#label), записанный в фигурных скобках.

### Параметр дашборда {#dashboard-param}

_Параметры дашборда_ позволяют создавать интерактивные дашборды, содержание которых изменяется в зависимости от выбора пользователя. Например, дашборд, показывающий сводную информацию о состоянии виртуальной машины, можно параметризовать идентификатором виртуальной машины.

### Проект (Project) {#project}

_Проект (Project)_ — верхнеуровневый объект конфигурации {{ monium-name }} с данными наблюдения. Используется для логического объединения нескольких связанных сервисов и назначения прав доступа к телеметрии командам проекта. Проект может содержать данные по ресурсам отдельного [каталога](../../resource-manager/concepts/resources-hierarchy.md#folder), тогда идентификатор проекта формируется как `folder__<идентификатор_каталога>`. Проект позволяет ограничить доступ к данным с помощью назначения [ролей](../../iam/concepts/access-control/roles.md).

### Прореживание {#downsampling}

1. _Прореживание данных при чтении_ — механизм, ограничивающий количество отображаемых точек на [графике](#graph). Подробнее в разделе [{#T}](decimation.md#reading-decimation).

1. _Прореживание исторических данных_ — механизм архивирования исторических данных в {{ monium-name }}. Для уменьшения объема хранимых исторических данных значения всех метрик, записываемых в {{ monium-name }}, спустя неделю агрегируются по сетке в 5 минут. Подробнее в разделе [{#T}](decimation.md#history-decimation).

### Сервис (Service) {#service}

_Сервис (Service)_ — объект конфигурации {{ monium-name }}, определяет способ передачи данных в {{ monium-name }} — Pull или Push. Как правило, сервис соответствует клиентскому приложению, которое передает метрики. Например, веб-сервер, база данных или другой компонент.

### Шард (Shard) {#shard}

_Шард_ — объект конфигурации {{ monium-name }}, которому принадлежат метрики из конкретной пары [сервиса](#service) и [кластера](#cluster), а также настройки хранения этих метрик, например [TTL](#ttl).

### Язык запросов (Expression Language) {#query-language}

_Язык запросов_ (Expression Language или просто Expression) — способ описать вычисления и преобразования метрик при настройке графиков и алертов, а также для выгрузки данных через API. Подробнее в разделе [{#T}](querying.md).

### TTL (Time-to-Live) {#ttl}

_TTL_ — механизм автоматического удаления устаревших данных, для которых не поступали новые значения в течение определенного времени.

Например, TTL метрик 7 дней означает, что метрики будут автоматически удаляться, если временная метка их последней точки старше 7 дней.

Подробнее в разделе [{#T}](#ttl).

### Unified Agent {#unified-agent}

_Unified Agent_ — программа для сбора, хранения, преобразования и доставки метрик и логов в {{ monium-name }}.
