# Начало работы с {{ cloud-logging-name }}

{{ cloud-logging-name }} — распределенная и высокодоступная система для хранения, поиска, визуализации и анализа логов. Система входит в платформу {{ monium-name }} и использует ее базовые компоненты — управление доступом, модель данных и единый интерфейс.

## Поставка логов {#data-send}

Поставка логов в {{ monium-name }} настраивается так же, как и другие виды телеметрии. Если вы еще не работали с облаком {{ yandex-cloud }} или у вас нет приложения, из которого можно проверить отправку логов, начните с общего раздела [{#T}](../quickstart.md) — там описана базовая настройка окружения и поставка данных с помощью Otel Collector.

Также можно настроить передачу логов через [Fluent Bit](write/fluent-bit.md) или напрямую из приложения.

Если у вас есть облако {{ yandex-cloud }}, свое приложение и вы работали с поставкой телеметрии ранее, настройте отправку логов по параметрам ниже.

### Параметры {{ cloud-logging-name }} {#main-parameters}

* {% include [otel-only](../../_includes/monium/otel-only.md) %}

* Адрес для записи логов: `{{ api-host-monium }}:443`.

* Для записи логов потребуется [сервисный аккаунт](../../iam/operations/sa/create.md) с ролью `monium.logs.writer` и [API-ключ](../../iam/operations/iam-token/create-for-sa.md) с областью действия `yc.monium.logs.write`.


## Просмотр логов {#view-logs}

1. На главной странице сервиса [{{ monium-name }}]({{ link-monium }}) слева выберите **{{ ui-key.yacloud_monitoring.aside-navigation.menu-item.logs.title }}**.
1. Вверху укажите промежуток поиска с помощью временной шкалы, предустановленного промежутка или введите нужное значение времени.
1. В строке поиска введите запрос для поиска логов приложения:
   1. Выберите нужный проект `folder__<идентификатор_каталога>`. По умолчанию выбран проект в текущем каталоге.
   1. Введите запрос для поиска логов приложения.
   1. Нажмите **Выполнить запрос**.
1. Чтобы посмотреть логи для определенной записи, раскройте ее.

## Сервисный дашборд для логов {#logs-service-dashboard}

Система автоматически собирает [метрики о поступающих логах](logs-metrics.md) — объем данных, задержки доставки, количество ошибок по уровням severity.

Графики о поступивших логах доступны на сервисном дашборде.
