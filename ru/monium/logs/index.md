# Обзор {{ monium-logs-name }}

{{ monium-logs-name }} — распределенная и высокодоступная система для хранения, поиска, визуализации и анализа логов.

Логи помогают анализировать работу приложения и инфраструктуры: искать ошибки и исключения, разбирать инциденты, отслеживать изменения и выявлять причины деградации.

{{ monium-logs-name }} входит в платформу {{ monium-name }} и использует ее базовые компоненты — проектную модель, управление доступом, модель данных и общий интерфейс.

Благодаря единой платформе логи можно использовать совместно с [трейсами](../traces/index.md). Для этого используются идентификаторы `trace_id` и `span_id`, которые позволяют быстро перейти от записи лога по конкретному запросу к трейсам и наоборот.

{{ monium-name }} принимает данные только в формате OpenTelemetry. Для отправки логов используйте OpenTelemetry SDK в приложении или любой агент, который поддерживает OTel, например, [Otel Collector](../collector/opentelemetry.md) или [Fluent Bit](write/fluent-bit.md).

Преимущества {{ monium-logs-name }}:

* Быстрый поиск логов, который позволяет диагностировать ошибки в реальном времени ([ad-hoc сценарии](https://en.wikipedia.org/wiki/Ad_hoc_testing)).
* Отказоустойчивость: данные реплицируются между центрами обработки данных.
* Самодиагностика: система автоматически собирает метрики о поступающих логах — объем данных, задержки доставки, количество ошибок и так далее. Для диагностики есть сервисный дашборд с предустановленными графиками для различных метрик.
