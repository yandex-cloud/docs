# Мьюты 

Мьюты позволяют временно отключить уведомления от определенного алерта или группы алертов. Мьюты не влияют на процесс вычисления алерта, актуальный статус будет отображаться в интерфейсе.

Мьюты действуют в течение заданного периода, затем отключаются и через 7 дней удаляются по [TTL](../concepts/glossary.md#ttl).

## Параметры мьютов {#mute-parameters}

* **Период действия мьюта** — дата и время начала и окончания действия мьюта.
* **Название** — название мьюта.
* **Описание** — описание причин, по которым был выставлен мьют.
* **Селектор** — селектор для поиска алертов, на которые действует мьют.

Доступные метки для поиска алертов:

* `alert.id` и `sub_alert.id` — ID алерта;
* `alert.name` — название алерта;
* `alert.labels.LABEL_NAME` — лейблы алерта;
* `sub_alert.labels.LABEL_NAME` — метки разложения мультиалерта.

В селекторе поддерживаются [glob-выражения](https://ru.wikipedia.org/wiki/Шаблон_поиска), например `alert.id='*prod-???'`. Можно указывать несколько меток для более точного поиска, например `alert.id='*-prod', alert.labels.serviceProviderId=compute`.

{% note info %}

Подсказка в поле ввода подскажет вам доступные в проекте лейблы алертов и метки разложения мультиалерта.

{% endnote %}

При создании или редактировании мьюта в разделе **Алерты, которые будут замьючены** можно посмотреть, какие алерты подходят под селекторы мьюта.

## Примеры селекторов мьютов {#example}

* Мьют по ID алерта:
    ```
    alert.id='test_alert'
    ```
* Мьют всех алертов по шаблону ID:
    ```
    alert.id='test_alert*'
    ```
* Мьют всех алертов по шаблону названия:
    ```
    alert.name='*query*|*product*'
    ```
* Мьют подалерта по ID:
    ```
    sub_alert.id='500f279065221ba31a4feb9319b24eb2bd42c4ac'
    ```

## Состояния мьютов {#mute-states-regular}

В зависимости от своих настроек и текущего времени мьют может переходить между следующими состояниями:

* **Pending** — мьют запланирован, но пока не действует на алерты.
* **Active** — мьют активен и отключает все сообщения алертов и подалертов, которые подпадают под селекторы мьюта.
* **Expired** — мьют закончился и больше не действует на алерты.
* **Archived** — мьют закончился более суток назад.

Мьюты в статусах **Pending**, **Active** и **Expired** отображаются в специальном блоке на страницах состояния связанных алертов и подалертов. Мьюты в состоянии **Archived** не считаются связанными с алертами, даже если идентификатор алерта подпадает под селектор мьюта. Эти мьюты хранятся для истории и удаляются через механизм TTL спустя 6 дней после перехода в статус **Archived**.

{% note alert %}

Через сутки после времени окончания мьюта он переходит в статус **Archived**. Через 6 дней после перехода в этот статус мьют удаляется.

{% endnote %}

## Взаимодействие мьютов с эскалациями {#mutes-for-escalations}

Если на алерт {{ monium-name }} был настроен мьют, то эскалация не будет запущена.
