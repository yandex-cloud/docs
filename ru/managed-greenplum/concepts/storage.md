# Типы хранилища

В {{ mgp-name }} для кластеров БД используются сетевое и локальное хранилища. Сетевое хранилище использует сетевые блоки — виртуальные диски в инфраструктуре {{ yandex-cloud }}, а локальное — физические диски на серверах хостов БД.

В кластере {{ mgp-name }} тип хранилища у хостов-мастеров и хостов-сегментов может отличаться.

## Особенности локального хранилища {#local-storage-features}

* Одиночный хост-мастер с локальным хранилищем не отказоустойчив: при отказе локального диска данные теряются безвозвратно. Поэтому при использовании локального хранилища для хостов-мастеров можно выбрать только конфигурацию с двумя хостами-мастерами.

* На время, когда кластер {{ mgp-name }} остановлен, тарификация вычислительных ресурсов (vCPU и RAM):

    * для хостов, которые используют локальное хранилище, не останавливается;
    * для хостов, которые не имеют локального хранилища, останавливается.

## Особенности нереплицируемого сетевого хранилища {#network-nrd-storage-features}

Одиночный хост-мастер с нереплицируемым сетевым хранилищем не отказоустойчив: при отказе диска данные теряются безвозвратно. Поэтому при использовании нереплицируемого сетевого хранилища для хостов-мастеров можно выбрать только конфигурацию с двумя хостами-мастерами.
