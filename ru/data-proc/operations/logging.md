# Работа с логами

Вы можете следить за событиями кластера {{ dataproc-name }} в консоли управления. Чтобы воспользоваться этой возможностью, укажите в настройках кластера [лог-группу](../../logging/concepts/log-group.md). Это можно сделать при [создании кластера](./cluster-create.md) или при [изменении его настроек](./cluster-update.md).

Собирает и отображает логи кластера сервис [{{ cloud-logging-name }}](../../logging/).

## Просмотр записей лога кластера {#logging-cluster}

{% list tabs %}

* Консоль управления

    1. Перейдите на страницу каталога и выберите сервис **{{ dataproc-name }}**.
    1. Нажмите на имя нужного кластера и выберите вкладку **Логи**.
    1. (Опционально) Укажите настройки вывода:

        * [фильтр сообщений](../../logging/concepts/filter.md);
        * интересующие уровни сообщений: от `TRACE` до `FATAL`;
        * количество сообщений на странице;
        * интервал сообщений, один из стандартных или произвольный.

{% endlist %}

Подробнее о работе с логами см. в [документации {{ cloud-logging-full-name }}](../../logging/operations/index.md).
