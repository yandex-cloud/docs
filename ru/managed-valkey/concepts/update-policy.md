# Политика поддержки версий {{ VLK }}

_Политика поддержки версий {{ VLK }}_ направлена на предоставление пользователям расширенной по времени поддержки версий [{{ VLK }}](./index.md), позволяющей дольше пользоваться проверенными версиями СУБД.

Политика поддержки версий {{ mrd-full-name }} основана на [гарантии совместимости](./index.md#compatibility-warranty), позволяющей вместо определенной версии {{ VLK }} выбирать версию, _совместимую с_ этой конкретной версией. Политика поддержки версий применяется ко всем новым [кластерам](../../glossary/cluster.md), создаваемым в сервисе {{ mrd-name }}.

{% note info %}

До конца 2026 года все кластеры, созданные до введения настоящей политики поддержки версий, будут переведены на ее использование и обновлены с помощью заданий на [техническое обслуживание](./maintenance.md):

* кластеры с версией `7.2` — до версии, [совместимой с {{ VLK }} 7.2](#compatibility-table);
* кластеры с версией `8.0` — до версии, [совместимой с {{ VLK }} 8.0](#compatibility-table);
* кластеры с версией `8.1` — до версии, [совместимой с {{ VLK }} 8.1](#compatibility-table);
* кластеры с версией `9.0` — до версии, [совместимой с {{ VLK }} 9.0](#compatibility-table).

Переход будет автоматически выполняться в [окнах](./maintenance.md#maintenance-window) технического обслуживания с помощью запланированных заданий **с предварительным уведомлением пользователя за 90 дней**.

{% endnote %}

## Этапы жизненного цикла совместимых версий {#lifecycle-stages}

Для каждой версии {{ VLK }}, с которой [совместима](./index.md#compatibility-warranty) версия в реализации {{ yandex-cloud }}, сервис обеспечивает поддержку продолжительностью не менее, чем продолжительность поддержки последней совместимой версии в официальном жизненном цикле {{ VLK }}.

Состояние версии | Описание и ключевые [действия](#actions-available) | Срок
--- | --- | ---
Полная поддержка (`Supported`) | Версия полностью поддерживается. **Рекомендуется для новых проектов.** | До окончания поддержки последней совместимой версии, но не менее чем длительность официальной поддержки {{ VLK }} для данной версии
Устаревающая (`Deprecated`) | Поддержка версии приближается к завершению. **Создание новых кластеров невозможно. Существующие кластеры работают в штатном режиме.** | Шесть месяцев
Устаревшая (`Legacy`) | Поддержка версии официально прекращена. **Кластеры работают, но подлежат тарификации по повышенной ставке. Техническая поддержка ограничена.** | Шесть месяцев
Выведенная из эксплуатации (`EOL`) | Версия более не поддерживается. **Техническая поддержка не предоставляется. Кластеры работают, но подлежат тарификации по повышенной ставке.** | Не применимо

## Ключевые принципы политики поддержки версий {{ VLK }} {#key-principles}

Политика поддержки версий {{ VLK }} основывается на следующих ключевых принципах:

* **Расширенная поддержка версий**. {{ mrd-name }} предоставляет расширенную по времени поддержку версий {{ VLK }}, при которой каждая версия поддерживается дольше официального срока благодаря совместимости с новыми версиями {{ VLK }}.
* **Гарантия совместимости**. Сервис предоставляет услугу с гарантией полной обратной совместимости с выбранной версией {{ VLK }}.
* **Проверенная совместимость**. Каждая реализация СУБД от {{ yandex-cloud }} проходит набор [тестов](#compatibility-testing) совместимости с целевой версией. Обновления применяются только при полном и успешном прохождении тестов.
* **Активное участие в развитии {{ VLK }}**. Вовлеченность {{ yandex-cloud }} в развитие проекта {{ VLK }} позволяет команде сервиса {{ mrd-name }} глубоко понимать платформу и обеспечивать совместимость на базе более новых и эффективных реализаций.
* **Автоматические улучшения**. Пользователи автоматически получают преимущества новых версий без необходимости миграции.
* **Безопасность и производительность**. Постоянное обновление до актуальных версий обеспечивает максимальную безопасность и производительность.

## Политика обновлений {#update-policy}

Сервис {{ mrd-full-name }} может автоматически обновлять реализацию {{ VLK }} от {{ yandex-cloud }}, работающую на вашем кластере, не изменяя целевую версию совместимости. Такие обновления включают:

* обновление до более новых версий с сохранением [гарантии совместимости](#compatibility-warranty);
* специальные настройки конфигурации для обеспечения обратной совместимости;
* режим совместимости с адаптированными параметрами производительности;
* дополнительные [проверки](#compatibility-testing) на стороне {{ mrd-name }} для мониторинга поведения приложений.

{% note tip %}

При обновлении реализации СУБД целевая версия совместимости кластера не изменится, а ваше приложение продолжит работать без необходимости внесения изменений в его работу. Если потребуется, вы можете [обратиться]({{ link-console-support }}) в службу технической поддержки, чтобы уточнить подробности об обновлении.

{% endnote %}

### Прекращение обновлений {#update-termination}

Обновления реализации СУБД прекращаются после того, как становится невозможным поддерживать совместимость с более новыми версиями {{ VLK }}. В этом случае поддержка версии длится до завершения поддержки последней оригинальной совместимой версии {{ VLK }}.

## Уведомления {#notifications}

Сервис {{ mrd-full-name }} заблаговременно уведомит вас о предстоящих изменениях, связанных с переходом текущей версии {{ VLK }} кластера в другое [состояние](#lifecycle-stages):

* при переходе версии {{ VLK }} кластера в статус **Устаревающая** (начало действия ограничений) сервис за 90 дней уведомит о невозможности создания новых кластеров с этой версией и предложит запланировать обновление существующих кластеров до версии {{ VLK }}, совместимой с более новой версией, имеющей полную поддержку.
* при переходе версии {{ VLK }} кластера в статус **Устаревшая** (начало действия повышенного тарифа) сервис за 90 дней уведомит о приближающемся сроке завершения поддержки версии и о начале применения к таким кластерам повышенной ставки тарификации.

Уведомления о таких изменениях отправляются пользователям в соответствии с заданными [настройками уведомлений](../../console/operations/update.md#notifications).
