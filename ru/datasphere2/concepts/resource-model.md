# Взаимосвязь ресурсов в {{ ml-platform-name }}

_Ресурсы {{ ml-platform-name }}_ — объекты, которые создаются или используются в [проектах](project.md), такие как датасеты, Docker-образы, ноды и другие.

_Проекты_ — основное рабочее место пользователя в {{ ml-platform-name }}. В проектах хранятся [состояние интерпретатора](save-state.md), переменные, установленное ПО и прочая информация.

_Сообщества_ — способ организации групповой работы. [Сообщества](community.md) определяют область видимости проектов и ресурсов {{ ml-platform-name }}.

## Ресурсы {{ ml-platform-name }} {#resources}

В проектах {{ ml-platform-name }} можно использовать следующие типы ресурсов:
* [Датасеты](dataset.md) — способ хранения информации, который предоставляет быстрый доступ к большим объемам данных внутри проекта.
* [Секреты](secrets.md) — пары ключ-значение, в которых приватные данные (токены, ключи и прочее) хранятся в зашифрованном виде. Секреты создаются в проекте и закрепляются за ним. Созданные секреты можно использовать в коде ячейки как переменные окружения.
* Docker-образы — окружение операционной системы, в котором собран произвольный набор ПО, библиотек, переменных окружения и конфигурационных файлов. Подробнее см. [{#T}](../operations/user-images.md).
* Подключения к хранилищам S3 — бакеты {{ objstorage-name }} целиком или частично монтируются в файловую систему проекта для облегчения доступа к данным из кода. Подробнее см. [{#T}](../operations/data/connect-to-s3.md).
* Контрольные точки — [сохраненное состояние](save-state.md) ноутбука: код ячеек, вывод и значения переменных, а также данные хранилища проекта. Контрольные точки версионируются.
* Ноды и алиасы — развернутые сервисы, которые обеспечивают [эксплуатацию обученных моделей](deploy/) в онлайн-режиме.

## Совместное использование проектов и ресурсов {#sharing}

Для совместного использования проектов и ресурсов {{ ml-platform-name }} предусмотрена возможность публикации ресурсов в сообществах.

Публикация ресурса в сообществе означает, что все пользователи сообщества получат доступ к ресурсу. Это позволит им использовать ресурс в своих проектах в рамках сообщества.

Обмен ресурсами между сообществами позволяет использовать Docker-образы, датасеты, модели и прочие объекты разными командами внутри одной организации.

Видимость сообществ, проектов и ресурсов {{ ml-platform-name }} ограничена рамками [организации](../../organization/). Обмен между организациями невозможен.

Публиковать ресурсы {{ ml-platform-name }} могут пользователи с ролью `{{ roles-datasphere-admin }}`. Делиться ресурсами можно в любом сообществе, в котором пользователь является участником. Подробнее см. [{#T}](../security/index.md).

## Связь ресурсов {{ ml-platform-name }} с сервисами {{ yandex-cloud }} {#ml-cloud-connection}

Для оплаты сервиса {{ ml-platform-name }} используется [платежный аккаунт](../../billing/concepts/billing-account.md) {{ yandex-cloud }}.

Сообщества {{ ml-platform-name }} являются ресурсами организации. В одной организации может быть множество сообществ.

Для доступа к другим сервисам {{ yandex-cloud }} используются [каталоги](../../resource-manager/concepts/resources-hierarchy.md#folder). В них размещены ресурсы конкретного сервиса {{ yandex-cloud }}.
