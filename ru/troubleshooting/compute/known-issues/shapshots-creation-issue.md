# Устранение проблем с созданием снимков дисков


## Описание проблемы {#issue-description}

При работе со снимками виртуальных машин в Compute Cloud наблюдается одна или несколько указанных ниже проблем:

* Не создаются снимки дисков по расписанию;
* Не удается создать новый снимок диска;
* Не удается создать новое расписание снимков дисков; 
* При попытке создания снимка диска отображается сообщение об ошибке `Limit compute.snapshots.count exceeded` или `Limit compute.snapshots.size exceeded`;
* При попытке создания нового расписания отображается сообщение об ошибке `Limit compute.snapshotSchedules.count exceeded`.

## Решение {#case-resolution}

Проверьте текущие значния квот `compute.snapshots.count`, `compute.snapshots.size` и `compute.snapshotSchedules.count`.
Операции создания новых снимков будут завершаться с ошибкой при превышении связанных с ними квот.
Вы можете [создать запрос на увеличение квот](https://console.cloud.yandex.ru/cloud?section=quotas) из Консоли управления;

{% note info %}

Для создания запросов на увеличение квот вашему IAM-аккаунту необходима роль `quota-manager.requestOperator`, назначенная на облако, в котором необходимо увеличить квоту.

{% endnote %}

Более подробно о квотах мы рассказываем в документации на странице [«Квоты и лимиты для сервисов Yandex Cloud»](../../../overview/concepts/quotas-limits) и в соответствующих разделах документации для каждого из сервисов.

## Если проблема осталась {#if-issue-still-persists}

Если вышеописанные действия не помогли решить проблему, [создайте запрос в техническую поддержку](https://console.cloud.yandex.ru/support?section=contact).
В запросе укажите следующую информацию:

1. Идентификатор ВМ, для которой не получается создать снимок.
2. Полный текст сообщения об ошибке;
3. [HAR-файл](../../../support/create-har.md) с сохраненными результатами взаимодействия браузера с серверами Yandex Cloud при попытке создания нового снимка.
