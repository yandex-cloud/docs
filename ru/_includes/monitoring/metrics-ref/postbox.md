Имя метрики пишется в метку `name`.

Общие метки для всех метрик {{ postbox-name }}:

Метка | Значение
----|----
identity | Идентификатор [адреса](../../../postbox/concepts/glossary.md#adress) отправителя.
service | Метка сервиса, значение всегда `postbox`.

## Метрики приема писем {#inbound-metrics}

| Имя метрики<br>Тип, единицы измерения | Описание |
| --- | --- |
| `inbound_emails_count`<br>`RATE`, штуки | Количество писем, принятых сервисом к отправке. Если у письма несколько получателей, метрика увеличится на их количество. Например, если одно письмо адресовано трем получателям, значение метрики увеличится на три. |
| `inbound_issues_rate`<br>`RATE`, штуки | Количество проблем, возникших при приеме письма сервисом. Метка `kind` — категория проблемы, возможные значения: `temporary quota issue`, `permanent quota issue`, `malformed email`, `smtp session error`, `client notification issue`. |

## Метрики доставки писем {#delivery-metrics}

| Имя метрики<br>Тип, единицы измерения | Описание |
| --- | --- |
| `process_duration_ms`<br>`HIST_RATE`, миллисекунды | Длительность полного цикла обработки письма — от момента принятия сервисом до завершения попытки доставки. |
| `emails_processed_count`<br>`RATE`, штуки | Количество писем, по которым сервис завершил цикл обработки (вне зависимости от результата доставки). |
| `delivery_success_count`<br>`RATE`, штуки | Количество писем, успешно доставленных получателю. Метрика увеличивается отдельно для каждого получателя. |
| `delivery_retry_count`<br>`RATE`, штуки | Количество повторных попыток доставки письма получателю. Повторные попытки происходят, когда почтовый сервер получателя временно недоступен или возвращает временную ошибку. |
| `delivery_spam_count`<br>`RATE`, штуки | Количество писем, помеченных почтовым сервером получателя как спам. |
| `delivery_reject_count`<br>`RATE`, штуки | Количество писем, которые почтовый сервис получателя окончательно отказался принять. |

## Метрики уведомлений о событиях {#notification-metrics}

Имя топика для уведомлений о событиях пишется в метку `topic`.

| Имя метрики<br>Тип, единицы измерения | Описание |
| --- | --- |
| `notifications_published_count`<br>`RATE`, штуки | Количество уведомлений о событиях, успешно опубликованных в топик. |
| `notifications_error_rate`<br>`RATE`, штуки | Количество ошибок при публикации уведомлений в топик. |
