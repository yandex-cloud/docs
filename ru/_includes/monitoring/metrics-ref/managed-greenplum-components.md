Общие метки для всех метрик сервиса {{ mgp-name }}:

| Метка | Значение |
| :--- | :--- |
| `service` | Идентификатор сервиса: `managed-greenplum` |
| `resource_type` | Тип ресурса: `cluster` |
| `resource_id` | Имя кластера |
| `cid` | Идентификатор кластера |
| `host` | FQDN хоста |
| `fqdn` | FQDN хоста |

Некоторые из описанных метрик специфичны для всего кластера, а не для конкретного хоста. В таких метриках метки `host` и `fqdn` содержат FQDN хоста, на котором они собираются. Чаще всего это мастер-хост координатора (в интерфейсе отображается как `MASTER`), и значение метки может измениться при переключении роли `MASTER` => `REPLICA`. Поэтому для общекластерных метрик лучше не использовать эти метки при построении запросов в {{ monitoring-name }}.

## Составные сигналы {#managed-greenplum-composite-signals}

Составные метрики, агрегирующие состояние нескольких индикаторов для быстрой оценки статуса кластера или отдельного компонента. Значения сигналов интерпретируются следующим образом:

| Числовое значение | Статус | Описание |
| :--- | :--- | :--- |
| `2` | **OK** (Зеленый) | Все важные индикаторы кластера или компонента находятся в пределах допустимых граничных значений. Система работает штатно. |
| `1` | **Warn** (Желтый) | Обнаружены отклонения в работе одного или нескольких индикаторов. Кластер или компонент функционирует, но, возможно, требует внимания. |
| `0` | **Crit** (Красный) | Обнаружены критические отклонения. Наблюдается значительная деградация производительности, нестабильная работа или полная недоступность кластера либо компонента. |
| `-1` | **Unknown** (Серый) | Статус не определен или данные отсутствуют. |

| Имя<br/>Тип, единицы измерения | Описание |
| :--- | :--- |
| `greenplum_cluster_status.signal`<br/>`DGAUGE` | Комплексный сигнал статуса кластера. Учитывает доступность на чтение и запись, утилизацию счетчика транзакций, количество сегментов в статусах `down` и `not preferred role`. |
| `greenplum_pooler_status.signal`<br/>`DGAUGE` | Комплексный сигнал статуса пулера соединений (Odyssey). Учитывает доступность, утилизацию CPU и памяти. |
| `greenplum_connections_status.signal`<br/>`DGAUGE` | Комплексный сигнал статуса подключений. Учитывает общую утилизацию подключений, утилизацию подключений в состоянии `idle in transaction`, максимальную утилизацию по сегментам. |
| `greenplum_resources_status.signal`<br/>`DGAUGE` | Комплексный сигнал потребления ресурсов хостами. Учитывает Load Average, CPU, память, сетевые пакеты и трафик, дисковое пространство и IO. |
| `greenplum_queries_status.signal`<br/>`DGAUGE` | Комплексный сигнал выполнения запросов, основанный на задержках тестовых запросов на чтение и запись. |
