# Выравнивание по сетке

При использовании [агрегации при записи](../../monitoring/concepts/aggregation.md#aggregation-on-write) временные метки точек, записываемых в метрику-агрегат, выравниваются по сетке равной интервалу сбора метрик, указанному в настройках сервиса (параметр **Fetch interval**).

Этот параметр используется, в том числе если метрики записываются через Push API. Например, если метрики записываются через Push API с шагом в 5 секунд, а в настройках сервиса параметр **Fetch interval** выставлен в 15 секунд, то точки метрики-агрегата будут записываться с шагом в 15 секунд.

Для корректного вычисления агрегатов при использовании Push API необходимо, чтобы между временными метками в каждых двух последовательных push-запросах выдерживался одинаковый интервал. Иными словами, нельзя допускать «джиттера» во временных метках. Такое возможно в ситуации, когда вычисление значений метрик, передаваемых в Solomon, занимает значимое время (единицы секунд), а временная метка проставляется не до вычисления значений, а после.

{% note tip %}

Чтобы не реализовывать выравнивание временных точек по сетке на клиенте, включите выравнивание по сетке на стороне Solomon. Подробнее можно прочитать в разделе [{#T}](../../monitoring/concepts/grid.md).

{% endnote %}
