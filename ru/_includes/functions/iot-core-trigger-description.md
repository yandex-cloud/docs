Триггер для {{ iot-short-name }} предназначен для управления сообщениями, которыми обмениваются устройства и реестры. Он создается для топиков: забирает из них копии сообщений и передает в функцию для обработки. Триггер должен находиться в одном облаке с устройством или реестром, из топика которого он читает сообщения.  
 
 Триггер может быть создан:
 - Для [стандартного топика](../../iot-core/concepts/topic.md), реализованного сервисом.
 - Для топика с произвольными [сабтопиками](../../iot-core/concepts/topic.md#subtopic) и подстановочными символами.
 - Для [алиаса](../../iot-core/concepts/topic.md#aliases) топика.
 
Триггеру для {{ iot-short-name }} необходим сервисный аккаунт для вызова функции.
