1. В [консоли управления]({{ link-console-main }}) выберите каталог, в котором нужно создать кластер {{ k8s }}.
1. Выберите сервис **{{ managed-k8s-name }}**.
1. Нажмите кнопку **Создать кластер**.
1. Введите имя и описание кластера {{ k8s }}. Имя кластера должно быть уникальным в рамках {{ yandex-cloud }}.
1. Укажите **Сервисный аккаунт для ресурсов**, который будет использоваться для создания ресурсов.
1. Укажите **Сервисный аккаунт для узлов**, который будет использоваться узлами для доступа к [реестру Docker-образов](../../container-registry/concepts/registry.md).
1. (Опционально) Укажите **Ключ шифрования**, который будет использоваться для [шифрования секретов](../../managed-kubernetes/concepts/encryption.md).

   {% include [write-once-settings](write-once-setting.md) %}

1. Укажите [релизный канал](../../managed-kubernetes/concepts/release-channels-and-updates.md). Эту настройку невозможно изменить после создания кластера.
1. В блоке **Конфигурация мастера**:
   * В поле **Версия {{ k8s }}** выберите версию {{ k8s }}, которая будет установлена на мастере.
   * В поле **Публичный адрес** выберите способ назначения адреса:
     * **Автоматически** — чтобы назначить случайный IP-адрес из пула адресов {{ yandex-cloud }}.
     * **Без адреса** — чтобы не назначать публичный IP-адрес.
   * В поле **Тип мастера** выберите тип мастера:
     * **Зональный** — создается в подсети в одной зоне доступности.

     
     * **Региональный** — создается распределенно в трех подсетях в каждой зоне доступности.
   * Выберите [зону доступности](../../overview/concepts/geo-scope.md), в которой будет создан [мастер](../../managed-kubernetes/concepts/index.md#master).

     Шаг доступен только для зонального мастера.


   * В поле **Облачная сеть** выберите сеть, в которой будет создан мастер. Если сети нет, [создайте ее](../../vpc/operations/network-create.md).
   * В поле **Подсеть** выберите подсеть, в которой будет создан мастер. Если подсети нет, [создайте ее](../../vpc/operations/subnet-create.md).

     
     Для регионального мастера необходимо указать подсеть в каждой зоне доступности.


   * Выберите группы безопасности для сетевого трафика кластера.

     {% include [security-groups-alert](security-groups-alert.md) %}

1. В блоке **Настройки окна обновлений**:
   * В поле **Частота обновлений / Отключение** настройте окно для обновлений:
     * **Отключено** — отключение автоматических обновлений.
     * **В любое время** — обновления разрешены в любое время.
     * **Ежедневно** — обновления будут происходить во временном интервале, указанном в поле **Время (UTC) и продолжительность**.
     * **В выбранные дни** — обновления будут происходить во временном интервале, указанном в поле **Расписание по дням**.
1. В блоке **Сетевые настройки кластера**:
   * (опционально) Выберите [контроллер сетевых политик](../../managed-kubernetes/concepts/network-policy.md#policy-controllers):
     * **Включить сетевые политики**, чтобы задействовать Calico.
     * **Включить туннельный режим**, чтобы задействовать Cilium.
   * Укажите **CIDR кластера** — диапазон IP-адресов, из которого будут выделяться IP-адреса для подов.
   * Укажите **CIDR сервисов** — диапазон IP-адресов, из которого будут выделяться IP-адреса для сервисов.
   * Задайте маску подсети узлов и максимальное количество подов в узле.
1. Нажмите кнопку **Создать кластер**.
