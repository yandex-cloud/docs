Чтобы создать подключение к {{ CH }} over YT:

1. Перейдите на [страницу подключений](https://datalens.yandex-team.ru/connections).
1. Нажмите кнопку **Создать подключение**.
1. Выберите подключение **CH over YT**.
1. Задайте **Имя подключения**. Имя может быть произвольным.
1. Выберите **Метод аутентификации**:

   {% list tabs %}

    - OAuth-токен

      При выборе этого метода запросы в YT будут выполняться от имени пользователя, чей токен указан в поле **Токен YT**. Вы можете указать свой токен или токен робота. Другие пользователи {{ datalens-short-name }} смогут работать с объектами, построенными поверх этого подключения, без дополнительных прав в YT.

      1. Нажмите **Получить токен** или укажите вручную OAuth-токен для работы с YT.
      1. **Кластер**. Укажите кластер YT, на котором хранятся таблицы. Доступны Hahn, Arnold и Vanga.
      1. **Клика**. Укажите алиас запущенной клики. Если вы используете публичную клику, ознакомьтесь с [ограничениями](../../../datalens/tutorials/data-from-ch-over-yt.md#before-you-begin). Для работы с {{ datalens-short-name }} рекомендуем создать [приватную клику](https://yt.yandex-team.ru/docs/description/chyt/cliques/start).
      1. **Уровень доступа SQL запросов**. Выберите уровень доступа к SQL-запросам для пользователя.
      1. Нажмите **Создать**. Подключение появится в списке.

    - От имени пользователя

       Авторизация происходит по cookie на стороне YT от имени пользователя, просматривающего датасет, чарт или дашборд. Предоставьте доступ к таблице YT, указанной в датасете, другим пользователям, чтобы они могли просматривать отчеты.

      1. **Кластер**. Укажите кластер YT, на котором хранятся таблицы. Доступны Hahn, Arnold и Vanga.
      1. **Клика**. Укажите алиас запущенной клики. Если вы используете публичную клику, ознакомьтесь с [ограничениями](../../../datalens/tutorials/data-from-ch-over-yt.md#before-you-begin). Для работы с {{ datalens-short-name }} рекомендуем создать [приватную клику](https://yt.yandex-team.ru/docs/description/chyt/cliques/start).
      1. **Уровень доступа SQL запросов**. Выберите уровень доступа к SQL-запросам для пользователя.
      1. Нажмите **Создать**. Подключение появится в списке.

   {% endlist %}


{% note info %}

Вы можете проверить подключение к хосту перед созданием. Для этого нажмите кнопку **Проверить подключение**.

{% endnote %}
