# Квоты и ресурсы

В общем случае единая квота на все сервисы MDB выделяется для определенного сервиса в ABC. Каждому сервису ABC соответствует отдельное облако в MDB.

## Особенности заказа квот {#order-features}

Требования к заказу квоты:

* Заказ должен быть сформирован с учетом [ограничений и лимитов](#quota-limits).
* Укажите дата-центр размещения.
* Соотношение количества vCPU к гигабайтам RAM должно быть 1:4 или 1:8. Исключения возможны в случае заказа классов хостов с разным соотношением или при наличии оставшейся квоты.

Выдача квоты происходит после введения серверов в эксплуатацию и подготовки, которая может занять от одной до двух недель.

### Ограничения и лимиты {#quota-limits}

Для заказа доступны классы хостов **s3** на платформе Intel Cascade Lake. Классы хостов для баз данных имеют следующее соотношение количества vCPU к гигабайтам RAM:

* 1:4 — стандартные конфигурации (`standard`);
* 1:8 — конфигурации с увеличенным соотношением количества гигабайт RAM к количеству vCPU (`memory-optimized`).

Хранилище на локальных HDD-дисках (`local-hdd`) для классов хостов **s3** предоставляется только для {{ CH }} в виде массива RAID-10 эффективной емкостью 12 800 ГБ на хост с не менее 8 vCPU.

На [новом оборудовании](quotas-control.md#requirements) допускается использовать устаревшие классы хостов **db1** и **s2**.
