В {{ compute-name }} вы можете временно отключать одну или несколько [зон доступности](../../overview/concepts/geo-scope.md) для [групп виртуальных машин](../../compute/concepts/instance-groups/index.md). Это может быть полезно для сохранения работоспособности ваших сервисов, когда в зоне доступности проводится тестирование, обслуживание, наблюдаются неполадки или [инцидент](../../compute/concepts/instance-groups/zonal-inc/overview.md).

> Например, с помощью отключения зоны для группы ВМ вы можете решить проблему «серых отказов» вашего сервиса в этой зоне — когда проверки состояния и средства мониторинга не фиксируют отказы, но при реальной рабочей нагрузке часть запросов не доходит до ВМ.
> 
> Также полезно временно отключить зону для группы ВМ при зональном инциденте. Таким образом вы сможете контролируемо ввести ВМ из пораженной зоны в эксплуатацию при выходе из инцидента. Подробнее см. [Поведение группы ВМ при временном отключении зоны и инциденте в этой же зоне](../../compute/concepts/instance-groups/disable-enable-zone.md#zonal-inc-shift).

{% note warning %}

Временное отключение зон доступности не работает для групп ВМ, созданных [кластером {{ managed-k8s-name }}](../../managed-kubernetes/concepts/index.md#kubernetes-cluster) в качестве групп узлов.

{% endnote %}

Обратно включить зону доступности для группы ВМ можно в любой момент либо задать таймаут для автоматического включения.
