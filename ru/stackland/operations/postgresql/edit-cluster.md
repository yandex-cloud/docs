# Изменить настройки кластера PostgreSQL

Если у вас есть кластер [PostgreSQL](../../concepts/components/postgresql.md) в [проекте](../projects/create-project.md)), вы можете изменить его настройки.

{% list tabs group=instructions %}

- CLI {#cli}

  1. Откройте ресурс `PostgresqlCluster`. Например, с помощью команды `open postgresqlcluster.yaml`.
  1. Откройте файл ресурса и отредактируйте параметры:

      * `metadata.name` — имя кластера.
      * `metadata.namespace` — имя проекта.
      * `spec.instances` — количество копий кластера.
      * `spec.storage.size` — размер хранилища. Можно только увеличивать.
      * `spec.storage.autoScaling.enabled` — включение автоматического увеличения размера хранилища.
      * `spec.storage.autoScaling.maxSize` — максимальный размер хранилища.
      * `spec.storage.autoScaling.standardIncreasePercent` — процент увеличения размера хранилища.
      * `spec.storage.autoScaling.resizeTriggerPercent` — процент использования хранилища, после которого будет запущено увеличение размера хранилища.
      * `spec.storage.readOnlyTriggerPercent` — процент использования диска для перевода в режим «только для чтения».
      * `spec.enableSuperuserAccess` — при редактировании можно выключить доступ суперпользователя.
      * `spec.resources` — ограничение на ресурсы пода.
      * `spec.postgresConfiguration` — конфигурация PostgreSQL.
      * `spec.poolers.[ro|rw|r].port` — настройки порта для менеджера соединений.
      * `spec.poolers.[ro|rw|r].instances` — количество копий менеджера соединений.
      * `spec.poolers.[ro|rw|r].type` — тип кластера для менеджера соединений. Например, `ClusterIP`.
      * `spec.poolers.[ro|rw|r].odyssey.poolMode` — режим работы пула соединений с базой данных (session или transaction).
      * `spec.backup` — настройки резервного копирования.
      * `spec.deletionProtection` — защита от удаления. Значение `true` запрещает удаление кластера до отключения защиты.

  1. Примените манифест: `kubectl apply -f postgresqlcluster.yaml -n <название проекта>`. При необходимости можно прописать название проекта в параметр ресурса `metadata.namespace` и не использовать в команде.


- Консоль управления {#console}

  1. Если вы еще не открыли проект, выберите проект.
  1. В левом меню выберите **PostgreSQL Clusters**.
  1. Выберите кластер.
  1. Нажмите **Редактировать**.
  1. Измените поля:

      * **Название кластера**. Только строчные буквы, цифры и дефисы.
      * **Количество инстансов**, количество копий кластера.
      * **Вычислительные ресурсы**, где **Лимиты** — верхняя граница, **Запросы** — нижняя граница.
      * **Хранилище**, увеличьте размер хранилища или включите автоматическое увеличение.
      * **Менеджер соединений**, сервис для записи и чтения, используется для всех операций изменения данных и критически важных транзакций.
      * **Резервные копии**, настройки резервного копирования базы данных в S3-бакет.
      * **Защита от удаления** — переключатель. Включённая защита запрещает удаление кластера до её отключения.

  1. Нажмите **Сохранить**.

{% endlist %}

Готово, настройки кластера изменены.
