# Загрузить образы {{ speechsense-name }}

Образы компонента {{ speechsense-name }} не входят в базовую поставку {{ stackland-name }}. Перед активацией компонента загрузите образы в кластер.

## Перед началом работы {#before-you-begin}

1. Убедитесь, что сервисный аккаунт кластера имеет доступ к защищенным образам в реестре контейнеров.
1. Скачайте актуальную версию утилиты `sladm`, как описано в [Руководстве по установке](../../quickstart.md#download-files).

## Кластер с доступом в интернет {#online}

Если кластер имеет доступ в интернет, загрузите образы {{ speechsense-name }} напрямую из реестра контейнеров:

```bash
sladm pull \
  --image-bundle speechsense \
  --config config/
```

Где:

* `--image-bundle` — имя защищенного компонента. Укажите `speechsense` для загрузки образов {{ speechsense-name }}.

Команда создаст папку с образами контейнеров в формате OCI.

{% note info %}

Образы {{ speechsense-name }} включают модели машинного обучения и занимают значительный объем. Убедитесь, что на диске достаточно свободного места (не менее 100 ГБ).

{% endnote %}

## Изолированный кластер {#airgapped}

Если кластер не имеет доступа в интернет, загрузите образы на машину с доступом в интернет, перенесите их во внутренний контур и загрузите в локальный реестр кластера.

### Загрузить образы на машине с доступом в интернет {#download-images}

На машине с доступом в интернет выполните команду:

```bash
sladm pull \
  --image-bundle speechsense \
  --config config/
```

### Перенести артефакты во внутренний контур {#transfer}

Перенесите на машину с доступом к кластеру:

* папку с образами контейнеров (формат OCI);
* бинарный файл `sladm`.

Способ переноса зависит от политик безопасности вашей организации: съемные носители, защищенные файловые хранилища или изолированные сетевые сегменты.

### Загрузить образы в локальный реестр {#push-images}

На машине с доступом к кластеру выполните команду:

```bash
sladm push \
  --local-registry \
  --kubeconfig=<путь_к_kubeconfig> \
  --image-bundle-folder <путь_к_папке_с_образами>
```

Где:

* `--local-registry` — указывает на использование локального реестра кластера.
* `--kubeconfig` — путь к файлу kubeconfig для доступа к кластеру.
* `--image-bundle-folder` — путь к папке с образами контейнеров.

Если {{ speechsense-name }} устанавливается не на всех узлах кластера, можно задать список узлов, куда нужно скопировать образы, в формате `--target-registry <имя_хоста_или_IP_1>:5001,<имя_хоста_или_IP_2>:5001`.
