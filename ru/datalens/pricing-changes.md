# Изменения в правилах тарификации для {{ datalens-full-name }}

С 1 декабря в {{ datalens-full-name }} изменятся правила тарификации. В рамках единого нового тарифа всем пользователям будут доступны расширенные возможности сервиса, например: Нейроаналитик, отчеты, JavaScript-кастомизации в Editor, фоновый экспорт данных из чартов, стилизация UI и др. Новый тариф будет доступен бесплатно для индивидуального использования, платно — для командной работы.

## Новые правила тарификации {#new-rules}

### Что изменится {#changes}

Тарификация будет вестись по количеству _рабочих мест_ в рамках одного экземпляра {{ datalens-full-name }}.

* Каждому пользователю {{ datalens-full-name }} понадобится рабочее место, даже если пользователь только просматривает или редактирует дашборды.
* Рабочие места выдаются автоматически при первом входе в сервис новых пользователей.
* Администратор может управлять рабочими местами, подробнее см. [ниже](#manage-seats).
* Доступ к объектам в {{ datalens-full-name }} по-прежнему регулируется с помощью [ролей или прав доступа](./security/index.md), рабочее место не предоставляет доступ к объектам.

### Активные пользователи и рабочие места {#difference}

В [правилах тарификации](./pricing.md), которые действуют до 1 декабря 2025 года, тарифицируются только _активные пользователи_ {{ datalens-name }}. Активные пользователи — это пользователи, действия которых привели к запросу данных из источника: например, пользователь открыл или редактировал дашборд, чарт или датасет.

В новых правилах тарификации, которые вступят в силу 1 декабря 2025 года, рабочее место тарифицируется вне зависимости от активности пользователя, которому оно назначено.

### Управление рабочими местами {#manage-seats}

1 декабря в настройках {{ datalens-name }} в настройках сервиса появится возможность управления рабочими местами.

{% note info %}

Администратор (пользователь с ролью `datalens.admin`) всегда, даже без рабочего места, сможет зайти в настройки сервиса и управлять списком рабочих мест. Без рабочего места другие возможности сервиса администратору будут недоступны.

{% endnote %}

Администратор сможет:

* просматривать количество рабочих мест в экземпляре, в том числе свободных (которые никому не назначены);
* просматривать список пользователей с рабочими местами;
* переназначать рабочие места от пользователя к пользователю;
* увеличивать или сокращать число рабочих мест в экземпляре.

### Цены {#prices}

Стоимость использования {{ datalens-name }} в рамках одного экземпляра определяется следующим образом:
* 1 рабочее место — бесплатно;
* больше 1 рабочего места — 990,00 ₽ в месяц за каждое рабочее место.

Подсчет рабочих мест ведется в рамках месяца. Если в экземпляре больше одного рабочего места, общая стоимость списывается из расчета:

Количество рабочих мест × Стоимость одного рабочего места


{% cut "Пример расчета стоимости" %}

Рассмотрим на примере мая 2026 года, когда новые правила тарификации уже вступят в силу.

В экземпляре {{ datalens-name }} по состоянию на 1 мая 2026 года куплено 10 рабочих мест.

12 мая приобрели 5 рабочих мест, а 20 мая — еще 3 рабочих места.

Общая стоимость за май составит (10 + 5 + 3) × 990,00 ₽ = 17 820,00 ₽

{% endcut %}


Сокращение количества рабочих мест можно запланировать в любой день, но изменение вступит в силу только в первый день нового месяца.

Порядок оказания технической поддержки определяется в соответствии с выбранным [тарифным планом поддержки {{ yandex-cloud }}](../support/overview.md).

## Получение скидки {#discounts}

Для пользователей тарифа **Community** по состоянию на 16 ноября 2025 года действуют специальные условия.

Если до 1 декабря 2025 года вы перейдете на тариф **Business** или с 1 декабря до 31 декабря досрочно завершите пробный период, то получите скидку 50% с момента перехода и до конца 2026 года на все рабочие места.

{% note info %}

При завершении пробного периода стоимость рабочих мест за первый месяц будет рассчитана пропорционально количеству оставшихся в месяце дней. 

{% endnote %}


{% cut "Пример расчета стоимости со скидкой" %}

Рассмотрим на примере экземпляра {{ datalens-name }}, в котором по состоянию на 16 ноября 2025 года активен тариф **Community**, а 24 декабря привязали платежный аккаунт и досрочно завершили пробный период. В декабре 31 день, до конца месяца остается 8 дней, стоимость будет рассчитана пропорционально количеству дней.

24 декабря приобрели 10 рабочих мест, 28 декабря — 5 рабочих мест, а 29 декабря — еще 3 рабочих места.

Общая стоимость за декабрь составит (10 + 5 + 3) × (990,00 ₽ / 2) × (8 / 31) = 2 299,35 ₽

{% endcut %}


## Ограничение числа запросов из непублично встроенных объектов {#packet-queries}

От количества рабочих мест будет зависеть лимит на количество запросов из [непублично встроенных объектов](./security/private-embedded-objects.md) в пределах экземпляра {{ datalens-name }}. Обычные запросы, производимые пользователями в интерфейсе {{ datalens-name }}, а также запросы к [публично встроенным объектам](./concepts/datalens-public.md) не ограничены. 

* С 1 марта 2026 года будет зафиксировано ограничение: 2 000 запросов в месяц из непублично встроенных объектов на одно рабочее место. Это ограничение будет действовать не для каждого отдельно взятого рабочего места, а для всего экземпляра {{ datalens-name }} из расчета:

   > 2 000 запросов × Количество рабочих мест

* Рабочие места не связываются со встроенными объектами напрямую, их необязательно назначать внешним пользователям, которые просматривают эти объекты. Значение имеет только общее количество мест, купленное в пределах экземпляра, как свободных, так и назначенных пользователям.

   Таким образом, чем активнее используется экземпляр {{ datalens-name }} (чем больше в нем куплено рабочих мест), тем выше будет лимит на непубличные встраивания.

* Если c {{ datalens-name }} работают несколько пользователей, но при этом производится много запросов к непубличным встраиваниям, потребуется докупить рабочие места, исходя из расчета выше.

* Учитывается каждый запрос за данными из непублично встроенного объекта. Например, у вас встроен дашборд с 10 чартами и 5 селекторами. Тогда при обновлении страницы с дашбордом будет выполнено 15 запросов.
   
   Уже сейчас вы можете оценить количество запросов из непублично встроенных объектов с помощью подключения [Usage Analytics](./operations/connection/create-usage-tracking.md). Если общее количество таких запросов в экземпляре будет превышать ограничение, может потребоваться покупка дополнительных рабочих мест.
