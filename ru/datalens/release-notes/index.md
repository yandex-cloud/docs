## Изменения базовых возможностей {#base}


* С 8 по 16 декабря 2025 года в рамках ежегодного [фестиваля {{ datalens-short-name }}]({{ link-datalens-main }}/festival) провели серию офлайн- и онлайн-активностей: видео, прямые эфиры, кейсы и конкурсы, где участники познакомились с новыми возможностями {{ datalens-short-name }} — [AI‑инструментами](../concepts/neuroanalyst.md), [кастомизацией](../settings/appearance.md), [on-premises]({{ link-datalens-main }}/on-premises/docs/ru/) и миграцией.
* Появились новые работы в [{{ datalens-gallery }}]({{ link-datalens-main }}/gallery). Подробнее см. в чате [{{ datalens-short-name }}](https://t.me/YandexDataLens/28631/145412) в Telegram.
* Теперь [Нейроаналитик на дашборде](../dashboard/insights.md) может искать инсайты сразу в рамках целой вкладки на дашборде. Он проанализирует вопрос и содержимое дашборда, выберет релевантные для заданного вопроса чарты, возьмет из них данные, после чего подготовит итоговый инсайт. В качестве контекста используются описания дашбордов, чартов, датасетов и подключений, а также загружаемые данные.
  Чтобы попробовать, нажмите вверху дашборда кнопку **Нейроаналитик** — справа откроется вкладка **Нейроаналитик**.


* Реализовали возможность добавлять информацию [о подключении](../operations/connection/add-description.md) и [о датасете](../dataset/add-description.md).
* Добавили в редактор формул для [вычисляемых полей](../concepts/calculations/index.md):

  * [функции хеширования](../function-ref/hash-functions.md): `MD5`, `SHA256`, `CITYHASH64` и другие.
  * набор [нативных функций](../function-ref/native-functions.md) `DB_CALL` для доступа к функциям БД. Через функции `DB_CALL` можно:
    
    * безопасно использовать построковые и агрегатные функции из любых БД;
    * обращаться к типам, которые пока не поддерживаются в {{ datalens-short-name }};
    * оптимизировать свою работу и нагрузку на источники.

    {% note info %}
    
    Используйте нативные функции для точечных, специфичных операций, которые пока не реализованы в {{ datalens-short-name }}. Для стандартных действий (агрегации, математических) используйте встроенные функции {{ datalens-short-name }} — они могут быть лучше оптимизированы.

    {% endnote %}


* Реализовали возможность сохранить чарт в формате `PNG`. Для этого:

  1. Откройте чарт и в правом верхнем углу чарта нажмите ![image](../../_assets/console-icons/ellipsis.svg) → ![icon](../../_assets/console-icons/arrow-down-to-line.svg) **Сохранить как** → **Изображение**.

  1. Выберите разрешение:

     * стандартное 800 x 600 px;
     * широкоформатное 1600 x 720 px;
     * укажите вручную другое.
  
  1. При необходимости включите опцию **Отображать элементы интерфейса**.
  1. Нажмите кнопку **Скачать**.



## Исправления и улучшения {#fixes}

* Исправили некорректное сохранение исходного фона при копировании виджета на дашборде. Раньше, если исходный фон виджета был задан `Базовый фон чарта`, то при копировании он сбрасывался в прозрачный.
* Обновили внешний вид окна настройки селекторов и чартов.


* Восстановили отображение `Request-id` и `Trace-Id` в ошибках [непубличного встраивания](../security/private-embedded-objects.md).



### Исправления в Нейроаналитике {#neuroanalyst-fixes}

* Теперь в чартах сохраняется контекст чата с [Нейроаналитиком для создания вычисляемых полей](../concepts/calculations/formulas-helper.md):

  * при изменении формулы;
  * после закрытия чата.


### Исправления в чартах {#chart-fixes}

* Исправили проблему отображения цветов в [круговой диаграмме](../visualization-ref/pie-chart.md) — теперь цвета соответствуют выбранным в [настройках полей датасета](../dataset/create-dataset.md#setup-fields).
