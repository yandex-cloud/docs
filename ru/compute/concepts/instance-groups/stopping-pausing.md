# Остановка группы и приостановка процессов

Группу виртуальных машин можно остановить полностью или частично.

## Остановка группы {#stopping}

Полная _остановка группы_ прекращает работу всех ВМ и прерывает процессы: [автоматическое масштабирование](scale.md) группы, [автоматическое восстановление](autohealing.md) и [обновление](deploy/index.md) ВМ. 

[Статус группы](statuses.md#group-statuses) сначала меняется на `STOPPING`, а затем — на `STOPPED`. [Статусы всех ВМ](statuses.md#vm-statuses) сначала меняются на `STOPPING_INSTANCE`, а затем — на `STOPPED`. После этого использование ВМ перестает [тарифицироваться](../../pricing.md).

Все существенные изменения остановленной группы, например изменение размера группы или конфигурации виртуальных машин, будут применены только после запуска группы.
  
Остановить группу можно по [инструкции](../../operations/instance-groups/stop.md).
  
## Приостановка процессов {#pausing}

_Приостановка процессов_ в группе прерывает только процессы: автоматическое масштабирование группы, автоматическое восстановление и обновление ВМ. Все ВМ, запущенные на момент приостановки процессов, продолжают работать.

Статус группы меняется на `PAUSED`. Статусы всех ВМ остаются неизменными, их использование продолжает тарифицироваться.

Все существенные изменения группы с приостановленными процессами, например изменение размера группы или конфигурации виртуальных машин, будут применены только после возобновления процессов.
  
Приостановить процессы в группе можно по [инструкции](../../operations/instance-groups/pause-processes.md).
