# Мониторинг трансферов

В {{ data-transfer-name }} реализован мониторинг трансферов. Чтобы отслеживать процессы копирования и репликации данных, перейдите на вкладку **Мониторинг** и воспользуйтесь метриками.

## Репликация {#replication}

* **Maximum lag on delivery, [s]** — максимальное время запаздывания применения изменений на приемнике.

    *sinker.pusher.time.row_max_lag_sec* — разница в секундах между временем фиксирования изменения на источнике и временем его применения на приемнике. Вычисляется для каждой строки. Для снапшота за время фиксирования изменения на источнике принимается время начала копирования.

    На графике отображается максимальное значение по всем строкам из транзакции.

* **Tables sync progress, [rows]** — прогресс применения изменений на приемнике.

    *sinker.table.rows* — количество зафиксированных строк на приемнике.

    График показывает количество строк, зафиксированных для каждой таблицы в течение 15 секунд. Имя таблицы задается в метке `table`.

## Копирование {#copy}

* **Await to copy, [rows]** — количество оставшихся строк при копировании снапшота.

    *task.snapshot.remainder.table* — считается для каждой таблицы. После завершения копирования, все значения становятся равными нулю. Имя таблицы задается в метке `table`.
