# Создание эндпоинта источника

Чтобы создать эндпоинт источника:

{% list tabs %}

- Консоль управления

  1. В [консоли управления]({{ link-console-main }}) выберите каталог, в котором будет создан эндпоинт.
  1. В списке сервисов выберите **{{ data-transfer-name }}**.
  1. Нажмите кнопку **Создать эндпоинт**.
  1. В открывшемся окне убедитесь, что в поле **Направление** указано **Источник**.
  1. Укажите **Имя** эндпоинта. Используйте строчные латинские буквы и цифры.
  1. (опционально) Укажите **Описание** эндпоинта.
  1. В поле **База данных** выберите СУБД, из которой вы хотите передавать данные:
  
      {% cut "{{mmy-short-name}}" %}

      В блоке **Параметры эндпоинта** укажите:
      - **Идентификатор кластера**, из которого необходимо передавать данные.
      - **Имя базы данных** в кластере.
      - **Имя пользователя** базы данных.
      - **Пароль** пользователя для доступа к базе данных.
      - (опционально) **Идентификатор репликационного слота** для подключения к кластеру БД.
      - (опционально) **Белый список таблиц**. Будут передаваться данные только из таблиц белого списка.
      - (опционально) **Черный список таблиц**. Данные из таблиц черного списка передаваться не будут.
      - (опционально) **Часовой пояс базы** в формате UTC.
      - (опционально) **Идентификатор подсети**, через которую можно подключиться к кластеру БД.

      {% endcut %}

      {% cut "{{mpg-short-name}}" %}

      В блоке **Параметры эндпоинта**:
      - В поле **Идентификатор кластера** укажите идентификатор кластера, из которого необходимо передавать данные.
      - Укажите **Имя базы данных** в кластере.
      - Укажите **Имя пользователя** базы данных.
      - Введите **Пароль** пользователя для доступа к базе данных.
      - (опционально) Добавьте таблицы в **Белый список таблиц**. Будут передаваться данные только из таблиц белого списка.
      - (опционально) Добавьте таблицы в **Черный список таблиц**. Данные из таблиц черного списка передаваться не будут.
      - (опционально) Укажите **Идентификатор репликационного слота PostgreSQL** для подключения к кластеру БД.
      - (опционально) Укажите **Максимальный размер WAL для слота репликации**.
      - (опционально) В поле **Схема БД приёмника, в которой будет создана техническая таблица __consumer_keeper** введите имя схемы хранения.
      - (опционально) При необходимости выберите опцию **Объединить наследуемые таблицы** для объединения содержимого таблиц.
      - (опционально) Укажите **Сетевой интерфейс для эндпоинта**, через который можно подключиться к кластеру БД.

      {% endcut %}

      {% cut "{{MY}}" %}

      В блоке **Параметры эндпоинта** укажите:
      - **IP или FQDN хоста**, из которого необходимо передавать данные.
      - **Порт базы данных** для подключения к СУБД.
      - **Имя базы данных** в кластере.
      - **Имя пользователя** базы данных.
      - **Пароль** пользователя для доступа к базе данных.
      - (опционально) **Идентификатор репликационного слота** для подключения к кластеру БД.
      - (опционально) **Белый список таблиц**. Будут передаваться данные только из таблиц белого списка.
      - (опционально) **Черный список таблиц**. Данные из таблиц черного списка передаваться не будут.
      - (опционально) В поле **PEM-сертификат** нажмите кнопку **Загрузить файл** и загрузите [PEM-сертификат](../../../managed-mysql/operations/connect.md#Configuring-an-SSL-certificate) для шифрования передаваемых данных.
      - (опционально) **Часовой пояс базы** в формате UTC.
      - (опционально) **Идентификатор подсети**, через которую можно подключиться к кластеру БД.

      {% endcut %}

      {% cut "{{PG}}" %}

      В блоке **Параметры эндпоинта**:
      - В поле **IP или FQDN хоста** укажите IP-адрес или доменное имя хоста, из которого необходимо передавать данные.
      - В поле **Порт базы данных** укажите порт для подключения к СУБД.
      - Укажите **Имя базы данных** в кластере.
      - Укажите **Имя пользователя** базы данных.
      - Введите **Пароль** пользователя для доступа к базе данных.
      - (опционально) Добавьте таблицы в **Белый список таблиц**. Будут передаваться данные только из таблиц белого списка.
      - (опционально) Добавьте таблицы в **Черный список таблиц**. Данные из таблиц черного списка передаваться не будут.
      - (опционально) Укажите **Идентификатор репликационного слота PostgreSQL** для подключения к кластеру БД.
      - (опционально) Укажите **Максимальный размер WAL для слота репликации**.
      - (опционально) В поле **Схема БД приёмника, в которой будет создана техническая таблица __consumer_keeper** введите имя схемы хранения.
      - (опционально) При необходимости выберите опцию **Объединить наследуемые таблицы** для объединения содержимого таблиц.
      - (опционально) Укажите **Сетевой интерфейс для эндпоинта**, через который можно подключиться к кластеру БД.

      {% endcut %}

  1. Нажмите кнопку **Создать**.
