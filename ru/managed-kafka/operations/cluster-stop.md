# Остановка и запуск кластера

При необходимости вы можете остановить кластер {{ KF }} и запустить его заново. При остановке кластера все данные в нем сохранятся: они снова станут доступны, когда вы запустите кластер.

Время работы остановленного кластера не тарифицируется: вы продолжаете платить только за объем хранилища в соответствии с [тарифом](../pricing.md#prices-storage).

{% include [pricing-status-warning.md](../../_includes/mdb/pricing-status-warning.md) %}

## Остановить кластер {#stop-cluster}

{% list tabs %}

- API
  
  Воспользуйтесь методом API [stop](../api-ref/Cluster/stop.md) для остановки работы всех хостов кластера: передайте значение идентификатора требуемого кластера в параметре `cluster_id` запроса.
  
  Чтобы узнать идентификатор кластера, [получите список кластеров в каталоге](cluster-list.md#list-clusters).

{% endlist %}

## Запустить кластер {#start-cluster}

Кластер в статусе `STOPPED` можно запустить заново.

{% list tabs %}

- API
  
  Чтобы запустить кластер, воспользуйтесь методом API [start](../api-ref/Cluster/start.md): передайте значение идентификатора требуемого кластера в параметре `cluster_id` запроса.
  
  Чтобы узнать идентификатор кластера, [получите список кластеров в каталоге](cluster-list.md#list-clusters).

{% endlist %}
