# Просмотр логов кластера

{{ mkf-name }} позволяет получить логи кластера для просмотра и изучения.

Вы можете получить:

* [Простой фрагмент лога](#get-log).
* [Поток записей лога](#get-log-stream) в кластере (поддерживается семантика команды `tail -f`).

{% note info %}

Здесь под логом понимается системный лог кластера и его хостов. Этот лог не имеет отношения к логу раздела для топика {{ KF }}, в который брокер записывает поступающие от производителей сообщения.

{% endnote %}

## Получить лог кластера {#get-log}

{% list tabs %}

* Консоль управления

    1. Перейдите на страницу каталога и выберите сервис **{{ mkf-name }}**.
    1. Нажмите на имя нужного кластера и выберите вкладку **Логи**.
    1. Укажите период времени, за который нужно отобразить лог.

* API

    Воспользуйтесь методом API [listLogs](../api-ref/Cluster/listLogs.md) и передайте идентификатор кластера в параметре `clusterId` запроса.

    {% include [Ограничения метода на количество записей](../../_includes/mdb/mkf/log-limits.md) %}

    Идентификатор кластера можно получить со [списком кластеров в каталоге](cluster-list.md#list-clusters).

{% endlist %}

## Получить поток записей лога {#get-log-stream}

Этот способ работы с логами отличается от [получения простого фрагмента лога](#get-log) тем, что кластер может досылать новые записи лога по мере их появления. Это поведение эквивалентно семантике команды `tail -f` для работы с логами.

{% list tabs %}

* API

    Воспользуйтесь методом API [streamLogs](../api-ref/Cluster/streamLogs.md) и передайте идентификатор кластера в параметре `clusterId` запроса.

    {% include [Ограничения метода на количество записей](../../_includes/mdb/mkf/log-limits.md) %}

    Если не задавать значение параметра `toTime`, то в поток будут приходить новые записи лога по мере их появления.

    Идентификатор кластера можно получить со [списком кластеров в каталоге](cluster-list.md#list-clusters).

{% endlist %}
