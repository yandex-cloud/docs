# Топики и разделы

{{ KF }} — это распределенная система для обмена данными между приложениями по принципу «публикация-подписка».
Данные передаются в виде одного или нескольких сообщений.

Топик — это способ группировки потоков сообщений по категориям.
Один или несколько [процессов-производителей](producers-consumers.md) публикуют сообщения в топик, а [процессы-потребители](producers-consumers.md), подписавшись на топик, могут читать сообщения из него.

Для каждого топика {{ KF }} ведет лог сообщений, который разбит на несколько разделов.

Каждый раздел представляет из себя линейно упорядоченную последовательность сообщений. При работе с топиком процессы-производители публикуют сообщения в начало раздела, процессы-потребители читают сообщения из конца раздела. 

Использование разделов позволяет:
- Распределить нагрузку по хранению сообщений и обработке запросов от производителей и потребителей между несколькими [брокерами {{ KF }}](brokers.md).
- Обеспечить отказоустойчивость: раздел может реплицироваться на указанное число брокеров.
- Легко масштабировать количество производителей и потребителей в зависимости от нагрузки. 

{{ KF }} управляет лишь хранением сообщений в топике, обеспечивая:
- Репликацию разделов.
- Сжатие сообщений.
- Очистку лога [в соответствии с политикой](../operations/cluster-topics.md#create-topic) при устаревании сообщений в конце разделов или достижении заданного размера лога.

Процессами записи и чтения управляют производители и потребители:
- Процесс-производитель при публикации данных сам выбирает топик, в который нужно их опубликовать.

  При публикации конкретного сообщения процесс выбирает раздел топика, и записывает сообщение в начало этого раздела. 

- Процессы-потребители организуются в группы. Группа, подписавшись на топик, получает доступ ко всем разделам топика. Затем разделы распределяются между членами группы так, что к каждому разделу имеет доступ на чтение только один процесс-потребитель из группы.

  Каждый процесс-потребитель читает сообщения начиная с конца назначенных ему разделов, оперируя смещением внутри раздела.
