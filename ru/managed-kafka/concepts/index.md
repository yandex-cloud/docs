# Взаимосвязь ресурсов сервиса

Сервис {{ mkf-full-name }}  помогает разворачивать и поддерживать кластеры серверов {{ KF }} в инфраструктуре Яндекс.Облака.

{{ KF }} — это распределенная система для обмена данными между приложениями по принципу «публикация-подписка». 

Каждый кластер {{ mkf-name }} состоит из одного или нескольких [хостов-брокеров](brokers.md), на которых размещены [топики](topics.md) и их разделы. Хосты кластера могут находиться в разных зонах и даже разных регионах доступности. [Подробнее о географии Облака](../../overview/concepts/geo-scope.md).

Приложения-[производители](producers-consumers.md) данных [подключаются](../operations/cluster-connect.md) к кластеру и отправляют сообщения в определенные топики. [Потребители](producers-consumers.md), которые должны получить эти сообщения, [подписываются](../operations/cluster-connect.md) на эти топики и читают данные из них. При таком подходе, если приложение-производитель изменится, вам не понадобится заново настраивать приложения-потребители. 


Подробнее см. [в документации {{ KF }}](https://kafka.apache.org/intro).
