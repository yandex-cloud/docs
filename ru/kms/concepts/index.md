# Концепции {{ kms-name }}

{{ kms-name }} — сервис для создания и управления ключами шифрования в Яндекс.Облаке. С помощью сервиса можно управлять правами доступа к ключам шифрования, обеспечивать ротацию криптографического материала и непосредственно шифровать данные. На этапе Preview сервис позволяет создавать только симметричные ключи.

Сервис позволяет управлять _ключами_ и их _версиями_.


## Ключ {#key}

Ключ — это набор версий криптографического материала, которые можно использовать для шифрования или расшифровки данных.

Каждый ключ обладает следующими характеристиками:
* Основная версия, которая используется, если в запросе шифрования или расшифровки не указана конкретная версия.
* Алгоритм шифрования. Каждая новая версия ключа использует выбранный алгоритм — AES-128, AES-192 или AES-256 (в режиме AES-GCM).
* Период ротации - промежуток времени между автоматическими ротациями ключа.


### Ротация {#rotation}

_Ротация_ — это генерация новой версии ключа, которая сразу назначается основной версией. Ротацию можно проводить вручную, или автоматически, задав период ротации для ключа.

Ротация — это единственный способ создать новую версию ключа. Если вы хотите использовать новую версию по умолчанию, [назначьте другую версию основной](../operations/version.md#make-primary).


## Версия {#version}

Версия ключа — это непосредственно криптографический материал, который можно использовать для шифрования и расшифровки данных. У каждого ключа всегда есть версия по умолчанию, которая используется, если в запросе не задана конкретная версия. Версию по умолчанию нельзя удалить.

Для любой другой версии ключа удаление можно запланировать: по истечении указанного срока {{kms-name}} безвозвратно удалит версию, и ее нельзя будет использовать для шифрования.

{% note alert %}

Если вы удалите версию ключа, то все данные, зашифрованные с ее помощью, невозможно будет расшифровать — удаление версии на практике равносильно удалению данных.

{% endnote %}
