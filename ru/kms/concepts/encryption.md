# Применимые понятия криптографии

{{ kms-name }} помогает управлять ключами шифрования, но чтобы пользоваться сервисом полноценно стоит познакомиться с некоторыми понятиями криптографии.


## Симметричные ключи {#symmetric-keys}

Симметричным ключом называется ключ, который применяется и для шифрования, и для расшифровки данных. На этапе Preview сервис {{ kms-short-name }} поддерживает работу только с симметричными ключами.


## Двойное шифрование (envelope encryption) {#envelope-encryption}

Двойное шифрование (англ. envelope encryption) — практика двух двух видов ключей шифрования, которая упрощает и повышает безопасность данных: массив данных шифруется локально с помощью _ключа шифрования данных_, а этот ключ в свою очередь шифруется симметричным ключом {{ kms-short-name }}.

В результате ключи шифрования данных можно хранить вместе с данными: для расшифровки данных нужно сначала расшифровать ключ, с помощью которого они были зашифрованы. Таким образом контроль над ключами шифрования ключей дает возможность управлять доступом к зашифрованным данным.


## AAD-контекст {#aad}

{{ kms-name }} поддерживает дополнительные проверки при расшифровке ключами {{ kms-short-name }}: вы можете передать AAD-контекст (англ. Additional Authentication Data) в виде строки при запросе шифрования. Запрос на расшифровку таким образом зашифрованных данных должен включать указанный при шифровании AAD-контекст в качестве параметра.
