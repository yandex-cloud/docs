# Логи контейнера

Контейнер пишет логи в {{ cloud-logging-full-name }}, в [лог-группу по умолчанию](../../logging/concepts/log-group.md) для каталога, в котором находится контейнер.

Существует два вида логов:
* автоматические — логи запросов к контейнерам.
* пользовательские — логи, которые пишет пользовательский код в стандартный поток вывода (`stdout`) и стандартный поток вывода ошибок (`stderr`). Пользовательские логи имеют минимальный уровень логирования `UNSPECIFIED`.

Подробнее о работе с логами см. в [документации {{ cloud-logging-full-name }}](../../logging/).

Структурированные логи можно использовать при [одновременных вызовах экземпляра контейнера](container.md#concurrency), чтобы записывать в логи идентификаторы всех вызовов, которые обрабатывал контейнер, а не только последнего из них. Для этого необходимо указать в поле `request_id` уникальный идентификатор вызова. При [режиме работы](container.md#runtime) `HTTP-сервер` он передается в контейнер в заголовке `X-Request-Id`.
