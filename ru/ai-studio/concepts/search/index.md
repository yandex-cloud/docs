# Обзор технологий поиска {{ ai-search-name }}

{{ ai-search-name }} — это набор инструментов и технологий {{ ai-studio-name }}, которые позволяют моделям генерировать ответы на основе проверенных данных, например корпоративных документов, внутренних баз знаний или информации из интернета.

{{ ai-search-name }} предоставляет инструменты для поиска по двум типам источников:

* собственные данные пользователя (документы, инструкции, FAQ) — [инструмент File Search](../agents/tools/filesearch.md);
* выбранные домены в интернете — [инструмент Web Search](../agents/tools/websearch.md).

Вы можете использовать оба инструмента при создании [голосовых агентов](../agents/realtime.md) с помощью {{ realtime-api }} или в {{ responses-api }} при создании текстовых агентов и обращений к моделям генерации текста, чтобы формировать точные, актуальные и проверяемые ответы. Оба инструмента могут быть подключены одновременно, но модель сама решает, что нужно использовать, в зависимости от запроса пользователя, описания инструментов и заданного промпта.

## Как работает {{ ai-search-name }} {#how-aisearch-works}

Модели могут использовать для генерации текста только ту информацию, которая была заложена в них при обучении или передана в контексте запроса. Поэтому, чтобы выбранная вами модель из {{ model-gallery-name }} могла сгенерировать ответ с учетом вашей информации, данные необходимо подготовить и передать в контексте запроса. Обогащение контекста выполняется в несколько этапов:

1. Индексация данных. По умолчанию {{ ai-studio-name }} автоматически подготавливает данные для поиска. Вам необходимо только загрузить файлы в консоли управления или через Files API и создать поисковый индекс {{ vector-store-name }}. После этого {{ ai-studio-name }} разобьет данные на чанки (chunks)  нужного размера — фрагменты текста от нескольких строк до нескольких абзацев — а затем [токенизирует](../generation/tokens.md) их и сохранит в поисковом индексе.

   Чтобы исключить возможную потерю смысла при подготовке файлов, вы можете [самостоятельно разбить](../../operations/agents/create-prechunked-search-agent.md) данные на чанки и загрузить их в {{ ai-studio-name }} в формате [JSONL](https://jsonlines.org/).

1. Формирование поискового запроса. В зависимости от поставленной задачи и описанных в промпте условий использования инструментов поиска модель формулирует запрос к инструментам поиска по файлам или поиска в интернете.

1. Получение найденных фрагментов. Использованный инструмент возвращает наиболее релевантную информацию из поискового индекса в виде чанков и добавляет ее в контекст модели.

1. Генерация ответа. Модель использует найденную информацию как факты для ответа, сохраняя тональность, стиль и инструкции из промпта.
