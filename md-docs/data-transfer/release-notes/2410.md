# История изменений Data Transfer в октябре 2024

## Новые возможности {#new-features}

1. Поддержана настройка Debezium `tombstones.on.delete` для поставки данных в [Managed Service for Apache Kafka®](../operations/endpoint/target/kafka.md).

1. Поддержана версия 8.4 для источников данных [MySQL®](../operations/endpoint/source/mysql.md). 

1. Поддержан формат данных UTF-16 для репликации из источников [MySQL®](../operations/endpoint/source/mysql.md).

1. Для приемника [Apache Kafka®](../operations/endpoint/target/kafka.md) автоматически показывается максимально возможный размер сообщения.

## Решенные проблемы {#problems-solved}

1. Поддержан символ `-` в схеме для технических таблиц [PostgreSQL].

1. Устранена ошибка [трансформации данных](../concepts/data-transformation.md).


1. Повышена безопасность виртуальных машин, на которых работают трансферы.

1. Улучшена работа [диаграммы состояния трансфера](../operations/monitoring.md).
