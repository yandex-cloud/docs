# Аудитные логи и логи входа


В {{ org-full-name }} вы можете посмотреть логи входа и аудитные логи пользователей. Логи формируются на основе событий [{{ at-full-name }}](../../audit-trails/index.md) и связаны с ними через идентификаторы, но включают только некоторые типы событий с ограниченным объемом данных для каждого события.

Для удобства на страницах просмотра логов доступны фильтры:

* по времени;
* по IP-адресу;
* по типу и статусу операции или входа;
* по логину или идентификатору пользователя или операции.

Права на просмотр [логов](logs.md) пользователя зависят от его [роли](../security/index.md) в [организации](organization.md) или [пуле пользователей](user-pools.md):

* `organization-manager.viewer`, `organization-manager.editor`, `organization-manager.admin` или `organization-manager.organizations.owner` — логи любого пользователя организации на [странице пользователя](../operations/user-pools/user-get-logs.md#organization);
* `organization-manager.userpools.admin` и выше — логи всех пользователей пула на [странице в пуле](../operations/user-pools/user-get-logs.md#pool);
* остальные пользователи — только свои логи в портале [Мой аккаунт](my-account.md).

## Логи входа {#auth-logs}

_Логи входа_ отображают события аутентификации в приложениях, связанных с организацией (консоль {{ yandex-cloud }}, [{{ src-full-name }}]({{ link-src-docs }})), а также [внешних приложениях](applications.md), для которых настроен вход по SSO.

## Аудитные логи {#audit-logs}

_Аудитные логи_ формируются на основе событий, связанных с действиями пользователей в организации.

Полный список событий, поддерживаемых аудитными логами, можно [посмотреть](../operations/user-pools/user-get-logs.md) на странице пользователя в фильтре **{{ ui-key.yacloud_org.organization.users.Filters.placeholder_eu3h7 }}**.
