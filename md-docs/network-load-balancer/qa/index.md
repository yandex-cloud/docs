# Общие вопросы про {{ network-load-balancer-name }}

#### Я могу получить логи моей работы в сервисах? {#logs}

Да, вы можете запросить информацию о работе с вашими ресурсами из логов сервисов {{ yandex-cloud }}. Для этого обратитесь в [техническую поддержку]({{ link-console-support }}).

#### Могу ли я защитить сетевой балансировщик от случайного удаления? {#deletion-protection}

Да, сетевой балансировщик можно [защитить от случайного удаления](../operations/deletion-protection.md).

#### Что означает «обработка входящего/исходящего трафика балансировщиком»? {#traffic-processing}

Под «обработкой трафика» в {{ network-load-balancer-name }} в правилах тарификации понимается объем данных, который проходит через [обработчики трафика балансировщика](../concepts/listener.md). 

При этом «обработка трафика» и «трафик передачи данных» — две разные сущности. Если у балансировщика настроен публичный IP-адрес, то параллельно тарифицируется: обработка трафика балансировщиком по правилам [тарификации NLB](../../vpc/pricing.md#prices-traffic) и передача данных из облака в интернет по [правилам тарификации исходящего трафика](../../vpc/pricing.md#prices-traffic). 

Для входящего трафика ситуация отличается: входящий трафик из интернета не тарифицируется, поэтому применяются только правила тарификации NLB.
