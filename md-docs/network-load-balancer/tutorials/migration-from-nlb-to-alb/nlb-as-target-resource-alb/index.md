# Миграция сервисов с внешнего балансировщика NLB на L7-балансировщик ALB с целевым ресурсом — внутренним сетевым балансировщиком NLB

# Миграция сервисов с внешнего балансировщика NLB на L7-балансировщик ALB с целевым ресурсом — внутренним сетевым балансировщиком NLB


Вы можете мигрировать нагрузку с балансировщика [Yandex Network Load Balancer](../../../index.md) на L7-балансировщик [Yandex Application Load Balancer](../../../../application-load-balancer/index.md). Сетевой балансировщик распределяет трафик на поды вашего балансировщика, развернутые в кластере [Yandex Managed Service for Kubernetes](../../../../managed-kubernetes/index.md). Если функциональность, настроенная в вашем балансировщике, не поддерживается в L7-балансировщике, то вы можете добавить [внутренний сетевой балансировщик](../../../concepts/nlb-types.md) и подключить его в качестве целевого ресурса для L7-балансировщика. Внутренний сетевой балансировщик будет распределять трафик на поды [Ingress-контроллера NGINX](https://kubernetes.github.io/ingress-nginx/), развернутые в кластере Managed Service for Kubernetes, а в целевой группе L7-балансировщика будет указан IP-адрес внутреннего сетевого балансировщика.

{% note warning %}

Не изменяйте и не удаляйте балансировщик нагрузки и его дочерние ресурсы, созданные с помощью Managed Service for Kubernetes, через интерфейсы Yandex Cloud (консоль управления, Terraform, CLI и API). Это может привести к некорректной работе кластера.

{% endnote %}

В процессе миграции к L7-балансировщику будет подключен профиль безопасности [Yandex Smart Web Security](../../../../smartwebsecurity/index.md). Схема работы L7-балансировщика с подключенным профилем безопасности:

Вы можете создать инфраструктуру для миграции сервисов с помощью следующих инструментов:

* [Консоль управления](console.md) — используйте консоль управления Yandex Cloud, чтобы создать инфраструктуру в пошаговом режиме.
* [Terraform](terraform.md) — используйте этот способ, чтобы упростить создание ресурсов и управление ими, используя подход «инфраструктура как код» (IaC). Скачайте пример конфигурации Terraform, а затем разверните инфраструктуру с помощью [Terraform-провайдера Yandex Cloud](../../../../terraform/index.md).
