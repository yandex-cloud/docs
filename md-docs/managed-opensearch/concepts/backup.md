# Резервные копии в Managed Service for OpenSearch

Managed Service for OpenSearch обеспечивает автоматическое и ручное резервное копирование [индексов](indexing.md).

Для кластеров с неподдерживаемой [версией СУБД](update-policy.md#versioning-policy) восстановление из резервных копий недоступно.

## Создание резервной копии {#size}

Резервные копии создаются только на работающих кластерах.

Автоматическое резервное копирование выполняется каждый час, и отключить его нельзя.

О том, как вручную создать резервную копию, читайте в разделе [Управление резервными копиями](../operations/cluster-backups.md).

## Хранение резервной копии {#storage}

Особенности хранения резервных копий в Managed Service for OpenSearch:

* Все автоматические и [созданные вручную](../operations/cluster-backups.md) резервные копии — инкрементные. Это значит, что стартовая резервная копия содержит все сегменты индекса, а все последующие резервные копии содержат только _инкремент_ — разницу с предыдущей резервной копией. Это позволяет экономить место в хранилище и снижает стоимость использования ресурсов.

* При [получении информации о резервной копии](../operations/cluster-backups.md#get-backup) отображается полный размер данных, которые можно восстановить из нее, но тарифицируется только размер инкремента.

* Резервные копии хранятся в объектном хранилище в виде бинарных файлов.

* После создания резервная копия сжимается для дальнейшего хранения.

* Автоматические резервные копии хранятся две недели и изменить этот срок нельзя. Созданные вручную резервные копии хранятся бессрочно, до момента их удаления пользователем.

* На хранилище резервных копий не распространяется действие [квот](https://console.yandex.cloud/cloud?section=quotas) и лимитов для хранилища кластера.

* Резервные копии содержатся в объектном хранилище и не занимают место в хранилище кластера. При этом если в кластере есть N свободных гигабайт места, то хранение первых N гигабайт резервных копий не тарифицируется.

    
    Подробнее в разделе [Правила тарификации для Managed Service for OpenSearch](../pricing.md#rules-storage).


## Восстановление из резервной копии {#capabilities}

Восстанавливая кластер из резервной копии, вы создаете новый кластер с данными из резервной копии. При этом понадобится задать все его параметры, как и при обычном создании кластера. Если в каталоге не хватает [ресурсов](limits.md) для создания такого кластера, восстановиться из резервной копии не получится.

Чтобы восстановить кластер из резервной копии, следуйте [инструкциям](../operations/cluster-backups.md).
