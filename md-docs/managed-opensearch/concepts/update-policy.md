# Политика работы с версиями OpenSearch

## Политика поддержки версий {#versioning-policy}

Managed Service for OpenSearch работает только с теми версиями OpenSearch, которые поддерживает производитель. Информацию о сроках прекращения поддержки отдельных версий см. в [календаре производителя](https://opensearch.org/releases.html).

Managed Service for OpenSearch использует два типа версий OpenSearch:

- **Мажорные**.

  Поддерживаются две мажорные версии: текущая и предыдущая. При релизе новой мажорной версии в Managed Service for OpenSearch прекращается поддержка самой старой из доступных мажорных версий.

  Исключение — мажорная версия 1, она не поддерживается. До выхода третьей мажорной версии поддерживается только вторая мажорная версия.

- **Минорные**.

  Поддерживается последняя минорная версия OpenSearch для каждой доступной мажорной версии. При релизе новой минорной версии OpenSearch в Managed Service for OpenSearch поддержка предыдущей минорной версии прекращается.

{% note warning %}

* Для кластеров с неподдерживаемой версией OpenSearch создание новых хостов и восстановление из резервных копий недоступно.
* Кластеры с неподдерживаемой версией OpenSearch будут автоматически обновлены в соответствии с [политикой обновления](#update-policy).

{% endnote %}

## Политика обновления {#update-policy}

{% list tabs %}

- Мажорные версии

  Кластер Managed Service for OpenSearch на неподдерживаемой версии OpenSearch будет обновлен до следующей мажорной версии (отмечена как LTS).
  
  ![image](../../_assets/mdb/mos-update-policy-lts.svg)

- Минорные версии

  Кластер Managed Service for OpenSearch на неподдерживаемой версии OpenSearch будет обновлен либо до мажорной версии, если она доступна (отмечена как LTS), либо до самой новой минорной версии.
  
  ![image](../../_assets/mdb/mos-update-policy.svg)

{% endlist %}

На почту владельца кластеров заранее приходит уведомление о планируемом автоматическом обновлении. Когда приближается срок прекращения поддержки используемой версии OpenSearch, рекомендуется самостоятельно обновить кластер до более новой версии.
