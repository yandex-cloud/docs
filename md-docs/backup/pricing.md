# Правила тарификации для Yandex Cloud Backup


Чтобы рассчитать стоимость использования сервиса, ознакомьтесь с тарифами в этом разделе.

Цены на продукты сервиса также доступны в [Прайс-листе](https://yandex.cloud/ru/price-list).


Все цены в рублях и тенге указаны с НДС, все цены в долларах — без НДС.

## Из чего складывается стоимость использования Cloud Backup {#rules}

Стоимость Cloud Backup зависит от количества защищенных ресурсных единиц — виртуальных машин Yandex Compute Cloud и серверов Yandex BareMetal, а также суммарного объема хранилища, занятого резервными копиями.

### Защита ресурсных единиц {#vms}

ВМ Compute Cloud или сервер BareMetal начинают тарифицироваться в Cloud Backup после того, как они были привязаны к [политике резервного копирования](concepts/policy.md). Плата продолжает взиматься, пока ресурсная единица не будет отвязана от политики независимо от статуса ресурсной единицы.

Если вы удалите ВМ в [Yandex Compute Cloud](../compute/index.md) с помощью [консоли управления](https://console.yandex.cloud), ВМ также будет отвязана от всех политик. Если вы удалите ВМ с помощью CLI, Terraform или API, она не будет автоматически отвязана от политик, сделайте это самостоятельно.

Сервер BareMetal также нужно отвязывать от политик самостоятельно.

Минимальная единица тарификации — 1 защищенная ресурсная единица (ВМ или сервер BareMetal) в час.

### Использование хранилища {#backups}

Плата взимается за суммарный объем хранилища, занимаемый резервными копиями.

Минимальная единица тарификации — час хранения 1 МБ данных.

Во всех расчетах 1 ГБ = 2<sup>30</sup> байт, 1 МБ = 2<sup>20</sup> байт.

Если защищаемая ресурсная единица была остановлена или удалена, ее резервные копии продолжают храниться в Cloud Backup, и за их объем продолжает взиматься плата. На размер резервных копий влияют:
* заполненность диска ВМ или сервера BareMetal;
* количество измененных данных при регулярном резервном копировании;
* возможность сжатия данных.

{% note info %}

Чтобы сократить затраты, удалите ненужные резервные копии удаленных ВМ или серверов BareMetal, от аренды которых вы отказались.

{% endnote %}

Объем резервных копий защищаемой ресурсной единицы может быть как меньше размера диска самой ресурсной единицы, например, при малой заполненности диска и хорошей сжимаемости данных, так и больше, например, когда резервных копий много, а данные на них постоянно меняются и плохо сжимаются.

## Пример расчета стоимости {#price-example}

Пример расчета стоимости Cloud Backup за один месяц для следующей конфигурации:
* к политикам резервного копирования привязана 1 ВМ;
* суммарный объем резервных копий составляет 50 ГБ.


{% list tabs group=pricing %}

- Расчет в рублях {#prices-rub}

  > 1 ВМ × 280,8 ₽ + 50 ГБ × 4,968 ₽ = {% calc [currency=RUB] 1 × 280.8 + 50 × 4.968 %}
  
  > Итого: {% calc [currency=RUB] 1 × 280.8 + 50 × 4.968 %} — стоимость использования Cloud Backup для защиты 1 ВМ с суммарным размером резервных копий 50 ГБ за один месяц, вкл. НДС.
  
  Где:
  * 1 — количество ВМ, привязанных к политикам резервного копирования.
  * 280,8 ₽ — цена за подключение 1 ВМ к Cloud Backup в месяц, вкл. НДС.
  * 50 ГБ — суммарный объем резервных копий.
  * 4,968 ₽ — цена за хранение 1 ГБ резервных копий в месяц, вкл. НДС.

  * 1 — количество ВМ, привязанных к политикам резервного копирования.
