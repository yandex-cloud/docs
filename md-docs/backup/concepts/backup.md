# Резервные копии

{% note info %}

В момент создания резервной копии виртуальная машина или сервер BareMetal должны быть запущены.

{% endnote %}


## Типы копий {#types}

* _Полная копия_ содержит все данные, необходимые для восстановления ВМ или сервера Yandex BareMetal. Из полных копий ВМ и серверы BareMetal восстанавливаются быстрее, чем из инкрементальных, но такие копии занимают больше места в хранилище и создаются дольше.
* _Инкрементальная копия_ содержит только те данные, которые отличаются от предыдущей копии ВМ или сервера BareMetal. Из инкрементальных копий ВМ и серверы восстанавливаются дольше, чем из полных копий, поскольку при восстановлении используются цепочки инкрементальных копий. Такие копии создаются быстрее и занимают меньше места, чем полные. Невыгодно использовать инкрементальное копирование, если между двумя последовательными копиями накапливается много различий.

{% note info %}

Если вы используете политику с [опцией](policy.md#specification) быстрого резервного копирования `fastBackupEnabled`, и перед созданием [инкрементальной копии](backup.md#types) выполнялась антивирусная проверка, время резервного копирования может значительно увеличиться. Подробнее в [Взаимодействие с антивирусом](av-interaction.md).

{% endnote %}

Восстановить резервную копию ВМ на сервер BareMetal или резервную копию сервера BareMetal на ВМ невозможно.

{% note tip %}

Чтобы использовать преимущества обоих типов, сочетайте их. Например, создайте для одной ВМ или сервера BareMetal две [политики копирования](policy.md): в одной настройте создание полных копий раз в неделю, а в другой — создание инкрементальных копий раз в день.

{% endnote %}

### Особенности работы с LVM {#lvm}

Cloud Backup позволяет создавать резервные копии виртуальных машин и серверов BareMetal с логическими томами под управлением подсистемы [LVM](https://ru.wikipedia.org/wiki/LVM).

Восстановление ВМ и серверов BareMetal из таких резервных копий необходимо выполнять с учетом следующих особенностей:

{% list tabs group=backup_resource_type %}

- Виртуальная машина {#vm}

  * При восстановлении резервной копии на ту же ВМ, для которой эта резервная копия была создана, дополнительных действий не требуется: структура LVM-томов должна восстановиться без ошибок.
  * При восстановлении резервной копии на другую ВМ перед восстановлением необходимо создать на целевой ВМ в точности такую же разметку дисков и структуру томов LVM, какая была на исходной ВМ.

      {% note warning %}

      При восстановлении резервной копии с LVM-томами на виртуальную машину, отличную от исходной, успешное восстановление структуры LVM-томов не гарантируется.

      {% endnote %}

- Сервер BareMetal {#baremetal-server}

  * При восстановлении резервной копии на тот же сервер, для которого эта резервная копия была создана, дополнительных действий не требуется: структура LVM-томов должна восстановиться без ошибок.
  * При восстановлении резервной копии на другой сервер, отличный от исходного, [запросите](https://center.yandex.cloud/support/tickets/create) в службе технической поддержки специальный загрузочный [ISO-образ](https://ru.wikipedia.org/wiki/ISO-образ) и загрузите целевой сервер с этого образа.

{% endlist %}

## Хранение копий {#storage}

Резервные копии, создаваемые в Cloud Backup, хранятся во внутренних бакетах хранилища Yandex Object Storage. 

Надежность хранения данных в Object Storage соответствует международным и национальным стандартам. Подробнее в разделе [Соответствие требованиям](../../security/conform.md) документа о безопасности в Yandex Cloud.

Получить доступ к файлам копий и бакетам, в которых они хранятся, нельзя. Восстановить ВМ или сервер BareMetal из копии можно только средствами Cloud Backup.

### Архивы резервных копий {#archives}

Резервные копии объединяются в архивы. _Архив резервных копий_ — это совокупность всех резервных копий, созданных для одной ВМ или одного сервера BareMetal в рамках одной политики резервного копирования.

Использование архивов позволяет быстрее [удалять](../operations/backup-vm/batch-delete.md) резервные копии неактуальных ресурсов. Кроме того, удаление резервных копий целым архивом в меньшей степени нагружает [агент Cloud Backup](agent.md) и менее подвержено возникновению ошибок, чем удаление резервных копий по отдельности.

## Пофайловое восстановление {#file-by-file}

C помощью Cloud Backup вы можете восстановить из резервной копии отдельные файлы и директории на любые из ВМ, [подключенных](vm-connection.md) к сервису. Подробнее в разделе [Восстановить на виртуальной машине или сервере BareMetal отдельные директории и файлы](../operations/backup-vm/recover-file-by-file.md).

Пофайловое восстановление на серверы BareMetal в настоящий момент невозможно.
