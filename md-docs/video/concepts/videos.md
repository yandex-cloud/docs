# Видео в сервисе Cloud Video

С помощью Cloud Video вы можете загружать на [канал](index.md#channels) _видео_ для дальнейшей публикации на внешних ресурсах.

Поддерживается загрузка видео в различных форматах, например: [MP4](https://ru.wikipedia.org/wiki/MPEG-4_Part_14), [AVI](https://ru.wikipedia.org/wiki/Audio_Video_Interleave), [MKV](https://ru.wikipedia.org/wiki/Matroska), [FLV](https://ru.wikipedia.org/wiki/Flash_Video), [MOV](https://en.wikipedia.org/wiki/QuickTime_File_Format), [WebM](https://ru.wikipedia.org/wiki/WebM) и других. Вы можете [загружать](../operations/video/upload.md) видео как по одному файлу, так и по несколько файлов сразу.

Вы можете загрузить видео с несколькими аудиодорожками или добавить их после загрузки с помощью [нейроперевода](videos.md#translation). Дорожки выбираются в меню [плеера](player.md), подробнее см. в разделе [Выбрать аудиодорожку](../operations/player-control.md#audio).

В сервисе действуют [ограничения](limits.md) на разрешение публикуемых видео.

Для видео доступна загрузка пользовательских обложек. Обложка будет отображаться в интерфейсе Cloud Video, а также в [плеере](player.md) на сайте, где размещено видео.

Поддерживаются обложки в следующих форматах: [JPG](https://ru.wikipedia.org/wiki/JPEG), [PNG](https://ru.wikipedia.org/wiki/PNG) и [GIF](https://ru.wikipedia.org/wiki/GIF).

Вы можете [опубликовать](../operations/video/get-link.md) видео и выбрать тип доступа:

* **По временной ссылке** — видео будет доступно по специальной ссылке. {#temporary-link}

    Видео по временной ссылке доступно не более 12 часов.

* **Для всех пользователей** — видео будет доступно неограниченное время и всем, у кого есть ссылка на видео. {#permanent-link}

На сайте можно разместить как одиночные видео, так и [плейлисты](playlists.md) с видео в определенной последовательности. В плейлистах будут доступны к просмотру только видео с постоянными ссылками. Подробнее о публикации плейлистов см. в разделе [Получить код вставки или ссылку на плейлист](../operations/playlists/get-link.md).

## Статусы {#statuses}

### Статусы видео {#video-statuses}

* `Uploading` — исходный файл с видео загружается в хранилище Yandex Cloud.
* `Processing` — исходный файл с видео транскодируется в несколько вариантов видео с различным битрейтом и разрешением. При дальнейшем просмотре видео на клиентских устройствах видеоплеер подбирает оптимальный вариант для плавного воспроизведения при текущей скорости интернет-соединения.
* `Ready` — транскодирование выполнено, видео готово к просмотру.
* `Error` — ошибка при загрузке файла или транскодировании видео. Проверьте стабильность интернет-соединения, целостность файла, его формат и повторите попытку.

### Статусы публикации {#publication-statuses}

* `Published` — видео [опубликовано](../operations/video/publish.md).
* `Unpublished` — видео не опубликовано.

## Параметры публикации видео {#video-parameters}

Вы можете изменять следующие базовые настройки воспроизведения видео при [генерации](../operations/video/get-link.md) прямой ссылки или кода для вставки видео на сайт:

* Установка по умолчанию для воспроизведения звука при проигрывании.
* Автоматический запуск воспроизведения при открытии.
* Отображение элементов управления видео в плеере.

С помощью [Cloud Video Player SDK для IFrame](../sdk/iframe/index.md) вы можете задать дополнительные настройки управления видеоплеером и параметры воспроизведения видео на HTML-странице, такие как размер и зацикливание видео, уровень звука и др.

Также задать некоторые параметры плеера, например, цветовую палитру и расположение виджетов, можно с помощью [шаблонов](presets.md).

## Статистика просмотров видео {#video-statistics}

Для каждого видео вы можете [посмотреть статистику](../operations/video/get-statistics.md) просмотров с возможностью гибко настроить период расчета статистики.

В настоящее время доступны следующие статистические показатели просмотров:

* Количество просмотров — целое число.
* Доля просмотренного контента — процентное соотношение средней длительности просмотра к длительности видео.
* Общее время просмотра — суммарная длительность просмотров в часах и минутах.
* Средняя длительность просмотра — средняя длительность просмотра в часах и минутах.
* Линейная диаграмма с распределением количества просмотров на единицу времени со следующими уровнями дискретизации:

    * `5 минут`
    * `1 час`
    * `1 день`
* Круговая диаграмма с распределением просмотров по типам операционных систем устройств.
* Круговая диаграмма с распределением просмотров по типам устройств.
* [Тепловая карта](https://ru.wikipedia.org/wiki/Тепловая_карта) с данными о просмотрах фрагментов видео.

    Каждая точка на тепловой карте соответствует 30-секундному фрагменту видео.

    Просмотром на тепловой карте считается факт воспроизведения фрагмента видео, независимо от продолжительности воспроизведения. Например, если вы запустили видео и сразу же остановили его, то для первого фрагмента видео будет засчитан просмотр.

## ИИ-возможности {#ai-capabilities}

Чтобы предложить пользователям больше возможностей и увеличить аудиторию ваших видео, включите нейросетевые функции, построенные на ML-моделях Яндекса. Подробнее о работе нейросетевых функций см. в [блоге Яндекса](https://habr.com/ru/companies/yandex/articles/792608/). 

### Суммаризация {#summarization}

[Суммаризация](../operations/video/summarization.md#enable) позволяет автоматически создавать краткое содержание видео, которое передает его основные моменты и ключевую информацию. Суммаризация доступна для видео длительностью не более 4 часов. При включении суммаризации к видео автоматически добавляются таймкоды. Вы можете скорректировать предложенные таймкоды или [добавить свои вручную](#timecodes).

### Нейроперевод {#stranslation}

[Нейроперевод](../operations/video/translation.md) позволяет автоматически переводить видео на несколько языков и озвучивать его [разными голосами](https://habr.com/ru/news/905118/). Список поддерживаемых языков см. в [интерфейсе Cloud Video](https://video.yandex.cloud/).

### Субтитры {#subtitles}

Нейросетевые [субтитры](../operations/video/subtitles.md) помимо текста, который произносится в видео, содержат разделение на спикеров. Речь нового спикера будет начинаться с новой строки и тире, что повышает удобство восприятия речи. Список поддерживаемых языков см. в [интерфейсе Cloud Video](https://video.yandex.cloud/).

Также можно загрузить свои субтитры в форматах SRT и VTT. Максимальный размер файла субтитров — 500 МБ. Для одного видео вы можете загрузить несколько файлов субтитров на одном или разных языках.

## Таймкоды {#timecodes}

[Таймкоды](../operations/video/add-timecodes.md) позволяют разбить видео на главы, по которым можно переходить при просмотре видео. Время и название глав задается в следующем формате:

```
00:00:00 Название главы 1
hh:mm:ss Название главы 2
 ...
```

{% note info %}

Таймкоды должны начинаться с `00:00` или `00:00:00`. Время от названия главы отделяется пробелом. Каждый таймкод начинается с новой строки.

{% endnote %}

Также таймкоды можно сформировать автоматически, включив [суммаризацию](#summarization) видео.
