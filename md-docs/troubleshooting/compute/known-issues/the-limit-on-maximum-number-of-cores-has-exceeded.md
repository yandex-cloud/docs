# Устранение ошибки `The limit on maximum number of cores has exceeded`


## Описание проблемы {#issue-description}

При попытке создания виртуальной машины в Compute Cloud или группы узлов Managed Service for Kubernetes появляется сообщение `The limit on maximum number of cores has exceeded`.

## Решение {#issue-resolution}

Эта ошибка означает, что в вашем облаке превышена квота **Количество vCPU виртуальных машин**. Запросите ее повышение через [соответствующую форму](https://console.yandex.cloud/cloud?section=quotas).

Для создания и использования новых виртуальных машин Compute Cloud или узлов Managed Service for Kubernetes вам также может понадобиться повысить и другие квоты, такие как:

* **Общий объем HDD-дисков**;
* **Общий объем SSD-дисков**;
* **Общий объем снимков дисков**;
* **Количество дисков**;
* **Общий объем RAM виртуальных машин**;
* **Количество виртуальных машин**.

Обратите внимание на текущий уровень потребления по этим квотам.

## Если проблема осталась {#if-issue-still-persists}

Если вышеописанные действия не помогли решить проблему, [создайте запрос в техническую поддержку](https://center.yandex.cloud/support). При создании запроса укажите следующую информацию:

1. Идентификатор операции создания новой виртуальной машины или группы узлов Managed Service for Kubernetes, которая завершилась с ошибкой.
1. Текст сообщения об ошибке, получаемой при попытке создания новых объектов.
