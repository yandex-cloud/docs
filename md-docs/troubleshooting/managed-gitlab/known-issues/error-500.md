# Устранение ошибки HTTP 500 в Managed Service for GitLab


## Описание проблемы {#issue-description}

При попытке открыть инстанс Managed Service for GitLab отображается сообщение об ошибке `HTTP 500` или `HTTP 502`.

## Решение {#issue-resolution}

Наиболее частая причина этой проблемы заключается в исчерпании инстансом Managed Service for GitLab свободного места на диске. Значительное дисковое пространство могут занимать образы контейнеров и сборочные артефакты.

Чтобы отслеживать количество оставшегося дискового пространства на инстансе Managed Service for GitLab и исключить вероятность его исчерпания в дальнейшем, рекомендуем настроить мониторинг состояния инстанса Managed Service for GitLab [по этой инструкции](../../../managed-gitlab/operations/instance/monitoring.md) (функциональность находится [на стадии Preview](../../../overview/concepts/launch-stages.md)).

{% note info %}

При создании инстанса Managed Service for GitLab по умолчанию включена опция **Удалять образы без тегов при обслуживании**. Она позволяет выполнять периодическое удаление тех образов, на которые не ссылаются сборочные манифесты.

{% endnote %}

Для решения проблемы следует выполнить очистку раздела диска [по этой инструкции](../../../managed-gitlab/operations/instance/clean-up-disk-space.md).
Также вы можете [увеличить размер диска Managed Service for GitLab](../../../managed-gitlab/operations/instance/instance-update.md) инстанса.

Если это не поможет решить проблему – понадобится [создать запрос в техническую поддержку](https://center.yandex.cloud/support). 
При создании запроса укажите идентификатор проблемного инстанса Managed Service for GitLab и скриншот с деталями ошибки.
