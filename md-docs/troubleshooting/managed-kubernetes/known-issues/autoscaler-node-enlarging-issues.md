# Не увеличивается количество узлов в группе Managed Service for Kubernetes


## Описание проблемы {#issue-description}

Увеличение количества узлов в группе Managed Service for Kubernetes не происходит автоматически после увеличения параметров **Минимальное кол-во узлов** и **Начальное кол-во узлов**.

## Решение {#issue-resolution}

Изменение значений параметров **Минимальное кол-во узлов** и **Начальное кол-во узлов** не приводит к созданию новых узлов в группе узлов Managed Service for Kubernetes.

Чтобы новые узлы создались, нужно подать на кластер нагрузку, то есть нагрузить кластер работающими и потребляющими ресурсы деплойментами. После этого Cluster Autoscaler сможет создать новые узлы и поддерживать их количество в соответствии с измененными параметрами. Затем тестовую нагрузку можно снять.

## Если проблема осталась {#if-issue-still-persists}

Если вышеописанные действия не помогли решить проблему, [создайте запрос в техническую поддержку](https://center.yandex.cloud/support). При создании запроса укажите следующую информацию:

1. Идентификатор кластера Managed Service for Kubernetes.
1. Описание желаемого состояния кластера Managed Service for Kubernetes, например ожидаемое количество узлов в группе.
