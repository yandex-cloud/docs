# Как перенести работу в новую версию DataSphere

Вместе с интерфейсом в DataSphere изменилась и [ресурсная модель](../concepts/resource-model.md). Чтобы продолжить работу над проектами, созданными в старой версии DataSphere, перенесите их в новую версию [автоматически](#auto).

## Перенести проекты и данные автоматически {#auto}

Вы можете автоматически перенести проекты, состояние интерпретатора, ресурсы проекта и все данные из старой версии DataSphere в новую с помощью инструмента переноса проектов. Для этого в левом боковом меню выберите ![transfer](../../_assets/console-icons/folder-arrow-right.svg) **Перенос проектов** и следуйте подсказкам в интерфейсе.

{% note warning %}

После переноса проектов в новую версию DataSphere проекты старой версии будут удалены.

{% endnote %}

{% note info %}

Автоматически перенести ноды невозможно. Если в проектах для переноса есть ноды, самостоятельно удалите их, перенесите проекты и создайте ноды заново, либо [обратитесь](https://center.yandex.cloud/support) в техническую поддержку.

{% endnote %}

1. Выберите облако и проекты, которые вы хотите перенести. Для выбора вам доступны только облака, в которых есть проекты DataSphere, и вы имеете [роль](../security/index.md) `datasphere.admin` или `editor` на каталог с этим проектом.
1. Выберите [сообщество](../concepts/community.md), в которое вы будете переносить проекты. Сообщество должно иметь активный платежный аккаунт. Чтобы перенести проекты в сообщество, вам понадобится роль `datasphere.communities.admin` в этом сообществе или роль `editor` в организации.
1. Нажмите **Перенести выбранные проекты**.

После переноса проекты будут доступны в выбранном вами сообществе.
