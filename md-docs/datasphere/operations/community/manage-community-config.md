# Управление ограничениями сообщества

{% note warning %}

После 20.04.2026 создание новых [датасетов](../../concepts/dataset.md) станет невозможным. Для хранения данных используйте [файловые хранилища](../../concepts/filestores.md). Датасеты, созданные до 20.04.2026, продолжат работать без изменений.

{% endnote %}

{% note info %}

Вы можете ограничить доступные во всех проектах сообщества ресурсы и функциональность DataSphere, только если в сообществе у вас есть роль `datasphere.communities.admin`.

{% endnote %}

1. Откройте [главную страницу](https://datasphere.yandex.cloud) DataSphere. На панели слева выберите ![community-panel](../../../_assets/console-icons/circles-concentric.svg) **Сообщества**.
1. Выберите сообщество, в котором вы хотите настроить функциональность DataSphere.
1. Перейдите на вкладку **Ограничения**.
1. В блоке **Режим работы проектов**:

    * **Подключение внешних IDE** — в проектах будет доступна возможность [работы из любой локальной IDE](../projects/remote-connect.md), поддерживающей подключение к удаленным серверам Jupyter.
    * **Максимальное количество ВМ в проекте** — количество ВМ, которые будут доступны в проектах сообщества.

1. В блоке **Ресурсы**:

    * **Изменение размера хранилища проекта** — включите опцию, если хотите разрешить изменять размер хранилища проектов.
    * **Развертывание эксплуатации моделей** — в проектах сообщества можно будет развертывать эксплуатацию сервисов с помощью [нод](../../concepts/deploy/index.md#node) и [алиасов](../../concepts/deploy/index.md#alias).
    * **Шаблоны Yandex Data&nbsp;Processing** — в проектах сообщества будут доступны [шаблоны Yandex Data Processing](../../concepts/data-processing-template.md).
    * **Датасеты** — в проектах сообщества можно будет создавать [датасеты](../../concepts/dataset.md).

1. В блоке **Конфигурации вычислительных ресурсов** вы можете выбрать [конфигурации вычислительных ресурсов](../../concepts/configurations.md), которые будут доступны во всех проектах сообщества.
