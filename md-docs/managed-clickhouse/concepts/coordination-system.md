# Сервисы координации в {{ mch-name }}

_Сервис координации_ — это сервис, который управляет согласованностью между узлами кластера, обеспечивает [репликацию](replication.md) данных и выполнение распределенных DDL-запросов.

В {{ mch-name }} доступны следующие сервисы координации:

* {{ CK }} встроенный;
* {{ CK }} на отдельных хостах;
* {{ ZK }}.

{% note warning %}

После выбора сервиса координации сменить или отключить его невозможно. Хосты обоих сервисов учитываются в расчете использованной [квоты ресурсов]({{ link-console-quotas }}) в облаке и в [расчете стоимости](../pricing.md) кластера.

{% endnote %}

## Выбор сервиса координации {#coordination-system-selection}

Выбор между сервисами [{{ CK }}](#ck) зависит от сценария использования:

* Встроенный {{ CK }} размещается на хостах {{ CH }} и подходит для тестирования или приложений с низкой нагрузкой.
* {{ CK }} на отдельных хостах подходит для стабильных версий приложений и приложений с высокой нагрузкой.

[{{ ZK }}](#zk) также подходит для стабильных версий приложений и приложений с высокой нагрузкой. Однако {{ ZK }} — это устаревающая технология, поэтому вместо него рекомендуется использовать {{ CK }} на отдельных хостах. В дальнейшем {{ ZK }} станет недоступен для новых кластеров.

## {{ CK }} {#ck}

{{ CK }} — решение {{ CH }}, которое обеспечивает согласованность чтения и записи данных. {{ CK }} реализует совместимый с {{ ZK }} клиент-серверный протокол, поэтому для взаимодействия с ним можно использовать любой стандартный клиент {{ ZK }}. Однако снимки, журналы и межсерверный протокол {{ CK }} несовместимы с {{ ZK }}, поэтому использование хостов {{ CK }} и {{ ZK }} в одном кластере невозможно.

Подробнее о {{ CK }} читайте в [документации {{ CH }}]({{ ch.docs }}{{ lang }}/guides/sre/keeper/clickhouse-keeper).

В {{ mch-name }} сервис координации {{ CK }} доступен в следующих режимах:

* **{{ ui-key.yacloud.clickhouse.cluster.value_coordination-service-embedded-clickhouse-keeper }}** — {{ CK }} размещается на хостах {{ CH }}. Для обеспечения репликации кластер должен состоять из трех или более хостов {{ CH }}.
* **{{ ui-key.yacloud.clickhouse.cluster.value_coordination-service-separated-clickhouse-keeper }}** — {{ CK }} размещается на отдельных хостах. Для обеспечения репликации кластер должен состоять из двух и более хостов {{ CH }}, а также включать три или пять хостов {{ CK }}.

  Этот режим используется по умолчанию в [консоли управления]({{ link-console-main }}) при создании кластера из двух и более хостов {{ CH }} на шард или при настройке сервиса координации.

Включить сервис координации {{ CK }} можно:
  
* при [создании кластера](../operations/cluster-create.md);
* во время [изменения кластера](../operations/update.md), если кластер создан без сервиса координации.

После включения {{ CK }} он не может быть отключен.

## {{ ZK }} {#zk}

{{ ZK }} — один из первых сервисов координации с открытым исходным кодом. В отличие от {{ CK }} он обеспечивает только согласованность записи данных, но не чтения.

Подробнее о {{ ZK }} читайте в [документации {{ ZK }}](https://zookeeper.apache.org/).

Сервис координации {{ ZK }} размещается на отдельных хостах. Для обеспечения репликации в кластере {{ mch-name }} должно быть [три или пять хостов {{ ZK }}](../qa/cluster-settings.md#zookeeper-hosts-number).

{{ ZK }} используется по умолчанию при создании кластера {{ mch-name }} в [{{ yandex-cloud }} CLI](../../cli/index.md), {{ TF }} и API.


Если вы создаете кластер из двух и более хостов {{ CH }} на шард и в [сети](../../vpc/concepts/network.md) кластера есть [подсети](../../vpc/concepts/network.md#subnet) в каждой [зоне доступности](../../overview/concepts/geo-scope.md), три хоста {{ ZK }} будут добавлены автоматически, по одному в каждую подсеть. Если подсети есть только в некоторых зонах доступности, настройки хостов {{ ZK }} нужно указывать вручную.


Минимальное количество ядер для одного хоста {{ ZK }} зависит от суммарного количества ядер хостов {{ CH }}:

| Суммарное количество ядер хостов {{ CH }} | Минимальное количество ядер для одного хоста {{ ZK }} |
|-------------------------------------------|-------------------------------------------------------|
| Менее 48                                  | 2                                                     |
| 48 и более                                | 4                                                     |

Класс хостов и размер хранилища {{ ZK }} можно изменить при [изменении настроек кластера](../operations/update.md#change-resource-preset). Изменение настроек {{ ZK }} и подключение к его хостам не поддерживается.

Включить сервис координации {{ ZK }} можно:
  
* при [создании кластера](../operations/cluster-create.md);
* во время [изменения кластера](../operations/update.md), если кластер создан без сервиса координации.

После включения {{ ZK }} он не может быть отключен.
