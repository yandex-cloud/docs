# Настройка выделенной IP-адресации CDN-ресурсов

{% note warning %}

За услугу выделенной IP-адресации взимается плата согласно [правилам тарификации Yandex Cloud CDN](../../pricing.md). Количество выделенных IP-адресов [ограничено](../../concepts/limits.md#cdn-limits).

{% endnote %}

Чтобы получить [выделенный IP-адрес](../../concepts/dedicated-ip-addressing.md), закрепленный только за [CDN-ресурсами](../../concepts/resource.md) из конкретного каталога:

{% note info %}

Выделенная IP-адресация включается для каталога целиком. При выборе схемы оплаты **за каждый CDN-ресурс отдельно** плата будет взиматься за все CDN-ресурсы каталога. Если выделенная IP-адресация нужна не для всех ресурсов, рекомендуем создать для них дополнительные каталоги.

{% endnote %}

1. Убедитесь, что в каталоге, для которого запрашивается выделенная IP-адресация, уже есть как минимум один CDN-ресурс.
1. [Направьте](https://center.yandex.cloud/support/tickets/create) обращение в службу технической поддержки Yandex Cloud.

    В обращении укажите:
    * **Сервис** — `Cloud CDN`.
    * **Тема** — `Выделенная IP-адресация CDN-ресурсов`.
    * **Текст обращения** — [идентификатор каталога](../../../resource-manager/operations/folder/get-id.md), в котором требуется включить услугу.

1. После ответа технической поддержки в созданном обращении:
    * Подтвердите согласие с [правилами тарификации выделенной IP-адресации](../../pricing.md).
    * Согласуйте дату и время переключения на выделенную IP-адресацию.

      {% note alert %}

      При подключении выделенной IP-адресации происходит обновление типа балансировки трафика, обслуживание трафика ресурсов приостанавливается. Это занимает до 15 минут.

      {% endnote %}

    * Узнайте выделенный публичный IP-адрес, который будет использоваться для CDN-ресурсов каталога.

1. (опционально) Чтобы избежать временной недоступности вашего контента при подключении выделенной IP-адресации, заранее уведите трафик с CDN-ресурсов Yandex Cloud, например, направьте трафик напрямую в [источник](../../concepts/origins.md).

    {% note info %}

    При подключении выделенной IP-адресации, а также при временном переключении трафика с CDN-ресурса [кешированный контент](../../concepts/caching.md#server-side) сохраняется на CDN-серверах.

    {% endnote %}

    Для перенаправления трафика:
    1. В настройках вашего DNS-хостинга измените CNAME-запись со значения, указывающего на доменное имя Yandex Cloud CDN, на временное, например на [FQDN](../../../glossary/fqdn.md) источника:

        <br>

        > Пример исходной записи:
        > 
        > ```text
        > cdn.example.com. CNAME e1b83ae3********.topology.gslb.yccdn.ru
        > ```
        > 
        > Пример измененной записи:
        > 
        > ```text
        > cdn.example.com. CNAME <FQDN_источника>
        > ```

        {% note tip %}
        
        Если для DNS-хостинга вы используете Yandex Cloud DNS, смотрите инструкцию [Изменить ресурсную запись](../../../dns/operations/resource-record-update.md).
        
        {% endnote %}

    1. Дождитесь, пока обновленная CNAME-запись распространится по серверам DNS. На это может потребоваться несколько часов.
    1. Чтобы убедиться, что на CDN-ресурс больше не поступают запросы, [проверьте](get-stats.md) статистику ресурса.

1. Дождитесь сообщения от технической поддержки о том, что выделенная IP-адресация подключена. Проверьте, что доменному имени Yandex Cloud CDN вашего CDN-ресурса соответствует выделенный IP-адрес, указанный в обращении поддержки, например:

    ```bash
    host e1b83ae3********.topology.gslb.yccdn.ru
    ```

    Результат:

    ```bash
    e1b83ae3********.topology.gslb.yccdn.ru has address 188.72.111.11
    ```

    С этого момента начинается взиматься плата за использование выделенной IP-адресации.

1. Если вы уводили трафик с CDN-ресурса, восстановите исходную CNAME-запись.
