# Точки присутствия

_Точки присутствия_ (Points of Presence, POP) — это географически распределенные физические CDN-серверы, которые кешируют и доставляют контент конечным пользователям. Они располагаются в стратегически выгодных географических локациях с эффективным доступом к каналам связи. Благодаря этому контент доставляется пользователю по кратчайшему маршруту с минимальной задержкой.


Yandex Cloud CDN предоставляет доступ к 65 точкам присутствия с суммарной пропускной способностью более 7 Тбит/с.



{% note info %}

Мы постоянно развиваем и расширяем сеть Yandex Cloud CDN новыми точками присутствия. Актуальный список подсетей вы можете найти в разделе [Диапазоны публичных IP-адресов Cloud CDN](../../overview/concepts/public-ips.md#cloud-cdn).

{% endnote %}

Помимо количества точек присутствия, важнейшими параметрами являются их правильное расположение и техническая оснащенность.

Физическое расстояние между географическими точками на карте может кардинально отличаться от реального пути, который пройдет интернет-запрос. Например, два устройства в одном здании, но подключенные к разным провайдерам, будут обмениваться трафиком через пиринговые каналы между операторами связи. Эти пиринги могут находиться в другом городе или регионе.

Точки присутствия Yandex Cloud CDN размещаются исходя из особенностей сетевой инфраструктуры и обмена трафиком. Учитывается оптимальное расположение относительно каналов связи и точек обмена трафиком.

Ближайшая точка присутствия не всегда оптимальна по скорости доставки контента. Каналы связи могут быть загружены, что приводит к большим задержкам. Алгоритмы балансировки трафика в Yandex Cloud CDN позволяют избегать перегрузок точек присутствия и автоматически перенаправлять запросы к наиболее свободным серверам.

Подробнее об алгоритмах балансировки:

* Статья [Балансировщики нагрузки в облаке: повышение доступности и отказоустойчивости](https://yandex.cloud/ru/blog/posts/2025/04/load-balancers-guide)
* Видео [Под капотом Yandex Cloud CDN](https://runtime.strm.yandex.ru/player/video/vplvchkrwxqmrw76lajj)

На скорость доставки контента влияет также производительность серверов в точке присутствия. Важно, чтобы серверы не были перегружены и работали на современном оборудовании — как процессоры, так и диски для хранения кеша контента.

Yandex Cloud CDN построен на самом современном оборудовании и обеспечивает надежную и максимально производительную работу каждой точки присутствия.

## Вопросы и ответы {#qa}

{% cut "Если точки присутствия нет в моем городе?" %}

Если в вашем городе нет точки присутствия Yandex Cloud CDN, трафик автоматически направляется к наиболее близкой и свободной точке присутствия. Алгоритмы балансировки нагрузки учитывают не только географическую близость, но и текущую загрузку серверов и качество каналов связи, обеспечивая оптимальную скорость доставки контента.

{% endcut %}


{% cut "Доступны ли ресурсы за пределами России, Казахстана и Узбекистана?" %}



Да, ресурсы Yandex Cloud CDN доступны по всему миру, несмотря на то что точки присутствия расположены только в России, Казахстане и Узбекистане. Пользователи из других стран будут получать контент из точек присутствия в России, Казахстане и Узбекистане. Благодаря высокой пропускной способности и современному оборудованию качество доставки контента остается высоким даже для пользователей в других странах.

{% endcut %}

{% cut "Как получить список IP-адресов точек присутствия для настройки Firewall?" %}

Актуальный список IP-адресов точек присутствия Yandex Cloud CDN доступен в разделе [Cloud CDN](../../security/ip-list.md#cloud-cdn).

Чтобы снизить число сетей, от которых источник получает запросы, [подключите](../operations/resources/enable-shielding.md) экранирование источников. В этом случае запросы к источнику будут отправлять только промежуточные серверы кеширования. Список сетей экранирующих серверов можно получить в [технической поддержке](https://center.yandex.cloud/support).


{% endcut %}

{% cut "Где расположены экранирующие серверы?" %}

[Экранирующие серверы Yandex Cloud CDN](origins-shielding.md) расположены в Москве. Они обеспечивают дополнительный уровень кеширования между точками присутствия и источниками контента, снижая нагрузку на эти источники и повышая скорость доставки контента пользователям.


{% endcut %}
