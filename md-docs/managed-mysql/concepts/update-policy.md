# Политика поддержки версий MySQL®

Данный документ описывает жизненный цикл мажорных версий MySQL® в сервисе Yandex Managed Service for MySQL®. Политика основывается на [официальном графике поддержки Oracle MySQL®](https://www.mysql.com/support/eol-notice.html) и графике поддержки [Extended Support от Percona](https://www.percona.com/release-lifecycle-overview/#lifecycle) и предназначена для обеспечения баланса между стабильностью работы [кластеров](index.md) и поддержанием современной и безопасной инфраструктуры.

## Терминология и этапы жизненного цикла версий MySQL® {#version-lifecycle}

Для каждой мажорной версии MySQL® в сервисе Managed Service for MySQL® действует жизненный цикл продолжительностью 8–10 лет, включающий следующие этапы (состояния):

| Состояние версии | Описание и ключевые [действия](#actions-available) | Примерные сроки^1^                     |
|:------|:-------------|:----------------------------|
| Актуальная (`Actual`) | Самая последняя [LTS](https://ru.wikipedia.org/wiki/Долгосрочная_поддержка_программного_обеспечения)-версия с активной поддержкой. **Рекомендуется для всех новых проектов**. | с первого по пятый год |
| Поддерживаемая (`Supported`)  | Предыдущая LTS-версия с поддержкой `Extended Support` от Percona. Полностью поддерживается, создание новых кластеров разрешено. | с шестого по восьмой год |
| Устаревающая (`Deprecated`) | Версия с завершающейся поддержкой `Extended Support` от Percona. Создание новых кластеров заблокировано. Существующие кластеры работают в штатном режиме. | девятый год |
| Устаревшая с повышенной стоимостью (`Legacy Extra Paid`)^2^ | Версия, поддержка которой официально прекращена. Кластеры работают, но подлежат тарификации по повышенной ставке. Техническая поддержка ограничена. | десятый год |
| Выведенная из эксплуатации (`EOL`)^2^ | Версия более не поддерживается. Техническая поддержка не предоставляется. Кластеры работают, но подлежат тарификации по повышенной ставке. | одиннадцатый год и далее |

^1^ Сроки указаны относительно даты релиза мажорной версии. Точные даты перехода между состояниями публикуются в официальных анонсах сервиса.
^2^ Яндекс предварительно [уведомит](#notifications) об изменениях и начале тарификации версий по повышенной ставке в соответствии с условиями Договора.

## Ключевые принципы политики поддержки версий MySQL® {#key-principles}

Политика поддержки версий MySQL® основывается на следующих ключевых принципах:

* **Длительность поддержки**. Каждая LTS-версия находится в состоянии актуальной (`Actual`) или поддерживаемой (`Supported`) в течение восьми лет благодаря поддержке [Extended Support от Percona](https://www.percona.com/release-lifecycle-overview/#lifecycle).
* **Поэтапный вывод**. На девятый год версия переходит в состояние устаревающей (`Deprecated`), и создание новых кластеров становится невозможным.
* **Повышенная стоимость**^2^. На десятый год версия переходит в состояние устаревшей с повышенной стоимостью (`Legacy Extra Paid`). Кластеры подлежат повышенной тарификации, техническая поддержка ограничивается.
* **Завершение жизненного цикла**^2^. По достижении даты `EOL` (одиннадцатый год и далее) техническая поддержка не предоставляется. Восстановление из резервных копий невозможно. Кластеры работают, но подлежат повышенной тарификации.

^2^ Яндекс предварительно [уведомит](#notifications) об изменениях и начале тарификации версий по повышенной ставке в соответствии с условиями Договора.

## Политика обновлений {#update-policy}

* **Минорные обновления** (в пределах мажорной версии, например, 8.0.34 → 8.0.35) устанавливаются **автоматически** в рамках технического обслуживания, выполняемого в заданное для кластера [окно обслуживания](maintenance.md#maintenance-window) либо по требованию клиента. Эти обновления содержат исправления безопасности и ошибок. Обновление требует кратковременной поочередной перезагрузки хостов кластера.
* **Мажорные обновления** (смена основной версии, например, 8.0.x → 8.1.x) происходят в момент перехода версии из состояния устаревающей (`Deprecated`) в состояние устаревшей (`Legacy`). Запланированная операция обновления появляется в интерфейсе Yandex Cloud заранее и может быть отменена через [обращение](https://center.yandex.cloud/support) в службу технической поддержки. Версии в состоянии `EOL` невозможно обновить до новой мажорной версии. **Настоятельно рекомендуется выполнить обновление до наступления даты EOL**.

{% note info "Особенности Percona Server for MySQL®" %}

Благодаря поддержке `Extended Support` от Percona, версии MySQL® получают обновления безопасности и исправления ошибок дольше, чем при стандартном графике поддержки Oracle MySQL®.

{% endnote %}

## Уведомления {#notifications}

Сервис Managed Service for MySQL® заблаговременно уведомит вас о предстоящих изменениях:

* При переходе версии MySQL® в состояние **устаревающей (`Deprecated`)** (начало действия ограничений) сервис за 90 дней уведомит о невозможности создания новых кластеров с текущей версией MySQL® и порекомендует запланировать обновление существующих кластеров.
* При переходе версии MySQL® в состояние **устаревшей (`Legacy`)** (начало действия повышенного тарифа) сервис за 90 дней уведомит о начале действия повышенной ставки тарификации и предупредит о приближающемся прекращении поддержки.

Уведомления о таких изменениях отправляются пользователям в соответствии с заданными [настройками уведомлений](../../console/operations/update.md#notifications).

## График поддержки версий {#version-schedule}

Актуальный статус мажорных версий MySQL® от Percona основан на [официальном графике поддержки Oracle MySQL®](https://www.mysql.com/support/eol-notice.html) и графике поддержки [Extended Support от Percona](https://www.percona.com/release-lifecycle-overview/#lifecycle).

| Версия^5^ | Актуальная (`Actual`) | Поддерживаемая (`Supported`) | Устаревающая (`Deprecated`) | Устаревшая (`Legacy`) | `EOL` |
| :--- | :---: | :---: | :---: | :---: | :---: |
| **MySQL®&nbsp;8.4&nbsp;LTS** | 2024–2029 | 2029–2032 | 2033 | 2034 | 2035 |
| **MySQL®&nbsp;8.0&nbsp;LTS** | 2018–2023 | 2023–2028 | 2029 | 2029 | 2030 |
| **MySQL®&nbsp;5.7** | — | — | конец&nbsp;2026 | 2027 | конец&nbsp;2027 |

^5^ Точные сроки перехода между состояниями версий MySQL® объявляются дополнительно. Даты `EOL` соответствуют датам прекращения поддержки `Extended Support` от Percona.

{% note info %}

MySQL® версии 5.7 достигла завершения жизненного цикла (`EOL`) в октябре 2023 года. Кластеры этой версии подлежат принудительному обновлению в соответствии с политикой сервиса.

{% endnote %}
