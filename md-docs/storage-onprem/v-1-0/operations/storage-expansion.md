# Расширение хранилища

Вы можете расширить объем хранилища On-premises Yandex Object Storage в рамках установленной квоты, добавив новые каплы в кластер.

{% note warning %}

Вы не сможете использовать добавленное пространство, если оно превышает установленную квоту (лицензию).

{% endnote %}

{% note info %}

По умолчанию при установке кластера квота составляет 75% от общего объема дисков. Это необходимо для служебных манипуляций с данными и процедуры замены дисков.

{% endnote %}

1. Получите список подов в пространстве имен `mastermind-gateway`:

    ```bash
    kubectl get pods -n mastermind-gateway
    ```

    Результат:

    ```text
    NAME                      READY   STATUS    RESTARTS        AGE
    gateway-98b66b***-*****   1/1     Running   4 (2d21h ago)   5d17h
    gateway-98b66b***-*****   1/1     Running   4 (2d21h ago)   5d17h
    gateway-98b66b***-*****   1/1     Running   6 (2d21h ago)   5d17h
    ```

1. Подключитесь к одному из подов `gateway`:

    ```bash
    kubectl exec -it -n mastermind-gateway <имя_пода> -- bash
    ```

    Где `<имя_пода>` — имя пода из списка, полученного на предыдущем шаге.

1. Проверьте количество свободных групп:

    ```bash
    mastermind group list -s init
    ```

    Количество доступных для построения каплов определяется как количество свободных групп, разделенное на `3`.

1. Постройте каплы. Для этого выполните команду в зависимости от класса хранения, который используется в вашем кластере:

    {% list tabs %}

    - Класс хранения EC

      ```bash
      mastermind couple build \
        -n s3-default \
        -c <количество_каплов> \
        --state coupled \
        --hurry \
        --size 6 \
        --min-dcs 3 \
        --group-total-space 916G \
        --ec \
        --data-parts 4 \
        --code-parts 2 \
        --allow-dc-sharing
      ```

    - Класс хранения R3

      ```bash
      mastermind couple build \
        -n s3-default \
        -c <количество_каплов> \
        --state coupled \
        --hurry
      ```

    {% endlist %}

    Где `<количество_каплов>` — количество каплов, которые нужно построить. По умолчанию один капл добавляет 1 ТБ в хранилище.
