# Замена дисков

При выходе из строя диска автоматика обнаруживает это и начинает процедуру восстановления и замены диска.

Процесс происходит в несколько этапов:

1. Автоматика пытается перемонтировать диск и проверить его состояние.
1. Если перемонтирование не помогло, начинается процедура развоза данных, которые лежали на этом диске.
1. После завершения служебных манипуляций с диском и данными статус диска меняется на `ERROR`, и можно производить замену.

{% note info %}

В процессе служебных манипуляций с диском и данными статус диска отображается в консоли управления во вкладке **Status**.

{% endnote %}

![Статусы дисков](../_assets/operations/disks-chart.png =300x)

Диск будет в статусе `WARN` во время переноса данных. После того как с диска будут перенесены все данные, его статус сменится на `ERROR`.

1. Определите, какой из дисков необходимо заменить:

    1. Перейдите в служебную Grafana (эндпоинт выдается при установке кластера).
    1. Перейдите во вкладку **Explore**.
    1. Выберите источник данных **Prometheus**.
    1. В строку запроса введите:

        ```text
        disks_stats_updater{name="to_replace"}
        ```

    1. Нажмите **Run query**.

        ![Запрос в Grafana](../_assets/operations/disks-stats-updater.png)

        В результате можно посмотреть хост, диск, серийный номер и слот:

        ```json
        {
          "container": "autoadmin",
          "endpoint": "metrics",
          "instance": "[2a02:6b8:c0e:119:0:6399:****:****]:5004",
          "job": "elliptics-autoadmin/autoadmin",
          "name": "to_replace",
          "namespace": "elliptics-autoadmin",
          "nodename": "onprem13vla.storage.yandex.net",
          "pod": "autoadmin-tnk6m",
          "serial": "********",
          "shelf": "/dev/sg37",
          "slot": "10",
          "storage_disk": "yes"
        }
        ```

1. Замените диск физически.

    {% note warning %}

    Замену диска производите только после того, как его статус изменится на `ERROR`. Замена диска в статусе `WARN` может привести к потере данных.

    {% endnote %}
