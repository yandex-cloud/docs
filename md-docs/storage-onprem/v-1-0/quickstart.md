# Управление продуктом On-premises Yandex Object Storage

Программное обеспечение предназначено для размещения в клиентской инфраструктуре объектного хранилища, совместимого с [Amazon S3 API](https://docs.aws.amazon.com/AmazonS3/latest/API/Type_API_Reference.html).

Для выполнения вам понадобится доступ к графическому интерфейсу или CLI.

Чтобы начать работать с объектным хранилищем:

1. [Настройте SAML-аутентификацию](#configure-saml).
1. [Создайте тенант](#create-tenant).
1. [Добавьте пользователя](#add-user-to-pak).
1. [Назначьте пользователю роль](#assign-role).
1. [Создайте учетные данные S3](#create-access-key).
1. [Установите и настройте AWS CLI](#configure-aws-cli).
1. [Создайте бакет](#create-bucket).
1. [Загрузите объект в бакет](#upload-files).
1. [Получите ссылку на скачивание объекта](#get-link).

В случае возникновения вопросов вы можете [обратиться](#support) в техническую поддержку.

См. также [Прочие действия](#see-also).


## Настройте SAML-аутентификацию {#configure-saml}

При первом входе в объектное хранилище настройте [SAML-аутентификацию](https://ru.wikipedia.org/wiki/SAML):
1. Получите у вашего провайдера аутентификации следующие данные:
    * домен для аутентификации пользователей;
    * метаданные SAML.

    {% note alert %}

    После первичной настройки аутентификация пользователей будет выполняться только через провайдер аутентификации SAML.

    {% endnote %}

1. Задайте настройки:

    {% list tabs group=instructions %}

    - Графический интерфейс {#console}

      1. На левой панели перейдите в раздел ![image](_assets/console-icons/sliders.svg) **Настройки**.
      1. На вкладке **Аутентификация** в правом верхнем углу нажмите ![image](_assets/console-icons/pencil.svg) **Редактировать**.
      1. В открывшемся окне введите полученные ранее **Домен пользователей** и **Метаданные SAML**.
      1. Нажмите **Сохранить**.

      {% note tip %}

      В дальнейшем в блоке **Настройки SAML** вы можете скопировать базовые параметры:
      * `Путь к метаданным`
      * `Путь к EntityID`
      * `Путь для входа`
      * `Путь для выхода`

      Чтобы сохранить метаданные SAML в формате XML, в правом верхнем углу нажмите ![image](_assets/console-icons/arrow-shape-down-from-line.svg) **Скачать SAML xml**.

      {% endnote %}

    {% endlist %}

В дальнейшем вы сможете аналогичным образом изменить настройки SAML-аутентификации.


## Создайте тенант {#create-tenant}

_Тенант_ — выделенное пространство для размещения бакетов. Предназначен для организационного разграничения объема хранилища и прав доступа пользователей к бакетам. Например, вы можете создать отдельные тенанты под разные проекты или окружения.

Чтобы создать тенант:

{% list tabs group=instructions %}

- Графический интерфейс {#console}

  1. На левой панели перейдите в раздел ![image](_assets/console-icons/folders.svg) **Тенанты**.
  1. В правом верхнем углу нажмите ![image](_assets/console-icons/plus.svg) **Создать**.
  1. В открывшемся окне:
      * В поле **Имя** введите имя тенанта.
      * В поле **Макс. размер** укажите максимальный суммарный объем хранилища всех бакетов тенанта или выберите опцию **Без ограничений**.
  1. Нажмите **Создать**.

- CLI {#cli}

  1. Инициализируйте CLI с помощью команды в терминале:

      ```bash
      cli config init https://<адреc_графического_интерфейса_объектного_хранилища>
      ```

  1. Аутентифицируйтесь в объектном хранилище:

      ```bash
      cli login
      ```

      На открывшейся странице браузера введите свои учетные данные.

  1. Создайте тенант, задав в команде его имя:

      ```bash
      cli tenants create <имя_тенанта>
      ```

      В результате команды будет выведен идентификатор созданного тенанта.

{% endlist %}

## Назначьте пользователю роль {#assign-role}

Пользователь добавляется без каких-либо ролей, определяющих его права доступа к тенантам.

Вы можете назначить пользователю [глобальную роль](#assign-global-role) на все тенанты или [роль в конкретном тенанте](#assign-tenant-role).


### Назначьте пользователю глобальную роль {#assign-global-role}

{% list tabs group=instructions %}

- Графический интерфейс {#console}

  1. На левой панели перейдите в раздел ![image](_assets/console-icons/persons.svg) **Пользователи**.
  1. Напротив пользователя, добавленного ранее, нажмите ![image](_assets/console-icons/ellipsis.svg) и выберите ![image](_assets/console-icons/pencil.svg) **Изменить роль**.
  1. В открывшемся окне выберите глобальную роль для пользователя:
      * `VIEWER` — доступ к просмотру всех тенантов, бакетов и идентификаторов учетных данных S3 в объектном хранилище, а также их настроек.
      * `EDITOR` — доступ к просмотру, созданию, изменению и удалению тенантов, бакетов и учетных данных S3 в объектном хранилище, а также изменению их настроек, кроме назначения прав доступа пользователям.
      * `ADMIN` — доступ к просмотру, созданию, изменению и удалению тенантов, бакетов и учетных данных S3 в объектном хранилище, а также изменению их настроек, в том числе назначению прав доступа пользователям.
  1. Нажмите **Сохранить**.

  {% note tip %}

  Аналогичным образом вы можете ![image](_assets/console-icons/lock.svg) **Заблокировать** пользователя на уровне всего объектного хранилища.

  {% endnote %}

{% endlist %}


### Назначьте пользователю роль в тенанте {#assign-tenant-role}

{% list tabs group=instructions %}

- Графический интерфейс {#console}

  1. На левой панели перейдите в раздел ![image](_assets/console-icons/folders.svg) **Тенанты**.
  1. Выберите тенант, созданный ранее.
  1. На вкладке **Пользователи** в правом верхнем углу нажмите ![image](_assets/console-icons/plus.svg) **Добавить пользователя**.
  1. В открывшемся окне выберите пользователя и роль в тенанте:
      * `VIEWER` — доступ к просмотру всех бакетов и идентификаторов учетных данных S3 в тенанте, а также их настроек и настроек тенанта.
      * `EDITOR` — доступ к просмотру, созданию, изменению и удалению бакетов и учетных данных S3 в тенанте, а также изменению их настроек и настроек тенанта, кроме назначения прав доступа пользователям.
      * `ADMIN` — доступ к просмотру, созданию, изменению и удалению бакетов и учетных данных S3 в тенанте, а также изменению их настроек и настроек тенанта, в том числе назначению прав доступа пользователям.
  1. Нажмите **Добавить**.

  {% note tip %}

  В дальнейшем на вкладке **Пользователи** вы можете ![image](_assets/console-icons/person-pencil.svg) **Изменить роль** или ![image](_assets/console-icons/trash-bin.svg) **Удалить** пользователя из тенанта.

  {% endnote %}

- CLI {#cli}

  В терминале выполните команду, указав идентификатор тенанта, адрес электронной почты пользователя и роль:

  ```bash
  cli tenants members change \
    <идентификатор_тенанта> \
    <адрес_электронной_почты> \
    <роль>
  ```

  Доступны следующие роли:
  * `viewer` — доступ к просмотру всех бакетов и идентификаторов учетных данных S3 в тенанте, а также их настроек и настроек тенанта.
  * `editor` — доступ к просмотру, созданию, изменению и удалению бакетов и учетных данных S3 в тенанте, а также изменению их настроек и настроек тенанта, кроме назначения прав доступа пользователям.
  * `admin` — доступ к просмотру, созданию, изменению и удалению бакетов и учетных данных S3 в тенанте, а также изменению их настроек и настроек тенанта, в том числе назначению прав доступа пользователям.

{% endlist %}


## Создайте учетные данные S3 {#create-access-key}

С помощью _учетных данных S3_ вы можете управлять статическими ключами доступа к бакетам тенанта с возможностью разграничения прав.

_Статические ключи доступа_ — это стандартный способ аутентификации в Amazon S3 API и совместимых инструментах, например [AWS CLI](https://aws.amazon.com/cli/), [AWS SDK](https://aws.amazon.com/products/developer-tools/) или файловых браузерах.

{% list tabs group=instructions %}

- Графический интерфейс {#console}

  1. Создайте учетные данные S3:

      1. На левой панели перейдите в раздел ![image](_assets/console-icons/folders.svg) **Тенанты**.
      1. Выберите тенант, созданный ранее.
      1. На вкладке **Учетные данные S3** в правом верхнем углу нажмите ![image](_assets/console-icons/plus.svg) **Новые учетные данные S3**.
      1. В открывшемся окне введите имя учетных данных S3 и выберите роль в тенанте:
          * `VIEWER` — доступ к просмотру всех бакетов тенанта и объектов в них, а также настроек бакета.
          * `EDITOR` — доступ к просмотру, созданию, изменению и удалению бакетов тенанта и объектов в них, а также изменению настроек бакетов, кроме назначения прав доступа пользователям.
          * `ADMIN` — доступ к просмотру, созданию, изменению и удалению бакетов тенанта и объектов в них, а также изменению настроек бакетов, в том числе назначению прав доступа пользователям.
      1. Нажмите **Добавить**.

  1. Создайте статический ключ доступа:

      1. На странице учетных данных S3 в правом верхнем углу нажмите ![image](_assets/console-icons/key.svg) **Управление ключами**.
      1. В открывшемся окне нажмите ![image](_assets/console-icons/plus.svg) **Создать новый ключ доступа**.
      1. Сохраните идентификатор и секретный ключ.
      1. Нажмите **Закрыть**.

- CLI {#cli}

  1. Создайте учетные данные S3, выполнив в терминале команду:

      ```bash
      cli service-accounts create <имя_учетных_данных> \
        --tenant <идентификатор_тенанта>
      ```

  1. Создайте статический ключ доступа, выполнив в терминале команду:

      ```bash
      cli access-keys create <идентификатор_учетных_данных> \
        --tenant <идентификатор_тенанта>
      ```

{% endlist %}


## Установите и настройте AWS CLI {#configure-aws-cli}

[AWS CLI](https://aws.amazon.com/cli/) — это популярная консольная утилита для работы с объектными хранилищами, совместимыми с Amazon S3 API.

1. Установите AWS CLI:

    {% list tabs group=operating_system %}

    - Linux {#linux}

      1. В терминале выполните команду:

          ```bash
          curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" --output "awscliv2.zip"
          unzip awscliv2.zip
          sudo ./aws/install
          ```

      1. После завершения установки перезапустите терминал.

    - macOS {#macos}

      1. В терминале выполните команду:

          ```bash
          curl "https://awscli.amazonaws.com/AWSCLIV2.pkg" --output "AWSCLIV2.pkg"
          sudo installer -pkg AWSCLIV2.pkg -target /
          ```

      1. После завершения установки перезапустите терминал.

    - Windows

      1. Скачайте [установочный файл](https://awscli.amazonaws.com/AWSCLIV2.msi) AWS CLI.
      1. Откройте файл и следуйте инструкциям программы установки.

    {% endlist %}

    Подробнее об установке AWS CLI см. в [документации AWS](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html).

1. Настройте AWS CLI:

    {% list tabs group=instructions %}

    - AWS CLI {#aws-cli}

      1. В терминале введите команду для настройки AWS CLI:

          ```bash
          aws configure
          ```

      1. Введите значения для параметров:
          * `AWS Access Key ID` — идентификатор статического ключа, полученный ранее.
          * `AWS Secret Access Key` — секретный ключ, полученный ранее.
          * `Default region name` — регион `us-east-1`.

      1. Задайте эндпоинт S3 API:

          ```bash
          aws configure set endpoint_url https://<эндпоинт_S3_API>/
          ```

          {% note tip %}

          Обычно эндпоинт S3 API размещается в той же доменной зоне, что и графический интерфейс. Например, если адрес графического интерфейса `console.example.com`, эндпоинт S3 API будет на домене `s3.example.com`.

          {% endnote %}

          {% cut "Пример получившихся конфигурационных файлов" %}

          * `~/.aws/config`:

            ```text
            [default]
            region = us-east-1
            endpoint_url = https://<эндпоинт_S3_API>/
            ```

          * `~/.aws/credentials`:

            ```text
            [default]
            aws_access_key_id = <идентификатор_статического_ключа>
            aws_secret_access_key = <секретный_ключ>
            ```

          {% endcut %}

    {% endlist %}

Подробнее о настройке AWS CLI см. в [документации AWS](https://docs.aws.amazon.com/cli/latest/userguide/cli-configure-quickstart.html).


## Создайте бакет {#create-bucket}

_Бакет_ — это выделенная часть тенанта для пользовательских данных. Каждый бакет объектного хранилища имеет уникальное название, по которому выполняются запросы.

Данные в бакетах хранятся в виде объектов. Вы можете логически структурировать данные с помощью нескольких бакетов или с помощью префиксов внутри одного бакета.

{% list tabs group=instructions %}

- Графический интерфейс {#console}

  1. На левой панели перейдите в раздел ![image](_assets/console-icons/folders.svg) **Тенанты** и выберите тенант.
  1. На вкладке **Бакеты** нажмите ![image](_assets/console-icons/plus.svg) **Создать бакет**.
  1. Укажите параметры:

      1. Введите имя бакета.
      1. При необходимости ограничьте максимальный размер бакета.

          Размер `0` означает отсутствие ограничений и аналогичен включенной опции **Без ограничений**.

      1. Выберите класс хранилища по умолчанию:

          * `STANDARD`
          * `COLD`
          * `ICE`

          Более «холодные» классы предназначены для длительного хранения объектов, работать с которыми планируется реже.

      1. При необходимости включите версионирование. Чтобы не включать версионирование, не нажимайте кнопки в блоке **Версионирование**.

          {% note warning %}

          Операция включения необратима: отключить версионирование нельзя, можно только приостановить создание новых версий. После приостановки версионирования новые объекты будут сохраняться с версией `null`.

          {% endnote %}

          {% note info %}

          При включенной блокировке версии объекта версионирование нельзя приостановить.

          {% endnote %}

      1. При необходимости добавьте метки:

          1. Нажмите **Добавить**.
          1. Введите метку в формате `ключ: значение`.
          1. Нажмите **Enter**.

  1. Нажмите **Создать** для завершения операции.

- CLI {#cli}

  В терминале выполните команду, указав имя бакета и идентификатор тенанта:

  ```bash
  cli buckets create \
    <имя_бакета> \
    --tenant <идентификатор_тенанта>
  ```

  На имя бакета накладываются следующие ограничения:
  * Имена бакетов должны быть уникальны для всего объектного хранилища, то есть нельзя создать два бакета с одинаковыми именами даже в разных тенантах. Помните об этом, если планируете создавать бакеты автоматически через API.
  * Длина имени должна быть от 3 до 63 символов.
  * Имя может содержать строчные буквы латинского алфавита, цифры, дефисы и точки.
  * Первый и последний символы должны быть буквами или цифрами.
  * Символы справа и слева от точки должны быть буквами или цифрами. 
  * Имя не должно иметь вид IP-адреса (например `10.1.3.9`).

- AWS CLI {#aws-cli}

  В терминале выполните команду:

  ```bash
  aws s3api create-bucket \
    --bucket <имя_бакета>
  ```

  Где `--bucket` — имя бакета. На имя бакета накладываются следующие ограничения:
  * Имена бакетов должны быть уникальны для всего объектного хранилища, то есть нельзя создать два бакета с одинаковыми именами даже в разных тенантах. Помните об этом, если планируете создавать бакеты автоматически через API.
  * Длина имени должна быть от 3 до 63 символов.
  * Имя может содержать строчные буквы латинского алфавита, цифры, дефисы и точки.
  * Первый и последний символы должны быть буквами или цифрами.
  * Символы справа и слева от точки должны быть буквами или цифрами. 
  * Имя не должно иметь вид IP-адреса (например `10.1.3.9`).

  Результат:

  ```text
  {
    "Location": "/<имя_бакета>"
  }
  ```

  Подробнее о команде в [документации AWS](https://docs.aws.amazon.com/cli/latest/reference/s3api/create-bucket.html).

{% endlist %}


## Загрузите объект в бакет {#upload-files}

_Объекты_ размещаются в бакетах и содержат пользовательские данные произвольного формата в том виде, в котором они были загружены.

Идентификатор объекта — строковый ключ.

Вместе с объектом сохраняются пользовательские и системные метаданные.

{% list tabs group=instructions %}

- AWS CLI {#aws-cli}

  Чтобы загрузить объект, выполните команду, указав путь к локальному файлу, который нужно загрузить, имя вашего бакета и ключ, по которому объект будет храниться в бакете:

  ```bash
  aws s3 cp <путь_к_локальному_файлу> s3://<имя_бакета>/<ключ_объекта>
  ```

  Подробнее о команде в [документации AWS](https://docs.aws.amazon.com/cli/latest/reference/s3/cp.html).

{% endlist %}


## Получите ссылку на скачивание объекта {#get-link}

{% list tabs group=instructions %}

- AWS CLI {#aws-cli}

  В терминале выполните команду, указав имя бакета, ключ объекта и время жизни ссылки в секундах:

  ```bash
  aws s3 presign s3://<имя_бакета>/<ключ_объекта> --expires-in <время_жизни>
  ```

  Полученной ссылкой вы можете поделиться или использовать ее в своем сервисе для доступа к файлу.

  Подробнее о команде в [документации AWS](https://docs.aws.amazon.com/cli/latest/reference/s3/presign.html).

{% endlist %}

### Настроить бакет {#config-bucket}

#### Настроить жизненный цикл {#lifecycle}

Object Storage позволяет управлять жизненными циклами объектов в бакете.

Раз в сутки к жизненным циклам применяются изменения, актуальные на момент 00:00 UTC. Операция выполняется в течение нескольких часов.

{% list tabs group=instructions %}

- Графический интерфейс {#console}

  1. На левой панели перейдите в раздел ![image](_assets/console-icons/folders.svg) **Тенанты** и выберите тенант.
  1. На вкладке **Бакеты** выберите бакет.
  1. На вкладке **Жизненный цикл** нажмите ![image](_assets/console-icons/gear.svg) **Настроить**.
  1. Добавьте и настройте нужные правила.
  1. Нажмите **Сохранить**.

- CLI {#cli}

  1. Опишите конфигурацию жизненных циклов объектов в формате JSON. Например:

      ```json
      {
        "Rules": [
          {
            "ID": "Dummy",
            "Status": "Enabled",
            "Filter": {
              "And": {
                "Prefix": "dummy",
                "ObjectSizeGreaterThan": 1073741824,
                "ObjectSizeLessThan": 2147483648
              }
            },
            "Expiration": {
              "Date": "2026-03-30T21:00:00+00:00"
            }
          }
        ]
      }
      ```

      {% cut "Возможные параметры конфигурации" %}

      * `ID` — уникальный идентификатор правила. Должен быть меньше или равен 255 символам. Необязательный параметр.
      * `Status` — состояние правила. Обязательный параметр.
      * `Filter` — фильтр объектов. Необязательный параметр. Может содержать не более одного элемента каждого типа:

          * `Prefix` — префикс ключа объекта, идентифицирующий один или несколько объектов, к которым применяется правило. Под действие правила попадают объекты с указанным префиксом ключа. Необязательный параметр.
          * `ObjectSizeGreaterThan` — минимальный размер объекта в байтах. Под действие правила попадают объекты, размер которых больше или равен указанному. Необязательный параметр.
          * `ObjectSizeLessThan` — максимальный размер объекта в байтах. Под действие правила попадают объекты, размер которых меньше или равен указанному. Необязательный параметр.
          * `Tags` — метки объекта. Необязательный параметр. Под действие правила попадают объекты, которым присвоена указанная метка. Передается в виде списка пар `Key` и `Value`, например:

              ```json
              "Tags": [
                {
                  "Key": "some",
                  "Value": "5"
                },
                {
                  "Key": "other",
                  "Value": "17"
                }
              ]
              ```

          * `And` — логический оператор **И** (`AND`) для фильтров. Необязательный параметр. Используйте этот фильтр, чтобы сочетать в одном правиле фильтрацию по префиксу, размеру или меткам объекта. С помощью логического оператора `And` вы можете настроить фильтрацию одновременно по нескольким меткам. Для этого в ключе `Tags` передайте метки в виде списка пар `Key` и `Value`.

          Если фильтр объектов не задан, под действие правила попадают все объекты в бакете.

      * `Expiration` — параметр правила для удаления любых объектов. Необязательный параметр. Может содержать:
          * `Days` — количество дней после создания объекта, когда действие правила вступает в силу. Необязательный параметр.
          * `Date` — дата, после которой вы хотите, чтобы действие вступило в силу. Формат — [ISO 8601](https://ru.wikipedia.org/wiki/ISO_8601), например `YYYY-MM-DDT00:00:00Z`. Необязательный параметр.
          * `ExpiredObjectDeleteMarker` — удаляет маркер удаления, для которого больше не существует неактуальных версий объекта. Принимает значения `true` или `false`. Необязательный параметр.

      * `NoncurrentVersionExpiration` — параметр правила для удаления неактивных версий объектов. Необязательный параметр. Может содержать:
          * `NoncurrentDays` — количество дней истечения срока действия. Обязательный параметр.

      * `AbortIncompleteMultipartUpload` — параметр правила для удаления всех частей составных загрузок, не завершенных за указанное количество дней. Необязательный параметр. Может содержать:
          * `DaysAfterInitiation` — количество дней после инициации загрузки. Обязательный параметр.

      Необходимо указать хотя бы один из параметров `Expiration`, `NoncurrentVersionExpiration`, `AbortIncompleteMultipartUpload`.

      {% endcut %}

  1. В терминале выполните команду:

      ```bash
      cli buckets update lifecycle <имя_бакета> \
        --tenant <идентификатор_тенанта> \
        --file <путь_к_файлу_конфигурации>
      ```

{% endlist %}


#### Настроить права доступа {#acl}

Чтобы настроить доступ к бакету, назначьте разрешения на бакет:

{% list tabs group=instructions %}

- Графический интерфейс {#console}

  1. На левой панели перейдите в раздел ![image](_assets/console-icons/folders.svg) **Тенанты** и выберите тенант.
  1. На вкладке **Бакеты** выберите бакет.
  1. На вкладке **Права доступа** нажмите ![image](_assets/console-icons/plus.svg) **Добавить правило доступа**.
  1. Выберите учетные данные S3 из списка или введите идентификатор вручную.
  1. Выберите одно или несколько разрешений.
  1. Нажмите **Сохранить**.

- CLI {#cli}

  1. Опишите конфигурацию прав доступа в формате JSON. Например:

      ```json
      {
        "Grants": [
          {
            "Grantee": {
              "Type": "CanonicalUser",
              "DisplayName": "********",
              "ID": "********"
            },
            "Permission": "READ"
          }
        ]
      }
      ```

      {% cut "Возможные параметры конфигурации" %}

      * `Grants` — список грантов доступа. Обязательный параметр. Каждый грант содержит:

          * `Grantee` — получатель доступа. Обязательный параметр. Содержит:

              * `Type` — тип получателя доступа. Обязательный параметр.
              * `DisplayName` — отображаемое имя получателя. Обязательный параметр.
              * `ID` — идентификатор получателя доступа. Обязательный параметр.

          * `Permission` — разрешение на доступ. Обязательный параметр. Возможные значения:

              * `FULL_CONTROL` — полный доступ ко всем операциям.
              * `READ` — чтение объектов.
              * `WRITE` — запись объектов.
              * `READ_ACP` — чтение списка разрешений доступа.
              * `WRITE_ACP` — изменение списка разрешений доступа.

      {% endcut %}

  1. В терминале выполните команду:

      ```bash
      cli buckets update acl <имя_бакета> \
        --tenant <идентификатор_тенанта> \
        --file <путь_к_файлу_конфигурации>
      ```

{% endlist %}


#### Настроить политику доступа {#policy}

Политики доступа (bucket policy) устанавливают права на действия с бакетами, объектами и группами объектов.

{% list tabs group=instructions %}

- Графический интерфейс {#console}

  1. На левой панели перейдите в раздел ![image](_assets/console-icons/folders.svg) **Тенанты** и выберите тенант.
  1. На вкладке **Бакеты** выберите бакет.
  1. На вкладке **Политика доступа** нажмите ![image](_assets/console-icons/gear.svg) **Настроить**.
  1. Укажите идентификатор политики.
  1. Добавьте и настройте нужные правила.
  1. Нажмите **Сохранить**.

- CLI {#cli}

  1. Опишите конфигурацию политики доступа в виде схемы данных в формате JSON. Например:

      ```json
      {
        "Version": "2012-10-17",
        "Statement": {
          "Effect": "Allow",
          "Principal": "*",
          "Action": "s3:GetObject",
          "Resource": "arn:aws:s3:::<имя_бакета>/*",
          "Condition": {
            "Bool": {
              "aws:SecureTransport": "true"
            }
          }
        }
      }
      ```

      {% cut "Возможные параметры конфигурации" %}

      * `Version` — версия описания политик доступа. Необязательный параметр.
      * `Statement` — правила политики доступа:
          * `Effect` — запрет или разрешение запрошенного действия. Возможные значения: `Allow` и `Deny`.
          * `Principal` — идентификатор субъекта запрошенного разрешения. Необязательный параметр. Возможные значения:
              * `"*"`.
              * `"CanonicalUser": "<идентификатор_субъекта>"`.

          * `Action` — действие, которое будет разрешено при срабатывании политики. Возможные значения: `s3:GetObject`, `s3:PutObject` и `*` если необходимо применять политику ко всем действиям.
          * `Resource` — ресурс, к которому будет применяться правило.
          * `Condition` — условие, которое будет проверяться. Необязательный параметр.

          Вы можете настроить для правила одновременно несколько условий и задать для каждого условия несколько ключей. При этом такие условия и их ключи будут проверяться с логикой `И`. То есть для выполнения правила запрос должен будет соответствовать сразу всем указанным признакам.

          Для каждого ключа условия вы можете задать одновременно несколько значений. При этом такие значения будут проверяться с логикой `ИЛИ`. То есть для выполнения правила запрос должен будет соответствовать любому из заданных значений ключа условия.

      {% endcut %}

  1. В терминале выполните команду:

      ```bash
      cli buckets update acl <имя_бакета> \
        --tenant <идентификатор_тенанта> \
        --file <путь_к_файлу_конфигурации>
      ```

{% endlist %}


#### Настроить правила CORS {#cors}

Object Storage позволяет управлять конфигурацией CORS в бакете.

{% list tabs group=instructions %}

- Графический интерфейс {#console}

  1. На левой панели перейдите в раздел ![image](_assets/console-icons/folders.svg) **Тенанты** и выберите тенант.
  1. На вкладке **Бакеты** выберите бакет.
  1. На вкладке **CORS** нажмите ![image](_assets/console-icons/gear.svg) **Настроить**.
  1. Добавьте и настройте нужные правила.
  1. Нажмите **Сохранить**.

- CLI {#cli}

  1. Опишите конфигурацию правил CORS в формате JSON. Например:

      ```json
      {
        "CORSRules": [
          {
            "ID": "cors-rule-1",
            "AllowedHeaders": [
              "Content-Type",
              "Authorization",
              "X-Requested-With"
            ],
            "AllowedMethods": [
              "GET"
            ],
            "AllowedOrigins": [
              "*"
            ],
            "ExposeHeaders": null,
            "MaxAgeSeconds": 3600
          }
        ]
      }
      ```

      {% cut "Возможные параметры конфигурации" %}

      * `CORSRules` — список правил CORS. Обязательный параметр. Каждое правило содержит:
          * `ID` — уникальный идентификатор правила. Обязательный параметр.
          * `AllowedHeaders` — список заголовков, которые клиент может использовать в запросе. Обязательный параметр.
          * `AllowedMethods` — список HTTP-методов, разрешенных для кросс-доменных запросов. Обязательный параметр.
          * `AllowedOrigins` — список доменов, с которых разрешены кросс-доменные запросы. Обязательный параметр.
          * `ExposeHeaders` — список заголовков, которые браузер может получить из ответа. Обязательный параметр.
          * `MaxAgeSeconds` — максимальное время в секундах, в течение которого браузер может кэшировать результаты предварительного запроса. Обязательный параметр.

      {% endcut %}

  1. В терминале выполните команду:

      ```bash
      cli buckets update cors <имя_бакета> \
        --tenant <идентификатор_тенанта> \
        --file <путь_к_файлу_конфигурации>
      ```

{% endlist %}


#### Включить блокировки объектов {#object-lock}

В версионируемых бакетах можно настроить механизм блокировки версий объектов (object lock). Когда он включен, вы можете установить на версию объекта блокировку, то есть запрет на удаление или перезапись. Также для бакета можно настроить блокировки по умолчанию: они будут устанавливаться на все новые версии объектов.

{% note info %}

Блокировка версий объектов невозможна в бакетах с выключенным или приостановленным версионированием.

При включенной блокировке версионирование нельзя приостановить.

{% endnote %}

{% list tabs group=instructions %}

- Графический интерфейс {#console}

  1. На левой панели перейдите в раздел ![image](_assets/console-icons/folders.svg) **Тенанты** и выберите тенант.
  1. На вкладке **Бакеты** выберите бакет.
  1. На вкладке **Блокировки объектов** включите опцию **Включить блокировки**.

      Опция только включает или выключает возможность работать с блокировками. У существующих объектов блокировки не меняются.

  1. При необходимости включите опцию **Блокировки по умолчанию для новых версий**, укажите тип и срок блокировки по умолчанию.
  1. Нажмите **Сохранить**.

{% endlist %}
