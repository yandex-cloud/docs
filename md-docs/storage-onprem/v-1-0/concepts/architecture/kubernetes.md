# Требования к кластеру Kubernetes

On-premises Yandex Object Storage может обеспечить заявленные свойства при выполнении требований к конфигурации кластера:

1. Необходимо не менее трех хостов, на которых развернут Object Storage.
    Это влияет на надежность защиты данных в Elliptics, отказоустойчивость используемых сервисов и доступность S3 API.

    Также производительность масштабируется вместе с количеством хостов.

1. Object Storage лучше всего работает с большим количеством HDD на каждом из хостов.
    Разделение данных по дискам влияет не только на надежность, но и на производительность — Object Storage старается утилизировать все диски параллельно и равномерно.

1. Настоятельно рекомендуется иметь также быстрый SSD/NVMe-диск, из которого TopoLVM будет создавать PV.
    Это влияет на производительность шардов PostgreSQL, которые Goose активно использует под нагрузкой.

1. Рекомендуется иметь отдельные сетевые интерфейсы для слоя Kubernetes и внешних подключений из-за большого количества горизонтального трафика.
