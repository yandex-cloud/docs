# Схема решения

On-premises Yandex Object Storage предназначен для установки в кластер Kubernetes. Он состоит из:

* набора Helm-чартов и образов, реализующих S3-совместимое хранилище;
* набора системных зависимостей, необходимых для работы хранилища;
* нескольких deb-пакетов для установки на физические хосты.

S3-совместимое хранилище можно логически разделить на две части: слой хранения (выделен фиолетовым) и интерфейс S3 (выделен оранжевым).


## Слой хранения {#storage-layer}

Слой хранения состоит из:

1. *[Elliptics](elliptics.md)* — распределенное key-value хранилище бинарных данных, оптимизированное для работы с большим количеством HDD. Append-only.
    На уровне клиента Elliptics реализуются классы защиты данных и обеспечения избыточности (репликация, EC).

1. *[Mastermind](mastermind.md)* — подсистема над Elliptics, наблюдающая за состоянием кластера Elliptics и выполняющая его автоматическое обслуживание.


## Интерфейс S3 {#s3-interface}

Интерфейс S3 состоит из:

1. *[Goose](goose.md)* — реализует S3-совместимый API поверх Elliptics.
    Для хранения метаданных используется шардированный PostgreSQL.

1. *Console API + Zitadel* — подсистема для администрирования кластера S3: управление тенантами, правами доступа, ключами, квотами и другое.


## Хосты кластера {#cluster-hosts}

Дополнительно напрямую на хосты кластера устанавливаются:

* *autoadmin* — часть Elliptics, ответственная за мониторинг состояния дисков и реализацию процессов их замены;
* *Inspector* — часть Console API, ответственная за сбор отладочной информации при возникновении инцидентов.


## Установка и обновление {#installation-and-update}

За установку и обновление всех компонентов отвечает *space-operator*.


## Системные зависимости {#system-dependencies}

Остальные компоненты относятся к системным зависимостям:

1. *ArgoCD* — CD-система. Используется для упрощения доставки. Управляется через space-operator.
1. *Cloud-Native PG* и *Percona Server MongoDB* — операторы для управления базами данных PostgreSQL и MongoDB.
1. *Minio* — устанавливается в небольшом объеме поверх незанятых Elliptics дисков. Используется для хранения резервных копий баз данных и отладочной информации на случай инцидентов, нарушающих работу Object Storage.
1. *Grafana, Mimir* и *Loki* реализуют подсистему Observability с хранением метрик и логов непосредственно в S3.
1. *TopoLVM* — CSI-плагин для Kubernetes. Используется для создания PV для баз данных, кеша Mimir/Loki и работы Minio.
1. *MetalLB* — для создания внешних виртуальных IP S3 на хостах Kubernetes кластера.
1. *External Secrets* — для управления секретами в разных пространствах имен.
1. *cert-manager* — для внутренних сертификатов системы.
