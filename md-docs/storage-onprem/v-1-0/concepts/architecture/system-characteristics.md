# Сводка характеристик системы

## Масштабируемость и производительность {#scalability-and-performance}

Пропускная способность On-premises Yandex Object Storage горизонтально масштабируется и зависит от:

* количества хостов, а следовательно:
    * дисков для данных;
    * сетевых интерфейсов;
    * копий [Elliptics](elliptics.md) и [Goose](goose.md);
    * количества шардов S3DB;

* количества дисков на одном хосте — вплоть до пропускной способности интерфейса дисковой полки;
* объема дисков — за счет параллельной записи в бэкенды Elliptics, вплоть до ограничения iops самих дисков.

При вертикальном масштабировании влияние оказывают:

* производительность и число ядер CPU — в первую очередь для обработки сетевых соединений и работы PostgreSQL;
* объем и производительность RAM — для кеширования;
* крайне важно — производительность дисков для PostgreSQL;
* производительность дисков для Elliptics.


## Консистентность {#consistency}

Чтение данных из Object Storage консистентно, и все изменения данных в S3, сделанные по одному из эндпоинтов, будут сразу видимы с любого из эндпоинтов. Это обеспечивается алгоритмом работы с Elliptics и взаимодействия Goose с PostgreSQL.


## Надежные конфигурации {#reliable-configurations}

В разделах выше были рассмотрены вопросы надежности хранения данных, отказоустойчивости и высокой доступности отдельных компонентов Object Storage.

Здесь сведены рекомендуемые конфигурации для обеспечения доступности чтения и записи данных при отказе хостов кластера Kubernetes.

Для поддержки отказа одного хоста необходимо:

* не менее трех хостов;
* класс избыточности — репликация x3 или EC 4+2;
* стандартная конфигурация с двумя репликами PostgreSQL.

{% note tip %}

Рекомендуется использовать не менее четырех хостов. Три хоста для минимальной отказоустойчивости, а четвертый – для поддержки производительности при отказе одного из хостов.

{% endnote %}

Для поддержки отказа двух хостов необходим один из вариантов конфигурации:

Параметр                       | Вариант 1     | Вариант 2
------------------------------ | ------------- | ----------
**Количество хостов**          | Не менее 5    | Не менее 6
**Класс избыточности**         | Репликация x3 | EC 4+2
**Количество реплик PostgreSQL** | 4             | 4
