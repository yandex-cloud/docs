# Mastermind

В On-premises Yandex Object Storage ответственность [Elliptics](elliptics.md) ограничена чтением и записью данных в бэкенды, а также мониторингом состояния дисков.

Автоматическое администрирование и обслуживание кластера Elliptics выполняет Mastermind:

* чтение и хранение состояния кластера Elliptics (статус бэкендов, каплов);
* создание новых каплов с учетом топологии кластера;
* починка каплов при отказах дисков или хостов;
* запуск задачи дефрагментации;
* расчет веса каплов за запись.

На рисунке ниже показана схема работы Mastermind:

![Схема работы Mastermind](../../_assets/architecture/mastermind-scheme.png =700x)

Для хранения снимка состояния кластера и данных запускаемых задач (jobs), например создания каплов и дефрагментации, Mastermind полагается на MongoDB, а для блокировок (locks) — на ZooKeeper.

И MongoDB, и ZooKeeper установлены в конфигурациях высокой доступности с одним главным хостом (primary) и двумя репликами по умолчанию.

Это позволяет полноценно функционировать при отказе одного хоста. Для поддержки отказа двух хостов необходимо использовать пять копий вместо трех.

Все сервисы Mastermind также устанавливаются в трех копиях. Хосты не хранят свои данные локально, а используют внешние хранилища и полагаются на согласованное решение между копиями MongoDB и ZooKeeper. Это позволяет функционировать при отказе до двух хостов.

{% note info %}

Все устанавливаемые в Kubernetes сервисы учитывают топологию кластера.

Это относится ко всем компонентам:

* часть сервисов, как и Elliptics, работает строго в одной копии на каждом хосте;
* часть сервисов, например Mastermind, работает в трех-пяти копиях при любом количестве хостов и всегда использует корректные `topologySpreadConstraints`.

{% endnote %}

Помимо каплов Mastermind оперирует такой абстракцией, как пространства имен. Пространство имен объединяет в себе каплы с одинаковым классом избыточности и параметрами записи. Информацию о пространствах имен, каплах и весах на запись в дальнейшем использует [Goose](goose.md) при реализации S3 API.
