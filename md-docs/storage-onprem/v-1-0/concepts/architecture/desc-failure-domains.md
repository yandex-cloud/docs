# Описание доменов отказа

## Один линк {#single-link}

* **Конфигурация**
    * Пиры для интерфейса `vxlan1` не настраиваются.
    * Используются четыре S3 IP-адреса.

* **Точки отказа**
    * Отказ одного линка не приводит к отказу кластера.
    * SPOF: отказ коммутатора.

* **Особенности**
    * Для исключения SPOF необходимо подключение к четырем коммутаторам.

## Два линка LACP {#two-links-lacp}

* **Конфигурация**
    * Пиры для интерфейса `vxlan1` не настраиваются.
    * Используются четыре S3 IP-адреса.

* **Точки отказа**
    * Отказ одного линка не влияет на работу системы.
    * Отказ одного члена коммутатора (половины линков) не влияет на работу системы.
    * При использовании MLAG возможна организация на двух физически разных коммутаторах, что обеспечивает более высокую надежность по сравнению с одним стеком.
    * SPOF: коммутатор — если физически используется один стек или коммутатор.

* **Особенности**
    * Два коммутатора с MLAG обеспечивают отказоустойчивость access-свичей.
    * Требуется наличие Core-сети, к которой подключены эти свичи.

## Два линка — S3 + internal {#two-links-s3-internal}

* **Конфигурация**
    * Интерфейс `vxlan1` не настраивается.
    * Интерфейс `vxlan0` настраивается на внутренний интерфейс.
    * MetalLB настраивается на S3-интерфейс.
    * Используются четыре S3 IP-адреса.

* **Точки отказа**
    * Отказ одного линка не приводит к отказу кластера.
    * SPOF: отказ коммутатора.
    * SPOF: при отказе линка S3 не сработает MetalLB, так как MetalLB проверяет доступность по внутреннему линку.
    * Отказ с полным обрывом (link down в Linux) обрабатывается корректно.

* **Особенности**
    * Повышенная производительность за счет распределения части трафика по другим интерфейсам.
    * Повышенная безопасность, так как клиент не может перегрузить внутренние линки.

* **L2/L3**
    * Возможна организация в одном L2-сегменте с разными L3-сетями.
    * Рекомендуется организация в разных L2-сегментах.
    * Возможна организация в одном L2-сегменте с одинаковыми L3-сетями, но это требует дополнительных настроек в виде маршрутов до /32 внутренних хостов.


## Два линка в два разных коммутатора, смешанный трафик {#two-links-different-switches}

* **Конфигурация**
    * Настраиваются оба VXLAN-интерфейса.
    * Используются восемь S3 IP-адресов.

* **Точки отказа**
    * Отказ одного линка не влияет на работу системы.
    * SPOF: отказ одного коммутатора (одного L2-сегмента) приведет к недоступности половины S3-адресов (их невозможно анонсировать в другой L2-сегмент).

* **Особенности**
    * Для исключения SPOF необходимо подключение к четырем коммутаторам.


## Четыре линка, два LACP, разделение трафика S3 и internal {#four-links-two-lacp}

* **Конфигурация**
    * Настраиваются оба VXLAN-интерфейса.
    * Используются четыре S3 IP-адреса.

* **Точки отказа**
    * Отказ одного линка не влияет на работу системы.
    * Отказ любого члена (или коммутаторов MLAG) не влияет на работу системы.
    * SPOF: если физически используется один стек или коммутатор.

* **Особенности**
    * Требуется один стек или два MLAG-коммутатора. Это наиболее приемлемый вариант без использования BGP/RIP/OSPF/ExtLB на стороне клиента.


## Четыре линка, в две L2-сети, разделение трафика S3 и internal {#four-links-two-l2}

* **Конфигурация**
    * Настраиваются оба VXLAN-интерфейса.
    * Используются восемь S3 IP-адресов.

* **Точки отказа**
    * Отказ одного линка не влияет на работу системы.
    * SPOF: отказ одного коммутатора (одного L2-сегмента) приведет к недоступности половины S3-адресов (их невозможно анонсировать в другой L2-сегмент).

* **Особенности**
    * Для исключения SPOF необходимо подключение к четырем коммутаторам.
