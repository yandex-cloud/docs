# Избыточность данных

Для защиты данных On-premises Yandex Object Storage использует один из двух методов избыточности: [избыточное кодирование](#erasure-coding) или [репликацию](#replication). Выбор метода определяет размер фрагмента данных и количество копий в кластере. Репликация обеспечивает более высокую производительность, а избыточное кодирование позволяет эффективнее использовать дисковое пространство.

Object Storage автоматически поддерживает необходимый уровень избыточности: контролирует количество копий фрагментов данных или блоков четности и обеспечивает их актуальность. При недоступности узла хранилища данные с него перераспределяются по исправным узлам. Когда узел снова становится доступен, устаревшие копии на нем обновляются.


## Избыточное кодирование (Erasure Coding) {#erasure-coding}

При использовании избыточного кодирования:

1. Входящий поток данных разбивается на фрагменты.
1. Каждый фрагмент разбивается на **X** блоков размером 1 МБ и создается **Y** блоков четности для обеспечения избыточности.
1. Все блоки распределяются по узлам хранилища, количество которых должно быть больше или равно **X + Y** — по одному блоку на узел.
1. На узлах хранилища блоки хранятся в обычных фрагментах по 256 МБ (для хранения резервных копий используются фрагменты размером 2 ГБ), но такие фрагменты не реплицируются, поскольку избыточность уже достигнута. Кластер может выдержать отказ любых **Y** узлов хранилища без потери данных.

Значения **X** и **Y** указаны в названиях режимов помехоустойчивого кодирования. Например, в режиме **5+2** каждый фрагмент данных разбивается на пять блоков размером 1 МБ и создаются два блока четности размером 1 МБ.


## Репликация {#replication}

При использовании репликации:

1. Входящий поток данных разбивается на фрагменты размером 256 МБ.
    Для хранения резервных копий используются фрагменты размером 2 ГБ.

1. Каждый фрагмент реплицируется.
1. Реплики размещаются на разных узлах хранилища так, чтобы на каждом узле хранилась только одна реплика определенного фрагмента.

Высокая производительность репликации сокращает время восстановления избыточности данных в кластере после выхода из строя одного из его узлов. Производительность репликации зависит от следующих условий:

* Количество доступных узлов хранилища.
    Репликация выполняется параллельно, поэтому чем больше доступно источников и мест назначения репликации, тем выше ее скорость.

* Производительность дисков на узлах хранения.
* Производительность сети.
    Все реплики перемещаются между узлами хранилища по сети. Например, пропускная способность 1 Гбит/с может стать узким местом системы.

* Распределение данных в кластере. На некоторых узлах хранения может быть больше данных для репликации, чем на других, что может привести к перегрузке этих узлов.
* Активность ввода-вывода в кластере во время репликации.

Поддерживаются следующие режимы записи данных:
* Синхронная запись в три копии.
* Кворумная запись: две копии записываются синхронно, третья — асинхронно. Этот режим используется по умолчанию.
