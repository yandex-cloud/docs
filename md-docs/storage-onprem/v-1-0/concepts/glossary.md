# Термины и сокращения

В продукте On-premises Yandex Object Storage используются следующие основные понятия:

Понятие | Описание
------- | --------
Бакет                                   | Выделенная часть хранилища Object Storage для пользовательских данных.
Блокировка версии объекта (object lock) | Защита версии объекта от удаления или перезаписи.
Версионирование бакета                  | Средство хранения нескольких копий объекта в бакете.
Жизненные циклы объектов в бакете       | Правила автоматического удаления объектов и изменения их класса хранилища.
Загрузка данных через HTML-форму        | Способ загрузки объектов в хранилище из браузера. Позволяет анонимным пользователям загружать объекты в бакет.
Капл                                    | Ячейка для записи данных в Object Storage. Все части бинарных данных всегда либо пишутся в капл, либо читаются из капла. Размер капла зависит от класса избыточности: например, три бэкенда для репликации `x3` или шесть бэкендов для `EC 4+2`.
Квоты и лимиты                          | Ограничения на использование продукта.
Класс хранилища                         | Возможность удешевить хранение объектов, разделив их на объекты с частым доступом и объекты долгого хранения.
Метки                                   | Пары ключ-значение для логической маркировки бакетов.
Объект                                  | Файл с данными в произвольном формате.
Подписанный (pre-signed) URL            | Способ, с помощью которого анонимному пользователю можно выдать доступ к операции с хранилищем.
Политика доступа                        | Механизм управления правами на действия с объектами и бакетами.
Резервное копирование                   | Организация резервного копирования информации, хранящейся в Object Storage.
Составная (multipart) загрузка          | Способ загрузки больших объектов.
Список управления доступом              | Механизм предоставления доступа к объектам и бакетам.
Тенант                                  | Логическая структура, которая обеспечивает изоляцию бакетов и пользователей. Внутри тенанта создаются бакеты, сервисные учетные записи и статические ключи доступа для работы с S3 API. По умолчанию пользователь имеет доступ к бакетам только в своем тенанте. При необходимости можно настроить доступ к другим тенантам.
Частичное изменение объекта             | Механизм частичного изменения и дозаписи объектов.
Шифрование                              | Алгоритм шифрования данных.
