# Правила тарификации для Yandex DataLens



Чтобы рассчитать стоимость использования сервиса, ознакомьтесь с тарифами в этом разделе.

Цены на продукты сервиса также доступны в [Прайс-листе](https://yandex.cloud/ru/price-list).


Все цены в рублях и тенге указаны с НДС, все цены в долларах — без НДС.

## Тарифный план DataLens {#effective-rules}

В сервисе Yandex DataLens действует единый тарифный план: бесплатно для индивидуального использования, а для командной работы оплата ведется по количеству [рабочих мест](pricing.md#seats).

### Рабочие места {#seats}

* Каждому пользователю Yandex DataLens требуется рабочее место, даже если он только просматривает или редактирует дашборды.
* Рабочие места выдаются автоматически при первом входе в сервис новых пользователей.
* Администратор может управлять рабочими местами. Подробнее в [статье](settings/seats.md).
* Доступ к объектам в Yandex DataLens регулируется с помощью [ролей или прав доступа](security/index.md), рабочее место не предоставляет доступ к объектам.

{% note info %}

По умолчанию в экземпляре активна автоматическая покупка рабочих мест: при первом входе пользователя в сервис ему сразу будет назначено рабочее место. Администратор может [отключить эту опцию](settings/seats.md#purchase-automatically) в настройках, и тогда новые пользователи смогут получать рабочие места сразу при входе только из числа свободных рабочих мест. Если таких не окажется, администратор должен будет вручную назначать новым пользователям рабочие места.

{% endnote %}

Подсчет рабочих мест ведется в рамках месяца, независимо от активности пользователей. Если в экземпляре больше одного рабочего места, общая стоимость списывается из расчета:

> Количество рабочих мест × Стоимость одного рабочего места

Сокращение количества рабочих мест можно запланировать в любой день, но изменение вступит в силу только в первый день следующего календарного месяца.

Порядок оказания технической поддержки определяется в соответствии с выбранным [тарифным планом поддержки Yandex Cloud](../support/overview.md).

### Ограничение числа запросов из непублично встроенных чартов {#packet-queries}

От количества рабочих мест зависит лимит на количество запросов из [непублично встроенных чартов](security/private-embedded-objects.md) (чартов в визарде, Editor, QL-чартов) в пределах экземпляра DataLens. Для непублично встроенных дашбордов суммируются запросы из добавленных на него чартов.

При этом обычные запросы, производимые пользователями в интерфейсе DataLens, а также запросы из [публично встроенных чартов](concepts/datalens-public.md) не ограничены.

* Ограничение на число запросов из непублично встроенных чартов: 2 000 запросов в месяц на одно рабочее место. Это ограничение действует не для каждого отдельно взятого рабочего места, а для всего экземпляра DataLens из расчета:

   > 2 000 запросов × Количество рабочих мест

* Считаются только успешно выполненные запросы на получение данных для отрисовки чартов. Если для отрисовки одного чарта выполняется несколько запросов, это учитывается как один запрос.

* Рабочие места не связываются со встроенными чартами напрямую, их необязательно назначать внешним пользователям, которые просматривают эти чарты. Значение имеет только общее количество мест, купленное в пределах экземпляра, как свободных, так и назначенных пользователям.

   Таким образом, чем активнее используется экземпляр DataLens (чем больше в нем куплено рабочих мест), тем выше будет лимит на непубличные встраивания.

* Если с DataLens работают несколько пользователей, но при этом производится много запросов из непубличных встраиваний, потребуется докупить рабочие места, исходя из расчета выше.

* Учитывается каждый запрос за данными из непублично встроенного чарта. Например, у вас встроен дашборд с 10 чартами и 5 селекторами. Тогда при обновлении страницы с дашбордом будет учитываться 10 запросов.
   
   [Оценить количество запросов из непублично встроенных чартов](settings/seats.md#embedded-queries-count) можно с помощью подключения [Usage Analytics](operations/connection/create-usage-tracking.md). Если общее количество таких запросов в экземпляре будет превышать ограничение, может потребоваться покупка дополнительных рабочих мест.
   
   При превышении лимита на количество запросов из непублично встроенных чартов и включенной [автоматической покупкой](settings/seats.md#purchase-automatically) рабочих мест автоматическая докупка новых рабочих мест не происходит.

### Пробный период {#trial}

Для новых пользователей DataLens действует пробный период — 15 дней, в течение которых командная работа будет бесплатной. При завершении пробного периода потребуется привязать платежный аккаунт для оплаты рабочих мест или сократить потребление в экземпляре до одного рабочего места.

* Если привязать платежный аккаунт до завершения пробного периода, то пробный период будет продлен еще на 15 дней, суммарно до 30 дней.
* Если не привязать платежный аккаунт, то все рабочие места в экземпляре, кроме одного, станут неактивными. В этом случае сокращение будет автоматически запланировано на дату окончания пробного периода, и его нельзя изменить до привязки платежного аккаунта.
* После привязки платежного аккаунта при завершении пробного периода начнется автоматическое платное потребление в соответствии с [числом рабочих мест](settings/seats.md#number-of-seats), которое отображается в настройках сервиса. При этом автоматическое сокращение потребления в экземпляре до одного рабочего места отменится.

Отсчет дней до окончания пробного периода отображается в индикаторе на боковой панели.

Пробный период можно использовать только один раз в рамках одного экземпляра DataLens: он включается автоматически при активации экземпляра. При завершении пробного периода цена за первый месяц использования рассчитывается пропорционально количеству оставшихся в месяце дней.

## Цены для региона Россия {#prices}

{% note info %}

Цены на ресурсы Yandex Cloud в разных регионах различаются. Подробнее о доступных регионах в разделе [Регионы](../overview/concepts/region.md).

Валюта, которой можно оплачивать ресурсы, зависит от юридического лица, с которым пользователь заключил договор. Подробнее о регистрации аккаунта в разделе [Регистрация аккаунта в Yandex Cloud](../billing/quickstart/index.md).

{% endnote %}

Стоимость использования DataLens зависит от количества рабочих мест в экземпляре. Для индивидуального использования, когда в экземпляре 1 рабочее место, DataLens доступен бесплатно. Если DataLens используется для командной работы, и в экземпляре больше 1 рабочего места, то каждое рабочее место будет тарифицироваться.
