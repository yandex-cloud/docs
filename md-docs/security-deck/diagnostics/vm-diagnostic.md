# Диагностика ошибок в работе модуля Управление уязвимостями (VM)

## Access denied to resource `Id` {#account-permissions}

У сервисного аккаунта недостаточно прав на один или несколько ресурсов в области сканирования. Убедитесь, что сервисный аккаунт имеет необходимые [роли](../security/index.md) для сканирования заданных ресурсов. После этого перезапустите сканирование.

## Resource `Id` was not found in scope {#resource-not-found}

Ресурс, указанный в области сканирования, был удален или перемещен. Проверьте актуальность конфигурации области сканирования в [настройках окружения](../operations/workspaces/update.md) в разделе **Ресурсы**. Удалите отсутствующий ресурс из списка целей сканирования или восстановите его, а затем перезапустите сканирование.

## OS is not supported {#unsupported-os}

Образ использует операционную систему, для которой у модуля VM нет актуальных баз уязвимостей. Сканирование этого образа пропускается. Рассмотрите возможность обновления образа до поддерживаемой версии ОС. Если этот тип ОС для вас критичен, обратитесь в [службу поддержки](https://center.yandex.cloud/support).

## An unexpected technical error occurred while scanning image {#internal-error}

Возникла непредвиденная проблема на стороне модуля Управление уязвимостями. Если ошибка повторяется, обратитесь в [службу поддержки](https://center.yandex.cloud/support). В сообщении укажите код ошибки и идентификатор запуска сканирования.
