# Конфигурация

Конфигурация — это группа [правил](#rules), которые можно [применить к письму](#usage) при его отправке. Если конфигурация применяется к письму, к нему применяются все правила, заданные в конфигурации. Применять правила по отдельности нельзя.

## Правила {#rules}

Конфигурация может содержать следующие правила:

* **Подписки** — получение [уведомлений об операциях с письмами](notification.md), например, таких как отправка письма, задержка его доставки и т.п. Позволяет перенаправлять уведомления об операциях в другие сервисы для анализа и настраивать автоматическое реагирование на выбранные операции.

* **Настройки доставки** — настройка обязательного использования TLS-шифрования при отправке писем от Yandex Cloud Postbox до серверов получателя. Позволяет обеспечить дополнительную безопасность передачи данных и защиту содержимого писем.

* **Настройки сбора статистики** — настройка сбора статистики [открытия писем](mail-opened.md) и [переходов по ссылкам в письмах](click-tracking.md). Позволяет автоматически включить сбор [метрик](statistics.md#metrics) `Открытия` и `Переходы` и анализировать их.

## Особенности доставки писем при включении обязательного TLS-шифрования {#delivery-options}

При оправке письма от клиента сервису Yandex Cloud Postbox всегда используется защищенное соединение по протоколу TLS. Протокол передачи зависит от выбранного способа отправки письма:
* HTTPS — если используется [API](../api-ref/authentication.md), совместимый с API Amazon;
* SMTP поверх TLS — если используется порт `465`;
* STARTTLS — если используется порт `587`.

Если опция **Настройки доставки**:
* не включена, Yandex Cloud Postbox все равно будет пытаться использовать TLS при отправке писем. Если при отправке письма возникнет ошибка TLS или сервер получателя не поддерживает TLS, письмо будет отправлено без шифрования.
* включена, Yandex Cloud Postbox будет требовать неистекшего сертификата TLS, выпущенного доверенным центром сертификации, и поддержку минимальной версии TLS, указанной в настройках конфигурации. Если сервер получателя не соответствует этим требованиям, письмо не будет доставлено, и вы получите уведомление о недоставке.

{% note warning %}

Перед тем как включить опцию, убедитесь, что серверы получателей поддерживают необходимые требования TLS. Если вы не уверены, что все серверы получателей настроены корректно, и хотите приоритизировать доставляемость писем, то эту опцию лучше не включать.

{% endnote %}

## Применение конфигурации к письму {#usage}

Применить конфигурацию к письму можно следующими способами:

* [привязав конфигурацию к адресу](../operations/bind-configuration.md) — тогда конфигурация будет применяться ко всем письмам, которые отправляются с него.

    Конфигурацию можно привязать только к адресу, для которого [подтверждено владение доменом](../operations/check-domain.md). К одному адресу можно привязать только одну конфигурацию, но одну конфигурацию можно привязать к нескольким адресам.

* передав имя конфигурации в заголовке `X-Ses-Configuration-Set`.

    Конфигурация должна находиться в том же каталоге, в котором находится адрес. Если конфигурация с указанным именем не будет найдена в каталоге, в котором находится адрес, при отправке письма возникнет ошибка. Конфигурация, переданная в заголовке `X-Ses-Configuration-Set`, переопределяет конфигурацию, привязанную к адресу.

* передав имя конфигурации в поле `ConfigurationSetName`, если используется [метод `SendEmail`](../aws-compatible-api/api-ref/send-email.md) AWS совместимого API.

    Конфигурация должна находиться в том же каталоге, в котором находится адрес. Если конфигурация с указанным именем не будет найдена в каталоге, в котором находится адрес, при отправке письма возникнет ошибка. Конфигурация, переданная в поле `ConfigurationSetName`, переопределяет конфигурацию, привязанную к адресу.

{% note info %}

Настройки конфигурации применяются к письму в момент его отправки. Если вы измените настройки конфигурации после отправки письма, это не повлияет на уже отправленные, но еще не доставленные письма.

{% endnote %}
