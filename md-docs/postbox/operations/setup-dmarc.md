# Настройка DMARC-политики

Чтобы защитить домен от подделки отправителей и повысить безопасность почтовых сообщений, необходимо настроить [DMARC-политику](https://ru.wikipedia.org/wiki/DMARC). Настройки вашей доменной зоны должны содержать TXT-запись `_dmarc.<домен>.`. Вы можете добавить запись у вашего регистратора или в сервисе [Yandex Cloud DNS](../../dns/index.md), если вы делегировали ваш домен.

**Пример создания DMARC-записи в Cloud DNS**

{% list tabs group=instructions %}

- Консоль управления {#console}

    1. В [консоли управления](https://console.yandex.cloud) выберите каталог, в котором находятся адрес и ваша доменная зона.
    1. Перейдите в сервис **Cloud DNS**.
    1. Выберите вашу доменную зону.
    1. Нажмите кнопку **Создать запись**.
    1. В поле **Имя** укажите `_dmarc`.

        {% note info %}

        Для других DNS-сервисов может потребоваться ввести имя записи целиком. Итоговая запись должна называться `_dmarc.<домен>.`.

        {% endnote %}

    1. В списке **Тип** выберите `TXT`.
    1. В поле **Значение** введите корректное значение DMARC-политики. Обратите внимание, что значение записи нужно взять в кавычки, например:

        ```text
        "v=DMARC1;p=none"
        ```

        Данная DMARC-запись означает, что если письмо не пройдет проверку DMARC, почтовый сервис не предпримет никаких действий, например не отклонит письмо или не отправит его в спам.

    1. Нажмите кнопку **Создать**.

    Ответы DNS-сервера кешируются, поэтому возможны задержки при обновлении ресурсной записи.
