# DNS-подключения

Для разрешения FQDN в [облачных сетях (VPC)](../../vpc/concepts/network.md#network) обычно используется облачный [DNS Resolver](dns-resolver.md), который обрабатывает DNS-запросы от облачных ресурсов. Резолвер доступен для использования только изнутри виртуальных машин и его нельзя использовать за пределами ВМ.

В настоящее время доступен только один тип DNS-подключений — входящее DNS-подключение (DNS Inbound).

В одной [виртуальной сети](../../vpc/concepts/network.md#network) можно создать не более одного входящего DNS-подключения.

## Входящее DNS-подключение {#dns-inbound}

*Входящее DNS-подключение* — это возможность использовать облачный DNS-резолвер на уровне облачной сети или подсети.

Создание входящего DNS-подключения в облачной сети дает возможность корпоративным DNS-серверам отправлять запросы для [внутренних зон Yandex Cloud](dns-zone.md#private-zones). Для этого корпоративные DNS-серверы должны быть настроены таким образом, чтобы DNS-запросы к [внутренним зонам Yandex Cloud](dns-zone.md#private-zones) отправлялись на IP-адрес созданного в VPC входящего DNS-подключения.
