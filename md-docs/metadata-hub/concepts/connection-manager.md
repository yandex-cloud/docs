# Подключение


С помощью {{ connection-manager-full-name }} вы можете управлять параметрами [подключений](connection-manager.md) к базам данных. Подключения создаются автоматически при создании кластера управляемых баз данных в {{ yandex-cloud }} для кластеров {{ PG }}, {{ CH }}, {{ MY }}, {{ VLK }} и {{ SD }}. Для остальных типов кластеров можно [создать подключение](../operations/create-connection.md) самостоятельно.
Кластера, созданные до появления возможности интеграции с {{ connection-manager-name }}, будут работать в старом режиме. Для них интеграцию можно включить отдельно в [дополнительных настройках каждого кластера](../quickstart/connection-manager.md#mdb-integration).
Подключения и секреты, которые создаются автоматически при создании кластера, нельзя ни редактировать, ни удалять — они изменяются автоматически при редактировании настроек пользователя в кластере управляемой базы данных.

Подключение содержит всю информацию о параметрах соединения с базой данных. Конфиденциальная часть этой информации, такая как пароль пользователя для доступа к базе данных, хранится в сервисе [{{ lockbox-full-name }}](../../lockbox/index.md) в виде [секрета](secret.md).


![backups](../../_assets/metadata-hub/conn-man-diagramm.svg)

Вы также можете [создать новые подключения](../operations/create-connection.md#on-premise-connection) с безопасным хранением пароля к пользовательским инсталляциям базы данных и использовать их в доступных сервисах {{ yandex-cloud }}.

{% note tip %}

Если вы создаете подключение к пользовательской инсталляции базы данных для использования с [{{ datalens-full-name }}](../../datalens/concepts/index.md), указывайте внешний адрес хоста.

{% endnote %}

Вы можете [настроить доступ к подключениям](../security/index.md) с помощью ролей {{ connection-manager-full-name }}. Роли можно назначать на [облако](../../resource-manager/concepts/resources-hierarchy.md#cloud), [каталог](../../resource-manager/concepts/resources-hierarchy.md#folder) или на отдельное [подключение](../operations/connection-access.md). Они будут действовать и на вложенные ресурсы.

По способу управления подключения бывают двух типов:

* _управляемое_ — созданное автоматически кластером управляемой базы данных;
* _ручное_ — созданное пользователем или сторонним сервисом.

## Зависимости {#dependencies}

У подключений могут быть _зависимости_, которые указывают на сервисы, использующие или управляющие подключениями. Зависимости [отображаются](../operations/view-connection.md#dependencies) в отдельном разделе для каждого подключения. Зависимость бывает двух типов: на использование или на управление подключением (для управляемых подключений).

Подключение, [созданное вручную](../quickstart/connection-manager.md#database-connections), нельзя удалить, пока у него есть хотя бы одна зависимость.
[Управляемое подключение](../quickstart/connection-manager.md#mdb-integration) нельзя отредактировать и удалить вручную, так как они содержат актуальные данные по подключению к создавшему их кластеру.

Примеры зависимостей:

#|
|| Зависимость | Тип ||
|| Кластеры управляемых баз данных, при создании которых подключения были автоматически добавлены | На управление ||
|| Резервные копии кластеров управляемых баз данных, которые используют подключение. После удаления кластера, создавшего подключение, его резервные копии хранятся неделю, поэтому подключение не удаляется сразу вместе с кластером | На использование ||
|#
