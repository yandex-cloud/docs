# Запросы к наборам данных с помощью Yandex WebSQL


{% note info %}

Эта функциональность находится на стадии [Preview](../../../overview/concepts/launch-stages.md).

{% endnote %}


С помощью сервиса [Yandex WebSQL](../../../websql/index.md) вы можете отправлять SQL-запросы к базам данных в кластерах, которые являются источниками данных для Data Catalog.

WebSQL — это сервис Yandex Cloud, который позволяет подключаться к кластерам управляемых баз данных, работать с БД, таблицами и схемами и выполнять запросы. Сервис работает в браузере, не требует дополнительной авторизации и предлагает удобные подсказки для работы с SQL-командами.

Подробнее о работе с WebSQL в [документации сервиса](../../../websql/operations/index.md).

Чтобы выполнить SQL-запрос:

{% list tabs group=instructions %}

- Консоль управления {#console}

    1. В [консоли управления](https://console.yandex.cloud) выберите [каталог ресурсов](../../../resource-manager/concepts/resources-hierarchy.md#folder), в котором создан каталог метаданных.
    1. [Перейдите](../../../console/operations/select-service.md#select-service) в сервис **Yandex MetaData Hub**.
    1. На панели слева выберите ![image](../../../_assets/console-icons/folder-magnifier.svg) **Data Catalog**.
    1. В открывшемся списке выберите каталог метаданных.
    1. На панели слева выберите ![image](../../../_assets/console-icons/database-magnifier.svg) **Поиск по метаданным**.
    1. Перейдите в объект метаданных с типом **Набор данных** одним из способов:

        * Нажмите на имя объекта с типом **Набор данных**.
        * Нажмите на имя объекта с типом, отличным от **Набор данных**. С помощью вкладки **Дочерние** переходите к дочерним объектам, пока не будет найден объект с типом **Набор данных**. Нажмите на имя этого объекта.

    1. В поле **Данные** нажмите ссылку **Открыть в WebSQL**.
    1. Выберите нужное подключение к базе данных кластера-источника. Будет выполнен переход в сервис WebSQL.
    1. В правой верхней панели введите SQL-запрос и нажмите **Выполнить**.

        С помощью команд SQL можно, например:

        * [выполнять мультизапросы](../../../websql/operations/query-executor.md#multiple-queries);
        * [выгружать результаты выполнения запроса](../../../websql/operations/query-executor.md#export-results);
        * [использовать AI-ассистента для анализа и оптимизации запросов](../../../websql/operations/ai-assistant.md);
        * [сохранять запрос в качестве шаблона и делиться им с другими пользователями](../../../websql/operations/templates.md#save-query);
        * [использовать запросы из истории запросов](../../../websql/operations/history.md).

{% endlist %}

{% note info %}

Если источником данных является кластер Yandex Managed Service for PostgreSQL, то c помощью команд SQL нельзя выполнять действия, требующие прав суперпользователя.

{% endnote %}
