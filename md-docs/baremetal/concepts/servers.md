# Серверы

Сервер BareMetal — это физический сервер, подключенный к [сети](network.md) двумя или более [сетевыми интерфейсами](#network-interfaces) с пропускной способностью 1, 10 или 25 Гбит/с. Аппаратные и сетевые ресурсы сервера физически изолированы и доступны только арендовавшему их пользователю.

Сервер может предоставляться пользователю в одной из [готовых конфигураций](server-configurations.md), в [своей конфигурации](server-custom-configurations.md), собранной по [запросу](../operations/servers/custom-server-lease.md) пользователя, или в [конфигурации под заказ](server-individual-configurations.md) с индивидуальными аппаратными характеристиками.

{% note warning %}

На серверах BareMetal по умолчанию используются оптимальные системные настройки [BIOS](https://ru.wikipedia.org/wiki/BIOS). Изменять эти настройки не рекомендуется.

{% endnote %}

На аппаратном уровне серверы BareMetal управляются через интерфейс [IPMI](https://ru.wikipedia.org/wiki/Intelligent_Platform_Management_Interface) с помощью контроллера Baseboard Management Controller (BMC).

Изменение пользователем любых настроек контроллера BMC вызовет:
* недоступность [KVM-консоли](../operations/servers/server-kvm.md) для управления сервером;
* невозможность остановить, запустить или перезапустить сервер через [консоль управления](https://console.yandex.cloud), Yandex Cloud [CLI](../../cli/cli-ref/baremetal/cli-ref/index.md) или [API](../../api-design-guide/index.md);
* невозможность переустановить операционную систему сервера;
* невозможность просматривать аппаратные характеристики работы сервера (температура процессоров, скорость вращения вентиляторов и т.п.).

{% note alert %}

При изменении пользователем настроек контроллера BMC сервис Yandex BareMetal не гарантирует работоспособность KVM-консоли и других инструментов управления сервером.

{% endnote %}

## Аренда серверов {#server-lease}

Сервер в готовой конфигурации можно арендовать на срок 1 день, 1, 3, 6 месяцев или 1 год. Конфигурации, доступные для аренды в каждом [пуле](#server-pools), можно посмотреть при заказе сервера в [консоли управления](https://console.yandex.cloud).

Сервер в [своей конфигурации](server-custom-configurations.md) можно арендовать на срок 1 месяц, 6 месяцев или 1 год.

Сроки и условия аренды сервера в [конфигурации под заказ](server-individual-configurations.md) определяются индивидуально.

## Доступ на сервер {#server-access}

Управлять сервером можно с помощью [KVM-консоли](../operations/servers/server-kvm.md), которая доступна в [консоли управления](https://console.yandex.cloud).

## Пулы серверов {#server-pools}

Пулы — инфраструктурно обособленные модули ЦОД, в которых физически размещаются серверы. Пулы распределены по [зонам доступности](../../overview/concepts/geo-scope.md), находящимся в [регионах](../../overview/concepts/region.md). В настоящее время сервис Yandex BareMetal доступен только в регионе Россия.

| Регион        | Зона доступности | Пулы                               |
|---------------|------------------|------------------------------------|
| `ru-central1` | `ru-central1-m`  | `ru-central1-m3`<br>`ru-central1-m4` |

## Резервирование питания серверов {#server-power-supply}

При аренде сервера можно подобрать конфигурации по типу используемой схемы резервирования питания:

* Серверы с одним блоком питания (`1 PSU`).
* Серверы с двумя и более блоками питания (`2+ PSU`).
* Серверы на платформе [Open Compute Project (OCP)](https://ru.wikipedia.org/wiki/Open_Compute_Project). В таких серверах вместо отдельных блоков питания используется общая шина питания для всех серверов в стойке, которая предоставляет резервируемое питание для каждого сервера.

## Сетевые интерфейсы серверов {#network-interfaces}

Любая [конфигурация](server-configurations.md) сервера BareMetal включает в себя как минимум два [сетевых интерфейса](https://ru.wikipedia.org/wiki/Сетевая_плата):

* сетевой интерфейс, подключенный к [публичной сети](public-network.md);
* сетевой интерфейс, подключенный к [приватной сети](private-network.md).

При этом сетевых интерфейсов может быть больше двух:
* В конфигурациях с сетевыми картами, поддерживающими скорость соединения 10 Гбит/с и 25 Гбит/с, кроме основных (высокоскоростных) сетевых интерфейсов, средства диагностики сервера могут отображать еще два (неактивных) сетевых интерфейса.

    Эти неактивные интерфейсы принадлежат встроенным сетевым картам материнской платы сервера. Они не подключены к сетям, и настраивать их не нужно.
* В конфигурациях с резервированием сетевого подключения [MC-LAG](mc-lag.md) сервер подключен к каждой из сетей не через один интерфейс, а одновременно через два сетевых интерфейса.

    Сетевые интерфейсы, подключенные к одной сети, на уровне операционной системы объединены в группы агрегирования, что увеличивает скорость соединения с сетью и обеспечивает отказоустойчивость.

{% note info %}

К сетевому интерфейсу (или группе агрегирования MC-LAG) сервера, которые подключены к [приватной сети](private-network.md), можно [привязать](../operations/servers/set-up-tagged-vlan.md) одновременно несколько [приватных подсетей](private-network.md#private-subnet).

{% endnote %}

## Статусы серверов {#server-status}

Сервер может находиться в одном из следующих статусов:

* `PROVISIONING` — сервер готовится к предоставлению в аренду пользователю. Сервер готовой конфигурации выдается за время от 30 секунд до нескольких минут. Сервер своей конфигурации требует на сборку и запуск до 4 календарных дней.
* `RUNNING` — сервер предоставлен в аренду и полностью доступен пользователю. В этом статусе электропитание сервера может быть включено или выключено.
* `STARTING` — на сервере включается электропитание.
* `STOPPING` — на сервере выключается электропитание.
* `RESTARTING` — сервер перезагружается.
* `REINSTALLING` — на сервере переустанавливается операционная система.
* `UPDATING` — происходит обновление настроек сервера или сети.
* `QUARANTINING` — сервер переводится в [карантин](#quarantine).
* `QUARANTINED` — сервер в карантине и недоступен пользователю по сети и через KVM-консоль.
* `RETURNING_FROM_QUARANTINE` — сервер возвращается из карантина, восстанавливается доступ по сети и через KVM-консоль.
* `DELETING` — сервер удаляется. На полное удаление сервера и очистку всех дисков может потребоваться несколько часов.
* `ERROR` — произошла ошибка, KVM-консоль недоступна. Обратитесь в [техническую поддержку](https://center.yandex.cloud/support).

## Карантин {#quarantine}

Карантин — промежуточное состояние сервера перед полным прекращением его аренды. В этом состоянии сервер все еще находится в пользовательском каталоге. К серверу на карантине нет доступа по сети и через KVM-консоль, при этом на сервере сохраняются все пользовательские данные и настройки. Срок карантина — 72 часа. По истечении этого времени c сервера полностью удаляются настройки и данные, а сам сервер удаляется из каталога пользователя.

В течение всего срока карантина можно восстановить сервер для дальнейшего использования. Подробнее на странице [Восстановить сервер из карантина](../operations/servers/restore-from-quarantine.md).

{% note info %}

Серверы в статусах `QUARANTINING`, `QUARANTINED`, `RETURNING_FROM_QUARANTINE` и `DELETING` учитываются в [общей квоте](limits.md) на количество серверов.

Серверы в статусах `QUARANTINING`, `QUARANTINED`, `RETURNING_FROM_QUARANTINE` и `DELETING` не тарифицируются.

Если в течение срока карантина сервер будет восстановлен для дальнейшего использования, то период времени, в течение которого этот сервер находился на карантине, будет включен в оплачиваемый срок аренды. 

{% endnote %}
