# Начало работы с логами ресурсов Yandex Cloud

Monium позволяет работать с логами сервисов Yandex Cloud, а также с любыми другими логами, загруженными в Monium. Например, из ваших приложений, вашей инфраструктуры или других облаков.

Эта инструкция поможет вам посмотреть логи, которые передают ресурсы Yandex Cloud.

{% note info %}

Как настроить передачу логов для ресурсов Yandex Cloud, см. в документации нужного ресурса. Например, настроить логирование кластера Apache Airflow™ можно при [создании](../../managed-airflow/operations/cluster-create.md) или [изменении](../../managed-airflow/operations/cluster-update.md) кластера.

{% endnote %}

Чтобы посмотреть логи:

1. Создайте ресурс Yandex Cloud и настройте его логирование.
   
   Список сервисов, которые поддерживают передачу логов в Monium, см. в разделе [Observability-инструменты мониторинга и логирования](../../overview/concepts/monitoring-logging-tools.md).

1. На главной странице [Monium](https://monium.yandex.cloud) слева выберите раздел **Логи**.
1. Вверху на шкале времени задайте промежуток поиска данных.
1. В строке поиска выберите метки запроса:

   * `project = folder__<идентификатор_каталога>` — идентификатор каталога, в котором находится ресурс Yandex Cloud.
   * `service = <лог_группа>` — имя или идентификатор лог-группы, в которую передаются логи. Например, `default`.
   * `resource_type = <тип_ресурса>` — тип ресурса Yandex Cloud, который передает логи. Например, `managed-airflow.cluster`, `k8s.cluster`, `alb.loadBalancer`.
   * `resource_id = <идентификатор_ресурса>` — идентификатор ресурса Yandex Cloud, который передает логи. Например, идентификатор кластера.
   
     Если не указать тип ресурса, будут показаны все логи, которые передаются в лог-группу. Если не указать идентификатор ресурса, будут показаны все логи для ресурсов заданного типа.

1. Чтобы отфильтровать логи по другим параметрам, выберите нужные метки из списка. Например, метка `level = WARN` покажет все логи с уровнем WARNING (предупреждение).

   Подробнее о составлении запросов см. в разделах [Модель данных в Monium](../concepts/data-model.md) и [Язык запросов в Monium](../concepts/querying.md).

1. Чтобы сразу просмотреть логи другого ресурса, нажмите **Добавить запрос** и укажите параметры нужного ресурса Yandex Cloud.
1. Нажмите **Выполнить запрос**.

Подробнее о работе с логами см. раздел [Поиск и анализ логов](logs-explorer.md).

## Сервисный дашборд для логов {#logs-service-dashboard}

Система автоматически собирает [метрики о поступающих логах](logs-metrics.md) — объем данных, задержки доставки, количество ошибок по уровням и так далее.

Графики о поступивших логах доступны на сервисном дашборде.
