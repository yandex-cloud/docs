# Monium Traces

Monium Traces — система распределенной трассировки в платформе Monium. Она позволяет отслеживать путь выполнения запросов через сервисы и компоненты вашей инфраструктуры: находить причины ошибок, выявлять узкие места и анализировать производительность.

Monium Traces принимает данные по протоколу [OpenTelemetry](https://opentelemetry.io/) (OTLP). Для отправки трейсов подходит любой совместимый инструмент: OpenTelemetry Collector, SDK и другие агенты.

## Какие задачи решает трейсинг {#use-cases}

В **микросервисной архитектуре** трейсинг помогает:

* **Локализовать ошибки** — определить, какой из множества сервисов вызвал сбой.
* **Анализировать производительность** — найти сервис или компонент, который замедляет выполнение запроса.
* **Отслеживать путь запроса** — увидеть, через какие сервисы и базы данных прошел запрос.
* **Разбирать редкие события** — понять, что происходит в момент аномального поведения системы.

В **монолитной архитектуре** трейсинг позволяет:

* **Анализировать внутреннюю структуру запросов** — отследить выполнение через слои логики (контроллеры, бизнес-логика, репозитории) и найти узкие места.
* **Диагностировать периодические сбои** — выявить причину нестабильного поведения, которое сложно воспроизвести.

## Настройка подключения {#connection-setup}

Для отправки трейсов в Monium Traces используйте любой инструмент, совместимый с протоколом OpenTelemetry (OTLP). Например, OpenTelemetry Collector или OpenTelemetry SDK.

Если вы уже знакомы с OpenTelemetry и работаете с облаком Yandex Cloud, для подключения достаточно параметров ниже.

Если вы только начинаете работу с трейсингом, воспользуйтесь [руководством по настройке поставки данных](../collector/index.md) — там описана установка агента, конфигурация и примеры для отправки телеметрии в Monium.

### Параметры подключения {#connection}

* Формат поставки данных в Monium — [OpenTelemetry (OTLP)](https://opentelemetry.io/docs/).

* Эндпоинт: `ingest.monium.yandex.cloud:443`.

* Авторизация: [сервисный аккаунт](../../iam/operations/sa/create.md) с ролью `monium.traces.writer` и [API-ключ](../../iam/operations/iam-token/create-for-sa.md) с областью действия `yc.monium.traces.write`.

* Заголовок проекта: `x-monium-project=folder__<идентификатор_каталога>`.

* Атрибуты ресурса: `service.name` (имя приложения) и `cluster` (окружение, по умолчанию `default`).

## Просмотр трейсов {#view-traces}

1. На главной странице [Monium](https://monium.yandex.cloud) слева выберите **Трейсы**.
1. Вверху укажите промежуток поиска с помощью временной шкалы, предустановленного промежутка или введите нужное значение времени.
1. Выберите режим: **Поиск по трейсам** или **Поиск по спанам**.
1. Введите [запрос](../concepts/querying.md) для поиска данных.

   По умолчанию поиск выполняется в текущем проекте — `folder__<идентификатор_каталога>`, вы можете выбрать другой.

1. Нажмите **Выполнить**.

1. Чтобы открыть отдельную трассировку, вверху справа в поле **Введите trace.id** укажите ее идентификатор.

Дополнительные возможности поиска и просмотра трейсов описаны в разделе [Поиск трейсов и спанов](operations/traces-explorer.md).
