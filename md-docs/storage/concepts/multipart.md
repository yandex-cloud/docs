# Составная (multipart) загрузка

Object Storage позволяет сохранять объекты размером в несколько терабайт (смотрите раздел [Квоты и лимиты в Object Storage](limits.md)). Чем больше объект, тем больше вероятность, что загрузка прервется ошибкой сети. Поэтому большие объекты эффективнее загружать частями меньшего размера. Такой способ сохранения объектов в Object Storage называется составная загрузка. Подробнее про поддержку составной загрузки читайте в разделе [Общий порядок составной (multipart) загрузки](../s3/api-ref/multipart.md).

Составную загрузку поддерживают многие из инструментов, перечисленных в разделе [Поддерживаемые инструменты](../tools/index.md).

В консоли управления Yandex Cloud при обзоре бакета вы можете увидеть сообщения следующего содержания:

- «N незавершенных загрузок..»
- «Дождитесь завершения загрузок или завершите/удалите их любым удобным для вас инструментом..»
- «Есть незавершенные составные загрузки..»

Любое из этих сообщений говорит о том, что в вашем бакете есть незавершенные составные загрузки. Не оставляйте эти сообщения без внимания, поскольку хранение не полностью загруженного объекта тарифицируется стандартным образом. Читайте раздел [Правила тарификации для Object Storage](../pricing.md).

Удалить незавершенную загрузку вручную можно по [инструкции](../operations/objects/deleting-multipart.md). Также вы можете настроить для бакета автоматическое удаление незавершенных загрузок в определенный момент времени — для этого создайте [жизненный цикл](lifecycles.md) (действие `AbortIncompleteMultipartUpload`, подробнее в [инструкции](../operations/buckets/lifecycles.md)).
