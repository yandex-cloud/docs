#### Description

{% note tip "**Automatic verification**" %}

This control automatically checks for Yandex Audit Trails service configuration at the organization level.

{% endnote %}

The main tool for collecting Yandex Cloud level logs is [Yandex Audit Trails](https://yandex.cloud/en/docs/audit-trails/concepts/). This service allows you to collect audit logs about events happening to Yandex Cloud resources and upload these logs to Yandex Object Storage buckets or Cloud Logging log groups for further analysis or export. For information on how to start collecting logs, see this [guide](https://yandex.cloud/en/docs/audit-trails/quickstart).

Audit Trails audit logs may contain two types of events: [management events](https://yandex.cloud/en/docs/audit-trails/concepts/events) and [data events](https://yandex.cloud/en/docs/audit-trails/concepts/events-data-plane).

[Management events](https://yandex.cloud/en/docs/audit-trails/concepts/format) are actions you take to configure Yandex Cloud resources, such as creating, updating, or deleting infrastructure components, users, or policies. [Data events](https://yandex.cloud/en/docs/audit-trails/concepts/format-data-plane) are updates and actions performed on data and resources within Yandex Cloud services. By default, Audit Trails does not log data events. You need to [enable](https://yandex.cloud/en/docs/audit-trails/quickstart#the-trail-creation) collection of data event audit logs individually for each supported service.

To learn more, see [Comparing management and data event logs](https://yandex.cloud/en/docs/audit-trails/concepts/control-plane-vs-data-plane).

To collect metrics, analyze Yandex Cloud-level events, and set up notifications, we recommend using [Yandex Monitoring](https://yandex.cloud/en/docs/monitoring). For example, it can help you track spikes in Compute Cloud workload, Application Load Balancer RPS, or significant changes in Identity and Access Management event statistics.

You can also use Monitoring to monitor the health of the Audit Trails service itself and track security events. You can export metrics to a SIEM system via the API, see this [guide](https://yandex.cloud/en/docs/monitoring/operations/metric/get).

Solution: Monitoring Audit Trails and security events using [Monitoring](https://github.com/yandex-cloud-examples/yc-audit-trails-monitoring)

You can export audit logs to [Cloud Logging](https://yandex.cloud/en/docs/logging) or Data Streams log group and to a customer's SIEM system to analyze information about events and incidents.

#### Guides and solutions

* You can enable Yandex Audit Trails at the folder, cloud, and organization level. We recommend enabling Yandex Audit Trails at the level of the entire organization. Thus you will be able to collect audit logs in a centralized manner, e.g., to a separate [security cloud](https://yandex.cloud/en/docs/audit-trails/operations/)
