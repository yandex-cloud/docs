#### Description

The rule checks whether log collection is set up for a Yandex Managed Service for GitLab instance.

The core log collection tool is [Yandex Audit Trails](https://yandex.cloud/en/docs/audit-trails/). It enables collecting audit logs about events occurring to Yandex Cloud resources and uploading these logs to an Object Storage bucket or a log group in Cloud Logging for further analysis or export.

Audit Trails events in Managed Service for GitLab are [control plane events](https://yandex.cloud/en/docs/audit-trails/concepts/format), which include creating, deleting, and modifying an instance, as well as execution events and more. For more information, see the [Audit Trails reference](https://yandex.cloud/en/docs/audit-trails/concepts/events#managed-service-for-gitlab).

#### Guides and solutions to use

**Guides and solutions to use**:

[Set up log collection](https://yandex.cloud/en/docs/audit-trails/quickstart) using Yandex Audit Trails:

1. [Create a bucket with restricted access](https://yandex.cloud/en/docs/storage/quickstart#the-first-bucket).
2. [Assign the required roles to the service accounts](https://yandex.cloud/en/docs/audit-trails/quickstart#before-you-begin).
3. [Create a trail](https://yandex.cloud/en/docs/audit-trails/quickstart#create-a-trail).

Audit Trails events in Managed Service for GitLab are [control plane events](https://yandex.cloud/en/docs/audit-trails/concepts/format), which include creating, deleting, and modifying an instance, as well as execution events and more. For more information, see the [Audit Trails reference](https://yandex.cloud/en/docs/audit-trails/concepts/events#managed-service-for-gitlab).
