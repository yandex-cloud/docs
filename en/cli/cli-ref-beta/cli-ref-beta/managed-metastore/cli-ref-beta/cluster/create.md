# yc beta managed-metastore cluster create

Creates a Metastore Cluster.

#### Command Usage

Syntax: 

`yc beta managed-metastore cluster create <FOLDER-ID>`

#### Flags

| Flag | Description |
|----|----|
|`-r`,`--request-file`|<b>`string`</b><br/>Path to a request file.|
|`--example-json`|Generates a JSON template of the request.<br/>The template can be customized and used as input for the command.<br/>Usage example:<br/><br/>1. Generate template: yc beta compute instance create --example-json > request.json<br/>2. Edit the template: vim request.json<br/>3. Run with template: yc beta compute instance create -r request.json|
|`--example-yaml`|Generates a YAML template of the request.<br/>The template can be customized and used as input for the command.<br/>Usage example:<br/><br/>1. Generate template: yc beta compute instance create --example-yaml > request.yaml<br/>2. Edit the template: vim request.yaml<br/>3. Run with template: yc beta compute instance create -r request.yaml|
|`--config-spec`|<b>`shorthand/json`</b><br/>Configuration of the Metastore Cluster.<br/>Shorthand Syntax:<br/>{<br/>resources = {<br/>resource-preset-id = str<br/>}<br/>}<br/>JSON Syntax:<br/>"{<br/>"resources": {<br/>"resource-preset-id": "str"<br/>}<br/>}"<br/>Fields:<br/>resources -> (struct)<br/>Configuration for computational resources for Metastore server instances.<br/>resource-preset-id -> (string)<br/>ID of the preset for computational resources allocated to an instance (e.g., CPU, memory, etc.).|
|`--deletion-protection`|Deletion Protection prevents deletion of the cluster.|
|`--description`|<b>`string`</b><br/>Description of the Metastore Cluster.|
|`--folder-id`|<b>`string`</b><br/>ID of the folder to create the Metastore Cluster in.|
|`--labels`|<b>`stringToString`</b><br/>Custom labels for the Metastore Cluster as '' key:value '' pairs. For example: {"env": "prod"}.|
|`--logging`|<b>`shorthand/json`</b><br/>Cloud logging configuration.<br/>Shorthand Syntax:<br/>{<br/>destination = folder-id=str \| log-group-id=str,<br/>enabled = bool,<br/>min-level = TRACE\|DEBUG\|INFO\|WARN\|ERROR\|FATAL<br/>}<br/>JSON Syntax:<br/>"{<br/>"destination": {<br/>"folder-id": "str",<br/>"log-group-id": "str"<br/>},<br/>"enabled": "bool",<br/>"min-level": "TRACE\|DEBUG\|INFO\|WARN\|ERROR\|FATAL"<br/>}"<br/>Fields:<br/>enabled -> (bool)<br/>Logs generated by the Metastore server are delivered to Cloud Logging.<br/>min-level -> (enum<DEBUG\|ERROR\|FATAL\|INFO\|TRACE\|WARN>)<br/>Minimum severity level for log entries. See [LogLevel.Level] for details.<br/>destination -> (oneof<folder-id\|log-group-id>)<br/>Oneof destination field<br/>folder-id -> (string)<br/>Logs will be written to the default log group of the specified folder.<br/>log-group-id -> (string)<br/>Logs will be written to the log group specified by its ID.|
|`--maintenance-window`|<b>`shorthand/json`</b><br/>Maintenance window.<br/>Shorthand Syntax:<br/>{<br/>policy = anytime={} \| weekly-maintenance-window={<br/>day = MON\|TUE\|WED\|THU\|FRI\|SAT\|SUN,<br/>hour = int<br/>}<br/>}<br/>JSON Syntax:<br/>"{<br/>"policy": {<br/>"anytime": {},<br/>"weekly-maintenance-window": {<br/>"day": "MON\|TUE\|WED\|THU\|FRI\|SAT\|SUN",<br/>"hour": "int"<br/>}<br/>}<br/>}"<br/>Fields:<br/>policy -> (oneof<anytime\|weekly-maintenance-window>)<br/>Oneof policy field<br/>anytime -> (struct)<br/>The cluster may be restarted for maintenance at any time.<br/>weekly-maintenance-window -> (struct)<br/>Maintenance is allowed only within the specified weekly window.<br/>day -> (enum<FRI\|MON\|SAT\|SUN\|THU\|TUE\|WED>)<br/>Day of the week when maintenance can occur.<br/>hour -> (int)<br/>Hour of the day in UTC when the maintenance window starts.|
|`--name`|<b>`string`</b><br/>Name of the Metastore Cluster. The name must be unique within the folder.|
|`--network`|<b>`shorthand/json`</b><br/>Network-related configuration options.<br/>Shorthand Syntax:<br/>{<br/>security-group-ids = str,...,<br/>subnet-ids = str,...<br/>}<br/>JSON Syntax:<br/>"{<br/>"security-group-ids": [<br/>"str", ...<br/>],<br/>"subnet-ids": [<br/>"str", ...<br/>]<br/>}"<br/>Fields:<br/>security-group-ids -> ([]string)<br/>User security groups.<br/>subnet-ids -> ([]string)<br/>IDs of VPC network subnets where instances of the cluster are attached.|
|`--service-account-id`|<b>`string`</b><br/>Service account used to access Cloud resources.|
|`--version`|<b>`string`</b><br/>Metastore server version.|
|`--async`|Display information about the operation in progress, without waiting for the operation to complete.|
