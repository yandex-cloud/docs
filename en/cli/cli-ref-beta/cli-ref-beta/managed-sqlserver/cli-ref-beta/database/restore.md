# yc beta managed-sqlserver database restore

Creates a new SQL Server database in the specified cluster from a backup.

#### Command Usage

Syntax: 

`yc beta managed-sqlserver database restore <BACKUP-ID>`

#### Flags

| Flag | Description |
|----|----|
|`-r`,`--request-file`|<b>`string`</b><br/>Path to a request file.|
|`--example-json`|Generates a JSON template of the request.<br/>The template can be customized and used as input for the command.<br/>Usage example:<br/><br/>1. Generate template: yc beta compute instance create --example-json > request.json<br/>2. Edit the template: vim request.json<br/>3. Run with template: yc beta compute instance create -r request.json|
|`--example-yaml`|Generates a YAML template of the request.<br/>The template can be customized and used as input for the command.<br/>Usage example:<br/><br/>1. Generate template: yc beta compute instance create --example-yaml > request.yaml<br/>2. Edit the template: vim request.yaml<br/>3. Run with template: yc beta compute instance create -r request.yaml|
|`--backup-id`|<b>`string`</b><br/>ID of a backup to be used.|
|`--cluster-id`|<b>`string`</b><br/>ID of the SQL Server cluster to restore a database in. To get the cluster ID, use a [ClusterService.List] request.|
|`--database-name`|<b>`string`</b><br/>Name of the SQL Server database that is being restored.|
|`--from-database`|<b>`string`</b><br/>Name of the database which backup is used to restore the database.|
|`--time`|<b>`time`</b><br/>Timestamp which is used for Point-in-Time recovery. (RFC3339)|
|`--async`|Display information about the operation in progress, without waiting for the operation to complete.|
