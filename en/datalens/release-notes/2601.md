# {{ datalens-full-name }} release notes: January 2026

* [Changes in basic features](#base)
* [Fixes and improvements](#fixes)

## Changes in basic features {#base}


* [Billing account](../settings/service-plan.md#billing-account) lists now include billing accounts associated with other [organizations](../concepts/organizations.md), provided the user was issued permissions for those organizations.


* Got the first version of the [public API](../operations/api-start.md) ready.

  
  {{ datalens-name }} provides a public API ([https://api.datalens.tech](https://api.datalens.tech)) to automate operations with dashboards, charts, datasets, and connections.

  [{{ datalens-name }} Public API](../operations/api-start.md) is a collection of methods similar to those used in the {{ datalens-name }} web interface. This API is described by the OpenAPI specification and authenticated via the Yandex Cloud IAM tokens.

  To use the API, you need an [IAM token](../../iam/operations/iam-token/create.md) and an [organization ID](../settings/index.md#service-settings).




## Fixes and improvements {#fixes}


* In [{{ datalens-gallery }}]({{ link-datalens-main }}/gallery), the **Contact the author** form now opens in a dialog instead of a new tab.
* Added the ability to [reassign](../settings/seats.md#reassign) a seat which is about to expire.
* Fixed the menu item background color swap when [customizing the UI](../settings/appearance.md#ui-customization) in some themes.


* In the [link settings](../dataset/create-dataset.md#links) window for datasets, added a tooltip and search by field name. Previously, with many long and similar field names, it was difficult to find and select the right one.
* Now, if no function is selected in the list on the left, the description window displays a tooltip and links to a user guide and dashboard with examples in the formula editor used to [create a new calculated field](../concepts/calculations/index.md#how-to-create-calculated-field) for datasets and charts.
* Fixed an issue where text from the UI was copied to the clipboard on some pages.

### Dashboard fixes {#dashboard-fixes}


* Added control over which dashboard tabs should display a [selector](../dashboard/selector.md). When switching between these tabs, the selector preserves its value.

  Select one of the following **Show in tabs** options for how the selector should be displayed on the dashboard:

  * `Current tab <tab_name>` (default for a single selector): The selector will be displayed on the specified tab.
  * `Selected tabs`: Select one or more tabs to display the selector on. By default, the current tab is selected.
  * `All tabs`: The selector will be displayed on all tabs. Available for a single selector.
  * `Based on <group_value> settings`: The selector will be displayed based on advanced settings of the widget with a group of selectors. Available only for a selector in a group of selectors.

  For a widget with multiple selectors, you can configure visibility in the **Group settings** tab.

  For more information, see [{#T}](../operations/dashboard/add-selector.md).

* Implemented preservation of selector values when switching between tabs. Now, if you set a selector value, switch to another tab, and then go back again, the selector's value will stay the same.


* Dashboards now allow you to [set up links](../operations/dashboard/dashboard-links.md) for charts on unopened widget tabs:

  * Select a chart at the top of the link setup window: once the chart's data loads, you will be able to set up a link.
  * In the link setup window, pick a selector. Click the chart's row in the **Link type** column of the widget list: once the chart's data loads, you will be able to set up a link.

* Fixed stable table rendering at initial dashboard loading.
* Fixed an issue where the color of [Mermaid](../dashboard/markdown.md#mermaid) diagrams in dashboard text widgets in the visual editor was different from their actual color on the dashboard.
* Fixed an error where, when setting up [aliases](../dashboard/link.md#alias) with some QL charts, the parameters configured in the QL chart UI did not appear in parameter selection.
* In dashboard edit mode, added a light to the ![image](../../_assets/console-icons/gear.svg) button for selectors with the `All tabs` and `Selected tabs` group visibility settings. 


### Fixes in charts {#chart-fixes}

* In [line chart](../visualization-ref/line-chart.md), fixed the line shape setup issue for multiple measures, where only one measure had its line shape rendered correctly.
* In [maps](../visualization-ref/map-chart.md), fixed the drag and drop issue when adding a field to the `Layer filters` section.
* Fixed auto-application of metric settings in [QL charts](../concepts/chart/ql-charts.md).
